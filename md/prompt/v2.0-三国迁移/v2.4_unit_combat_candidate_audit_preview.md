# v2.4 军队接战候选交战审计预判兼容层提示词

## 1. 目标

继续强化“武将、道路、交战”的可读链路：军队详情“接战预判”已经能对最多三名射程内敌军显示伤害、敌余、风险、态势、敌将和武将修正，本轮把每个候选目标对应的 `CombatAuditSummary` 也并入候选行，让玩家比较目标时能直接看到地形、河流、攻城、围城、死守和侧击等因素，而不是只在首选目标后看到一条交战审计。

## 2. 范围

- 修改 `WWIIHexV0/App/AppContainer.swift` 的 `selectedUnitCombatPreviewNotes` 只读派生文本。
- 复用已有 `CombatRules.combatAuditSummary(attacker:defender:in:)` 和 `CombatAuditSummary.logFragment`。
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改 `UnitInspectorView`；“接战预判”已经按内容展开显示。

## 3. 实现要求

- `previews` 中每个射程内候选目标已经持有 `audit: CombatAuditSummary`，候选行应使用对应目标自己的 audit。
- `combatTargetPreviewLine` 增加 `audit` 参数，并在 `audit.logFragment` 非空时追加到当前候选行。
- 移除或避免原先只对 `leadingPreview.audit` 单独追加的重复审计行，防止首选目标重复显示。
- 保留既有首选理由、接战官道、交战武将、候选敌将、候选武将修正、强度结算、风险、态势和候选数量提示。

## 4. 禁止项

- 不改 `CombatRules`、`MovementRules`、`GeneralInfluence` 的规则结果。
- 不改 `CommandValidator`、`CommandExecutor`、`WarCommandExecutor`、`RuleEngine`、真实移动、攻击、反击、撤退、补给、围城、道路或动态战区。
- 不新增路径安全、攻击合法性、隐藏敌军推断、胜率或敌方回合模拟。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 军队详情“接战预判”的前三个射程内候选目标行各自显示自己的交战审计因素。
- 首选目标不再重复显示单独审计行。
- 真实伤害、反击、补给、道路、围城、命令执行、日志结算和 Codable schema 保持不变。
- 文档明确该摘要只读复用真实交战审计来源，不代表完整胜率、路径安全或敌方回合预测。
