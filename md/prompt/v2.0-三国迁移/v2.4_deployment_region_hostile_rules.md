# v2.4 部署层与 Region 命令敌对边界提示词

## 1. 目标

继续收紧三国多势力、武将、道路和交战边界：部署层的前线接触和旧 Region 命令校验仍有少量“不是本方就算敌方”的判断。本轮只修明确的敌对防区接触、敌军存在和 RegionCommand attack 校验，让中立或非 hostile 单位不再生成敌方前线接触或通过 region 层攻击校验。

## 2. 范围

- 修改真实规则兼容层：
  - `WWIIHexV0/Rules/WarDeploymentManager.swift`
  - `WWIIHexV0/Rules/RegionCommandValidator.swift`
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`

## 3. 实现要求

- `WarDeploymentManager.rebuildSegments` 的单位级 `hasEnemyPresence` 使用 `division.faction.isHostile(to: zone.faction)`。
- `WarDeploymentManager.enemyZoneIdsTouching`、`regionTouchesEnemyZone` 和 `hexTouchesEnemyZone` 的相邻防区接触使用 `neighborFaction.isHostile(to: faction)`。
- `RegionCommandValidator` 的 `.attack` 目标阵营校验对齐 hex `CommandValidator.validateAttack`，使用 `target.faction.isHostile(to: attacker.faction)`。
- 保留非己方控制 hex/region 的部署压力分类不动，例如 `controller != faction` 或单位站在非己方控制 hex 的判断；这些表示未控制、争夺或前沿压力，不等同于敌军单位。

## 4. 禁止项

- 不修改 `Faction.isHostile(to:)` 语义。
- 不修改 `CommandValidator.validateAttack`、`RuleEngine`、`CommandExecutor`、`MovementRules`、`SupplyRules` 或真实伤害/移动/补给规则。
- 不把 `region.controller != faction`、`tile.controller != faction` 等非所属判断批量替换为 hostile。
- 不实现完整借道、同盟通行、共同作战堆叠、补给共享或外交宣战制度。
- 不新增持久字段，不修改 JSON schema，不改 Xcode project。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 部署层前线 segment、敌对防区接触和敌军存在判断不再由中立或非 hostile 单位/防区触发。
- RegionCommand attack 校验不再允许中立或非 hostile 目标通过。
- 非己方控制 hex/region 的部署压力、前沿分类和突破推进权威保持原样。
- 真实攻击、移动、道路、粮道、补给、伤害、部署推进和 AI 执行链路没有绕过 `Command` / `RuleEngine`。
