# v2.4 武将面板计划军令近敌对象摘要提示词

## 1. 目标

继续强化“武将、道路、交战”的可读链路：在 `GeneralCommandPanelView` 的“计划军令”摘要中，把源/目标代表 hex 的最近敌距从纯数字升级为“最近敌军对象 + 距离”，让玩家知道计划军令面对的是哪支敌对军队。

## 2. 范围

- 仅修改 `AppContainer` 内计划军令摘要的只读派生：
  - `plannedOperationEnemyDistanceText(for:)`
  - 相关 private nearest helper
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改 `PlayerPlannedOperation` schema。
- 不修改 `GeneralCommandPanelView` API 或计划军令 row 结构。
- 不修改 `BoardScene`、`MovementRules`、`CombatRules`、`GeneralInfluence`、`CommandValidator`、`RuleEngine`、`WarCommandExecutor` 或真实执行规则。

## 3. 实现要求

- 继续使用源/目标代表 hex 计算当前静态距离，不新增路径搜索或胜率计算。
- 最近敌军只统计 `Faction.isHostile(to:)` 判定为敌对且未溃散的军队。
- 最近敌军 tie-break 必须稳定：距离优先，其次军队展示名，其次 id。
- 有源和目标时显示源/目标各自最近敌军对象与距离，例如 `近敌：源曹骑兵军距2格/目袁步卒营距1格`。
- 只有目标或只有源时显示对应对象与距离。
- 无 hostile 敌对军队时不追加近敌摘要，保持既有无噪声行为。
- 摘要只读展示当前代表 hex 到最近敌军的静态距离，不代表完整行军路径安全、移动后攻击、敌方回合反制、隐藏敌军、同盟借道或真实胜率。

## 4. 禁止项

- 不改敌对关系计算，不改 `Faction.isHostile(to:)`。
- 不新增攻击合法性、路径安全、敌控区穿越、隐藏敌军推断或胜率计算。
- 不在 UI 层直接改 `GameState`。
- 不修改 public API 或 Codable schema。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 计划军令摘要能显示源/目标代表 hex 当前最近 hostile 敌军对象和距离。
- 无 hostile 敌军时不新增空噪声。
- 计划军令 row 结构和 `GeneralCommandPanelView` API 不变。
- 改动不影响真实移动、道路、敌控区、补给、交战、命令执行或 AI 战术塑形规则。
