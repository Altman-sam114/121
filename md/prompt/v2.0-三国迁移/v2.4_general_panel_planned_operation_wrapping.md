# v2.4 武将面板计划军令换行兼容层提示词

## 1. 目标

继续强化“武将、道路、交战”的可读链路：上一轮计划军令摘要已经包含武将、最终战术、源→目标、官道状态、源目受压和近敌对象/距离，本轮只改 `GeneralCommandPanelView` 的展示约束，降低长摘要在窄屏、长武将名、长郡县名和长军队名下被截断的概率。

## 2. 范围

- 仅修改 `WWIIHexV0/UI/GeneralCommandPanelView.swift` 的计划军令行展示。
- 可同步文档：
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改 `plannedOperationRows` tuple/API。
- 不修改 `AppContainer.selectedGeneralPlannedOperationRows` 或近敌/道路/受压摘要生成。
- 不修改 `PlayerPlannedOperation` Codable schema。

## 3. 实现要求

- 保留计划军令行的 SF Symbol 图标和原始摘要文本。
- 取消固定 4 行截断，让文本按内容纵向展开。
- 使用 SwiftUI 只读布局修饰，避免 UI 直接推导道路、敌军压迫、近敌距离或战术信息。
- 面板本身已处在 `RootGameView` 的 `ScrollView` 内，长计划军令通过滚动承接，不新增滚动容器。

## 4. 禁止项

- 不改 `MovementRules`、`CombatRules`、`GeneralInfluence`、`WarCommandExecutor`、`CommandValidator`、`RuleEngine`。
- 不改变真实移动成本、道路加成、敌控区、攻击、反击、撤退、补给、围城或粮道规则。
- 不新增路径安全、攻击合法性、隐藏敌军推断、胜率或敌方回合模拟。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 武将面板计划军令行不再固定截断为 4 行，长摘要可按内容换行展开。
- 计划军令文本、图标、API、App 层摘要、真实道路/交战/补给和命令执行规则保持不变。
- 文档明确本轮只是只读 UI 可读性改进，运行时 UI 高度与滚动体验仍需云端 CI、后续 Agent C artifact 复判或人工授权运行检查。
