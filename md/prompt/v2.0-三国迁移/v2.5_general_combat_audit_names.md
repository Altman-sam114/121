# v2.5 武将交战审计姓名提示词

## 目标

继续推进“武将、道路、交战”可解释性：上一轮 `CombatRules.combatAuditSummary` 已把武将攻防修正写入审计 factors，本轮把对应武将展示名也写入同一 factors，让攻击日志、反击日志、军队详情接战候选和交战审计更明确显示“哪个武将”影响了攻防。

## 范围

- 修改 `WWIIHexV0/Rules/CombatRules.swift`。
- 更新 `md/prompt/README.md` 和 `update_log.md`。

## 非目标

- 不改变 `GeneralInfluence.attackBonus` / `defenseBonus` 数值规则。
- 不改变伤害、反击、撤退、围城、道路、粮道、命令校验或执行链路。
- 不重构 `AppContainer`、`UnitInspectorView`、`GeneralCommandPanelView` 或战报 UI。
- 不改 JSON schema、武将数据、地图数据或 Xcode project。

## 架构依据

- 武将交战影响仍通过 `GeneralInfluence -> CombatRules` 生效。
- `GeneralCombatInfluenceSummary` 已提供攻守方武将展示名和攻防 bonus。
- `CombatAuditSummary.logFragment` 是攻击日志、反击日志和只读接战预览共享的审计文本来源之一。

## 实现要求

1. `combatAuditSummary(attacker:defender:in:)` 先生成一次 `GeneralCombatInfluenceSummary`，再传入 attack / defense profile，避免审计文本与武将修正来源分叉。
2. 攻方非零武将修正 factor 显示为 `攻方武将 <姓名> +N/-N`。
3. 守方非零武将修正 factor 显示为 `守方武将 <姓名> +N/-N`。
4. 展示名缺失时保留“未命名武将”或省略姓名，不暴露 raw `generalId`。
5. `attackDamage` / `counterAttackDamage` 的数值计算保持原有公式和默认 profile 调用兼容。

## 验收标准

- 有非零攻方武将修正时，交战审计 factors 含攻方武将展示名和 signed bonus。
- 有非零守方武将修正时，交战审计 factors 含守方武将展示名和 signed bonus。
- 无展示名时不泄漏 raw id。
- 真实战斗数值、道路、命令管线和数据 schema 保持不变。
