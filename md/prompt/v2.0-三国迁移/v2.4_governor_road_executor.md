# v2.4 太守修路命令兼容层

本文记录“太守会建议修路但不会真正落到规则系统”的小步实现提示词与交付边界。执行前仍必须阅读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和总提示词 `md/prompt/v2.0-三国迁移/codex-v2.0-三国aiagent迁移总提示词.md`。

## 目标

- 沿续“注重武将，做好道路、交战”的 v2.4 方向，让太守层的“修路”不只停留在审计文案。
- 新增 `Command.improveRoad(regionId:)`，经 `CommandValidator -> CommandExecutor -> RuleEngine` 校验执行。
- 允许当前行动势力在已控制且有己方控制 hex 的郡县修缮道路，消耗人口、军械和粮草。
- 执行时最多补两格战术道路，并把该郡县基础设施提升 1 点，上限 5。
- `GovernorAgent` 仍只写建议；`TurnManager` 在 `roadRepair` 焦点下把首个重点郡县转换为 `Command.improveRoad`，再按原有生产建议尝试 `Command.queueProduction`。

## 非目标

- 不实现完整修路 UI、玩家手动修路按钮、道路规划编辑器或跨郡县道路网络算法。
- 不修改 `regionToTheater`、`hexToTheater`、`hexToFrontZone`、占领权、军队坐标或战争指令执行规则。
- 不实现屯田、治安、贡赋、借道或臣属制度的完整内政外交规则。
- 不跑本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构边界

```text
GovernorAgent.plan
  -> GovernorDecisionRecord(focus: roadRepair)
  -> TurnManager.governorCommands
  -> Command.improveRoad(regionId)
  -> CommandValidator.validateRoadImprovement
  -> CommandExecutor.executeImproveRoad
  -> EconomyRules.improveRoad
  -> MapState.tiles[].hasRoad / RegionNode.infrastructure / EconomyState ledger
```

- 太守层仍不直接改 `GameState`。
- 修路命令只改道路、基础设施和经济库存，不推进战区或防区。
- 若修路消耗资源导致后续生产建议资源不足，生产命令会被规则层拒绝并写入中文命令结果。

## 验收标准

- `Command.improveRoad` 能被校验、执行和记录。
- 太守 `roadRepair` 焦点会生成修路命令，命令结果进入 `AgentDecisionRecord.commandResults`。
- 修路只消耗资源、补战术道路和提升郡县基础设施；不绕过统一规则系统。
- 文档说明从“修路只审计”更新为“修路最小规则命令已接入”。

## 风险

- 本轮只做最小道路修缮，不保证自动连成跨郡县主干路。
- 未做运行时 UI 烟测，AI 面板多条太守命令的实际换行和日志宽度仍待云端 CI、后续 Agent C artifact 复判或人工授权运行检查。
