# v2.4 Region 胜负分析复用官渡剧本条件提示词

## 目标

上一轮已让运行时 `VictoryRules` 支持官渡剧本 JSON 中 active 的 `controlObjective` 胜负条件。本轮继续收束兼容层，让 `RegionVictoryRules` / `RegionRuleSystem` 的只读战略分析也复用 `VictoryState.scenarioConditions`，避免 region 层仍只按旧阿登 `Bastogne / St. Vith` 城市名评估胜负。

## 范围

- `RegionVictoryAssessment` 增加可选中文原因说明，但保持 winner / reason 主字段兼容。
- `RegionVictoryRules.assessVictory(in:)` 先评估 active `controlObjective` 剧本条件。
- 剧本条件按 objective id 找到对应 hex，并读取 `HexTile.controller` 作为胜负控制权依据。
- 没有命中剧本条件时，继续保留旧阿登 region 城市名 fallback。
- 同步 README、`md/flow/*`、`update_log.md` 和 prompt 索引。

## 非目标

- 不让 `RegionVictoryRules` 修改 `GameState`。
- 不实现 `turns`、`turn`、`count`、`targetTemplateIds`、多目标计分或胜利详情 UI。
- 不删除旧 `Bastogne / St. Vith` fallback 和历史测试语义。
- 不改变移动、攻击、占领、道路、粮道、外交、武将或 AI 执行规则。

## 架构依据

- Hex 是战术权威；objective 控制权以 objective 所在 hex controller 为准。
- Region 是战略分析层，只读评估可以引用同一条件，但不能替代 hex 控制权。
- `RegionRuleSystem.analyze(_:)` 是分析/摘要入口，不是执行入口。
- 玩家、AI 和聊天命令仍必须通过 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine` 修改真实状态。

- `git diff --check`

## 验收标准

- `RegionVictoryRules` 对带有 `VictoryState.scenarioConditions` 的官渡状态优先使用剧本条件。
- 旧阿登 region 城市名 fallback 仍可返回原有 `VictoryReason`。
- `RegionVictoryRules` 只返回 `RegionVictoryAssessment`，不修改 `GameState.victoryState`。
- 文档说明运行时胜负和 region 只读分析已使用同一剧本条件边界。
