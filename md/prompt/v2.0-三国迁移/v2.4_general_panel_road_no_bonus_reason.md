# v2.4 武将面板官道无加成原因提示兼容层提示词

## 1. 目标

继续强化“武将、道路、交战”的可读链路：武将面板“道路与交战”已经能显示官道机动数量、最高加成和具体受益军队。本轮补齐反向解释：当当前选中武将麾下没有任何军队获得官道机动加成时，给出一条只读原因提示，说明是忠诚/满意不足、当前没有可借官道，还是需要进驻官道或依赖粮道/疾行/骑战类技能。

## 2. 范围

- 仅修改 `WWIIHexV0/App/AppContainer.swift` 的 `selectedGeneralInfluenceNotes` 只读摘要生成。
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`

## 3. 实现要求

- 继续以 `GeneralInfluence.movementSummary(for:in:)` 的 `roadBonus` 结果作为是否有官道受益的权威，不重新计算真实加成。
- 仅在 `maxRoadBonus == 0` 且已有麾下军队时追加“道路受益：...”原因提示。
- 原因提示可以只读参考 `GeneralAssignment.loyalty/satisfaction`、技能快照、当前军队所在 hex 是否有路、所在郡县是否有路。
- 提示必须短，服务玩家判断下一步是调忠诚/满意、把军队移到官道，还是依赖粮道/疾行/骑战技能。
- 若无法确定原因，不输出误导性提示。

## 4. 禁止项

- 不修改 `GeneralInfluence`、`MovementRules`、`CombatRules`、`SupplyRules`、`CommandValidator`、`RuleEngine`、`CommandExecutor`、真实移动、道路、敌控区、补给或交战规则。
- 不新增路径搜索、完整可达性、安全路径、胜率、隐藏敌军、同盟借道或敌方回合模拟。
- 不在 UI/App 层直接修改 `GameState`。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 武将面板在没有官道机动加成时，能显示简短原因提示。
- 已有官道受益军队摘要不变；有加成时不新增无加成原因噪声。
- 真实道路机动、路径、敌控区、粮道、交战、命令执行和日志结算保持不变。
- 文档明确该原因提示只是只读解释，不代表完整路径安全或可达性计算。
