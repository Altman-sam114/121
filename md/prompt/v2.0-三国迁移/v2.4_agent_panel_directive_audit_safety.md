# v2.4 AgentPanel 防区指令审计展示安全补丁

## 目标

继续推进三国迁移中的武将、道路、交战可见审计闭环，让 `AgentPanelView` 的防区指令卡片更清楚展示每条防区指令落出的命令类别与执行/拒绝计数，同时继续压制旧二战/阿登兼容调试词进入玩家可见面板。

## 范围

- 只修改 `WWIIHexV0/UI/AgentPanelView.swift` 的展示逻辑。
- 可新增本阶段 prompt，并同步 `md/prompt/README.md` 和 `update_log.md`。
- 允许复用 `CommandResultSummary.commandDisplayNameForDisplay`，把旧记录中的 `move`、`attack`、raw command name 映射为进军、交战、固守、修路、外交等中文安全类别。
- 防区指令卡片可追加类似“进军命令 2 成功 / 交战命令 1 拒绝”的只读摘要。
- 底部审计摘要可把 `directiveRecords` 条数纳入总览。

## 禁止项

- 不修改 `WarDirectiveRecord` schema。
- 不修改 `CommandResultSummary` schema。
- 不修改 `Command.displayName`、`Command.auditDisplayName` 或命令执行结果结构。
- 不修改 `TurnManager`、`WarCommandExecutor`、`RuleEngine`、道路、粮道、交战、外交敌对判断或 JSON 数据。
- 不运行 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构依据

- 玩家、AI 和未来聊天命令必须落到 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine`。
- `AgentPanelView` 只是审计 UI，不能重新推导规则，也不能改变执行结果。
- 新生成的命令结果应使用 `Command.auditDisplayName`；旧记录兼容展示通过 `CommandResultSummary.commandDisplayNameForDisplay` 收口。
- 原始 JSON、raw id、旧二战地名和机器字段仍可留在 Codable / debug 记录里，但不应默认显示在玩家面板。

## 实现步骤

1. 在防区指令卡片中读取 `directive.commandResults`。
2. 按 `commandDisplayNameForDisplay` 分组，统计每类命令的成功和拒绝数量。
3. 在防区指令卡片中追加一行中文摘要；无命令结果时不显示。
4. 扩展 `isRawDebugText(...)` 的旧题材过滤，覆盖 `Bastogne`、`St. Vith`、`Ardennes`、`German`、`Allied`、`Guderian`、`Heinz`。
5. 审计摘要中纳入 `directiveRecords.count`，但不展开 raw JSON。
6. 同步 prompt 索引和更新日志。

## 验收标准

- 防区指令卡片能显示按中文命令类别聚合的成功/拒绝摘要。
- 旧二战/阿登关键字不会作为普通 rationale、diagnostics 或 error 文案直接进入玩家可见面板。
- 审计摘要能体现防区指令条数。
- 本轮不改变任何规则、数据、命令 schema 或执行链。
