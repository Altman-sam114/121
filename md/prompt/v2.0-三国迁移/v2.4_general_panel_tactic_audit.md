# v2.4 武将战术 AI 面板审计提示词

> 本阶段承接 `v2.4_general_tactic_shaping.md`。执行前必须先读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和相关源码。

## 目标

- 让 AI 面板能直接解释武将层“做了什么战术选择”。
- 在 `AgentPanelView` 的武将审计块中展示 `GeneralDecisionRecord` 的动作、最终 tactic、风格、目标郡县和 rationale。
- 保持 `GeneralDecisionRecord` schema、规则执行链和战斗计算不变。

## 范围

- 主要文件：`WWIIHexV0/UI/AgentPanelView.swift`。
- 文档文件：`README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md`、`update_log.md`。
- 不修改 Xcode project、不新增资源、不改 `GeneralAgent`、`WarCommandExecutor` 或 `RuleEngine`。

## 实现要求

- 现有“武将”列表继续限制最多展示前 4 条记录，避免面板过长。
- 每条武将记录至少展示：
  - 武将姓名或 fallback id。
  - `action`。
  - 防区 id。
  - 攻守类型。
  - `tactic.displayName`，缺失时显示未定战术。
  - 指挥风格中文名，缺失时显示未定风格。
  - 目标郡县 raw id 列表，缺失时显示无目标。
  - `rationale`，最多三行，避免挤压后续内容。
- 使用 Dynamic Type 字体级别，不引入固定字号或图标按钮。

## 禁止项

- 不允许 UI 直接执行命令或修改 `GameState`。
- 不允许改 `GeneralDecisionRecord` Codable 字段。
- 不允许新增本机 Xcode build/test、模拟器、Probe、Smoke、Stage Regression、Dynamic Theater Regression、Full 或性能测试。
