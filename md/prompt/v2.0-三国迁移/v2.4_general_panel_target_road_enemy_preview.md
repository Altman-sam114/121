# v2.4 武将面板目标郡县官道与近敌预览提示词

## 1. 目标

继续强化“武将、道路、交战”的玩家决策闭环：在 `GeneralCommandPanelView` 中，当玩家已选中敌对目标郡县但尚未提交“进攻郡县”军令时，显示源/目标代表 hex 的官道态势和最近可见敌军距离，让玩家在下令前判断目标是否据道、受压和接敌距离。

## 2. 范围

- 仅修改只读 UI 派生与展示：
  - `AppContainer`
  - `GeneralCommandPanelView`
  - `RootGameView`
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改 `PlayerPlannedOperation` schema，不修改地图计划军令绘制，不修改真实移动、攻击、敌控区、道路修缮、粮道、补给、外交或命令执行规则。

## 3. 实现要求

- `AppContainer` 新增只读 `selectedGeneralTargetPreviewNotes`。
- 仅在当前武将防区存在、选中目标郡县存在、目标防区对当前防区 hostile 时生成预览。
- 目标官道摘要读取当前源/目标代表 hex：
  - 源来自当前选中防区和目标防区推导出的 `sourceRegionId`。
  - 目标来自选中目标郡县 representative hex。
  - 展示 `据道` / `离道`，若该代表 hex 被当前 viewer 可见 hostile 军队邻近或占据，追加 `/受压`。
- 目标近敌摘要只统计当前 viewer 可见、未毁灭、且 `Faction.isHostile(to:)` 判定 hostile 的敌军。
- `GeneralCommandPanelView` 只接收并展示预览 notes，不在 View 内推导规则。
- `RootGameView` 在所有 `GeneralCommandPanelView` 入口传入新预览 notes。
- 文案必须保持只读预览口径，不承诺完整路径安全、真实攻击合法性、胜率、同盟借道或敌方回合反制。

## 4. 禁止项

- 不改 `Command` / `ZoneDirective` / `WarCommandExecutor` / `CommandValidator` / `RuleEngine`。
- 不改 `MovementRules`、`CombatRules`、`SupplyRules`、`GeneralInfluence` 的规则数值或公式。
- 不新增持久字段，不修改 `PlayerPlannedOperation` Codable schema。
- 不让不可见敌军进入目标近敌或受压摘要。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 武将面板在选中敌对目标郡县时，能在目标名下方显示提交前的目标官道和目标近敌预览。
- 官道受压和近敌摘要只来自当前 viewer 可见 hostile 敌军，不泄漏隐藏敌军。
- 计划军令提交、地图标签、移动、攻击、道路、粮道、补给、外交和命令执行规则不变。
- 文档明确这是提交前只读态势预览，不是完整路径安全、胜率或攻击许可制度。
