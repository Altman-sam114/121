# v2.4 军队接战官道压制预判兼容层

## 目标

在军队详情的“接战预判”中，为首选目标增加一条只读“接战官道”摘要，把道路机动与交战判断连起来：本回合有多少个官道位置可以压制首选目标，其中多少相对安全，多少受敌控区压迫。

## 范围

- 修改 `WWIIHexV0/App/AppContainer.swift` 的 `selectedUnitCombatPreviewNotes` 派生文本。
- 复用 `MovementRules.movementRange` 和 `MovementRules.isEnemyZoneOfControl`。
- 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 禁止项

- 不改 `MovementRules` 的真实移动范围、道路成本、敌控区停止规则或武将道路加成。
- 不改 `CombatRules` 伤害、反击、撤退、死守、围城或兵种克制公式。
- 不改 `CommandExecutor`、`CommandValidator`、`RuleEngine` 或真实命令执行结果。
- 不新增道路规划状态、路径缓存、JSON schema、跨回合粮道评分或胜率模型。

## 当前架构依据

- Hex 仍是战术权威，`Division.coord` 和 `HexTile.controller` 不由 UI 预览修改。
- `selectedUnitMobilityPreviewNotes` 已展示可达官道总量；本轮只在接战首选目标处补充“哪些官道位能形成压制”的摘要。
- `MovementRules.movementRange` 是本回合可达范围的只读计算；`isEnemyZoneOfControl` 已使用 `Faction.isHostile(to:)`。
- `UnitInspectorView` 只渲染 `AppContainer` 传入的文本，不执行移动或攻击。

## 实现步骤

1. 在 `selectedUnitCombatPreviewNotes` 中创建并复用 `MovementRules`。
2. 对首选目标统计：`movementRange + 当前坐标` 中，位于官道且距离目标不超过当前军队射程的坐标。
3. 用 `isEnemyZoneOfControl` 把官道压制位拆成安全数量和受压数量。
4. 在首选理由之后追加 `接战官道：...` 摘要；三目标候选行、武将影响和交战审计保持原格式。
5. 文档记录本轮只是 UI 预判解释层，不改变真实移动和交战。

## 验收标准

- 选中军队且射程内有敌军时，“接战预判”显示首选目标的接战官道摘要。
- 摘要能区分无可用官道压制位、有官道压制位、安全位和敌控区压迫位。
- 真实移动、攻击、日志结算、命令校验和动态战略同步行为不变。

## 风险提示

- 本轮没有运行 UI 烟测；新增“接战官道”文本在窄屏、长军队名和 Dynamic Type 下的换行仍需云端 CI、后续 Agent C artifact 复判或人工授权运行检查。
- 摘要只看本回合可达官道压制位，不计算最佳路径、跨回合道路计划、完整粮道安全或同盟借道通行。
