# v2.4 军队接战首选理由预判兼容层

## 目标

在军队详情的“接战预判”中，为射程内首选敌军增加一条只读“首选理由”摘要，让玩家能直接看到当前排序依据：本轮可比较敌军数量、预计伤害、战后敌余、撤退/歼灭可能、反击风险和距离。

## 范围

- 修改 `WWIIHexV0/App/AppContainer.swift` 的 `selectedUnitCombatPreviewNotes` 派生文本。
- 保留既有三目标对比行、目标态势、风险、武将影响和交战审计。
- 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 禁止项

- 不改 `CombatRules` 伤害、反击、撤退、死守、围城或兵种克制公式。
- 不改 `CommandExecutor`、`CommandValidator`、`RuleEngine` 或真实命令执行结果。
- 不新增规则状态、存档字段、JSON schema、武将技能树、胜率模型或多回合围攻规划。
- 不绕过 `Command / ZoneDirective -> WarCommandExecutor -> RuleEngine`。

## 当前架构依据

- Hex 仍是战术权威，`Division.coord` 和 `HexTile.controller` 不由 UI 预览修改。
- `selectedUnitCombatPreviewNotes` 只读复用 `CombatRules.attackDamage`、`counterAttackDamage`、`GeneralInfluence` 和 `CombatAuditSummary`。
- 接战候选排序仍按预计伤害高、反击低、距离近、名称/id 稳定排序。
- `UnitInspectorView` 只渲染 `AppContainer` 传入的文本，不执行攻击。

## 实现步骤

1. 在 `selectedUnitCombatPreviewNotes` 取得 `leadingPreview` 后，先插入 `首选理由：...` 摘要。
2. 摘要使用同一套 `combatPreviewOutcome` 估算敌余、撤退、死守额外损失后的可能歼灭和反击可用性。
3. 三目标候选行保持原格式，继续展示伤害、敌余、风险、反击、态势和距离。
4. 文档记录本轮只是 UI 预判解释层，不改变真实结算。

## 验收标准

- 选中军队且射程内有敌军时，“接战预判”首行显示首选理由。
- 首选理由和后续首选/候选行使用同一套只读估算。
- 没有射程内敌军时仍不显示接战预判。
- 真实战斗、日志结算、命令校验和动态战略同步行为不变。

## 风险提示

- 本轮没有运行 UI 烟测；长军队名、窄屏和 Dynamic Type 下的换行仍需云端 CI、后续 Agent C artifact 复判或人工授权运行检查。
- 摘要不是胜率，也不计算撤退目的地、撤退失败、士气、疲劳、单挑或跨回合围攻。
