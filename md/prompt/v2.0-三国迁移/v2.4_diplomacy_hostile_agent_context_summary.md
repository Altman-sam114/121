# v2.4 外交敌对 AI 摘要兼容层提示词

## 1. 目标

继续推进三国多势力、武将、道路和交战迁移：规则入口、部署接触和 UI 预览已逐步接入 `DiplomacyState` hostile / atWar 口径，但 Legacy Agent context 和 MockAI 威胁估算仍有少量单位级敌军摘要直接使用 `Faction.isHostile(to:)`。本轮只迁移 AI 上下文源头的单位敌对过滤，不改 prompt schema、JSON schema 或命令执行管线。

## 2. 范围

- `WWIIHexV0/Agents/AgentContexts.swift`
  - `enemyDivisions` 来源。
  - `SupplySummary` 中 enemy supplied / lowSupply / encircled 统计来源。
- `WWIIHexV0/Agents/MockAICommander.swift`
  - `visibleEnemyStrength` 的单位敌对来源。

## 3. 实现要求

- 使用 `state.diplomacyState.isHostile(between:and:)`，保留缺外交建档 fallback 到 `Faction.isHostile(to:)`。
- 不改 `AgentContext`、`DivisionSummary`、`SupplySummary` Codable schema。
- 不改 `AgentPromptBuilder` 和 `MockAIClient` 消费逻辑；它们继续读取 `context.enemyDivisions`。
- 不改 `ZoneDirective -> WarCommandExecutor -> RuleEngine` 执行收口。

## 4. 禁止项

- 不修改 `Faction.isHostile(to:)` 或 `Faction.opponent`。
- 不改 LLM prompt schema、AgentDecision schema、JSON 数据或 Xcode project。
- 不把 objective / controller / region 占领判断改成外交关系。
- 不在本轮迁移 `ZoneCommanderAgent`、`StrategistAgent`、`RulerAgent` 或 `FrontLineManager` 的更大范围 hostile 判断。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- `AgentContext.enemyDivisions` 不再把外交非敌对单位放入敌军摘要。
- `SupplySummary` 的 enemy 统计不再把外交非敌对单位计入敌军补给态势。
- `MockAICommander.visibleEnemyStrength` 不再把外交非敌对单位计入威胁强度。
- Prompt / Codable schema / 命令执行链路不变。
- 本机只跑轻量检查，重验证交给 GitHub Actions。
