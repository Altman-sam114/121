# v2.4 HUD、菜单与军机面板显示三国化提示词

## 目标

继续围绕“武将、道路、交战”的可读复判推进三国迁移，把主 HUD、新局入口、macOS 菜单、信息面板开关、军机审计面板和 AI 回合交互日志中的残留英文或旧 `AI` 外显词收束为中文三国语义。

## 范围

- `WWIIHexV0/UI/NewGameButton.swift`
  - 新局按钮显示为“新开战局”。
- `WWIIHexV0/App/WWIIHexV0MacApp.swift`
  - macOS 菜单和命令显示为“战局”“结束回合”“新开战局”。
- `WWIIHexV0/UI/RootGameView.swift`
  - 信息面板开关显示为“军情”，紧凑 tab 中 `AI` 改为“军机”。
  - 地图 accessibility label 使用“三国六角战场”，避免主界面辅助文本继续出现工程术语。
- `WWIIHexV0/UI/InfoPanelToggle.swift`
  - 备用信息面板开关显示为“军情”。
- `WWIIHexV0/UI/AgentPanelView.swift`
  - 标题和主字段显示为“军机谋议”“执行者”，命令 fallback 显示为“军令”。
- `WWIIHexV0/App/AppContainer.swift`
  - AI 回合消息显示为“军机回合”。
  - `MockAI` / `System` 来源和 no-op 摘要显示为模拟军机、系统和中文军机提示。
  - 兼容 fallback 指挥者显示为军机将领/观察军机/防区军机，不再把 raw zone id 放进玩家可见名称。
  - 点击军队日志使用 `Division.thematicDisplayName`，避免旧单位名泄漏到玩家交互日志。
  - 郡县选择日志不再直接裸露 `RegionId.rawValue`。
- 文档同步：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`

## 非目标

- 不改 `AgentDecisionRecord`、`WarDirectiveRecord`、`CommandResultSummary`、`Command`、`ZoneDirective`、`RuleEngine`、`WarCommandExecutor` 或 JSON / Codable schema。
- 不改 AI 决策、回合推进、命令执行、武将塑形、道路机动、交战计算、补给或外交 hostile 规则。
- 不迁移 `DiplomacyPanelView`、单位兵牌符号、检查器 raw id、数据文件名称或源码类型名。
- 不运行 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 当前架构依据

- `RootGameView` 和 HUD 只负责显示与用户入口，不应写规则。
- `AgentPanelView` 是审计面板，读取记录并展示；调试 JSON 和底层 raw id 仍保留给 Codable / 回归审计。
- `AppContainer.interactionLog` 是玩家可见交互日志边界；本轮只改文案与展示名，不改变 `GameState` 状态写入。
- 军队显示名已有 `Division.thematicDisplayName`，可在中文名缺失时按势力与兵种生成三国语义 fallback。

## 实现步骤

1. 替换 HUD 新局按钮、macOS 菜单和信息面板开关文案。
2. 将紧凑面板 tab `AI` 改为“军机”，将 `AgentPanelView` 标题和主字段改为三国语境。
3. 将命令结果 fallback 从 `Order` 改为“军令”。
4. 将 `AppContainer` AI 回合消息改为“军机回合”。
5. 将本地 mock / no-op 来源、摘要和兼容 fallback 指挥者名称改为军机语义。
6. 将军队点击日志从 `division.name` 改为 `division.thematicDisplayName`。
7. 将郡县选择日志改为只显示郡县名，避免 raw id 作为主 UI 文案。
8. 更新 README、核心流程文档、流程图、prompt 索引和 `update_log.md`。

- 目标 UI 文件中指定英文残留扫描无命中。

- 规则行为、命令管线、AI 决策、记录 schema、rawValue 和调试 JSON 保持不变。
- 文档、prompt 索引和 `update_log.md` 同步记录本轮范围、轻量检查和未跑重测试原因。
