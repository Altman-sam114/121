# v2.4 军队未接敌时最近敌军态势预判兼容层

## 目标

当选中军队当前射程内没有敌军时，在“接战预判”的最近敌军接近信息中补充敌军态势，让玩家在靠近前看到目标地形、据城/据关、临官道、粮道状态和撤退姿态。

## 范围

- 修改 `WWIIHexV0/App/AppContainer.swift` 的 `combatOutOfRangePreviewNotes` 派生文本。
- 复用既有 `combatTargetStanceText(for:)`，与射程内候选目标行使用同一套态势摘要。
- 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 禁止项

- 不改 `HexTile`、`SupplyState`、`RetreatMode`、`MovementRules`、`CombatRules` 或 `GeneralInfluence`。
- 不改攻击、反击、撤退、死守、围城、道路、粮道或补给规则。
- 不改 `CommandExecutor`、`CommandValidator`、`WarCommandExecutor`、`RuleEngine` 或真实命令执行结果。
- 不新增完整胜率、跨回合追击、路径安全、同盟借道或 AI 决策变化。

## 当前架构依据

- 射程内候选目标行已经通过 `combatTargetStanceText(for:)` 展示目标地形、据城/据关、临官道、补给短名和撤退姿态短码。
- 未接敌接近预判由 `selectedUnitCombatPreviewNotes -> combatOutOfRangePreviewNotes` 生成，当前已展示最近敌军、接近武将、接近参考和官道接近信息。
- 这些预览都只读读取 `gameState` 与规则派生，不执行命令、不修改地图、军队、道路、补给、兵力或动态战区。

## 实现步骤

1. 在 `combatOutOfRangePreviewNotes` 中取得最近敌军后，调用 `combatTargetStanceText(for: nearest.target)`。
2. 若存在态势摘要，追加 `接近态势：...`。
3. 保留既有接战距离、接近武将、接近参考、官道接近和官道入射程判断。
4. 射程内存在敌军时保持既有三目标接战预判不变。

## 验收标准

- 选中军队且当前射程内没有敌军但地图上存在敌军时，接战预判显示最近敌军的 `接近态势：...`。
- `接近态势` 与射程内候选目标行的态势摘要来源一致。
- 射程内有敌军时既有三目标接战预判保持不变。
- 真实移动、攻击、补给、道路、日志结算、命令校验和动态战略同步行为不变。

## 风险提示

- 本轮没有运行 UI 烟测；新增“接近态势”文本在窄屏、长军队名和 Dynamic Type 下的换行仍需云端 CI、后续 Agent C artifact 复判或人工授权运行检查。
- 摘要只读展示当前最近敌军的站位状态，不模拟移动后同回合攻击、跨回合追击、完整路径安全、同盟借道或真实胜率。
