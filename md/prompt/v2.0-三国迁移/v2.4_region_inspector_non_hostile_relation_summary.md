# v2.4 郡县检查器非敌对关系摘要提示词

## 1. 目标

继续强化三国多势力态势可读性：在 `RegionInspectorView` 已能分开显示可见敌军和可见非敌对军队的基础上，给可见非敌对军队追加势力、外交关系、紧张度和非敌对标记，让玩家查看郡县时知道这些军队为什么不是当前交战对象。

## 2. 范围

- 仅修改只读 UI 派生与展示：
  - `MapDisplayAdapter.inspectorState`
  - `RegionInspectorState`
  - `RegionInspectorView`
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改地图、外交执行、移动、攻击、道路、粮道、补给或命令执行规则。

## 3. 实现要求

- `RegionInspectorState` 新增只读 `visibleNonHostileRelationSummaries`。
- 摘要只从当前可见、非己方、非 hostile 且未毁灭的军队生成，最多显示三条。
- 每条摘要应包含：
  - 军队名。
  - 所属势力短名。
  - 当前外交关系状态和紧张度；若 `DiplomacyState` 没有对应 country / relation，则显示“关系未建档”。
  - “非敌对”标记。
  - 若能从现有武将分配快照取到武将名，可追加武将名。
- `RegionInspectorView` 新增“非敌对关系”行，继续只消费 `RegionInspectorState` 字符串，不在 View 内推导外交关系。

## 4. 禁止项

- 不改 `Faction.isHostile(to:)` 当前最小实现。
- 不改 `DiplomacyState` schema、外交命令、关系更新规则或 `Command.proposeDiplomacy`。
- 不把非敌对军队加入合法攻击目标、敌控区、粮道阻断、道路压迫或补员阻断。
- 不修改 `MovementRules`、`CombatRules`、`SupplyRules`、`CommandValidator`、`RuleEngine`、`CommandExecutor`、`TurnManager`。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 郡县检查器继续区分“可见敌军”和“可见非敌对军队”。
- “非敌对关系”能显示可见非敌对军队的势力、外交关系/紧张度或“关系未建档”、非敌对标记和可用武将名。
- 真实攻击、敌控区、道路压迫、粮道、补给、外交命令和规则执行行为不变。
- 文档明确这是只读态势摘要，不是完整同盟、借道、停战或臣属制度。
