# v2.4 - 军机面板与方面展示安全收敛提示词

## 目标

继续推进三国迁移的玩家可见审计边界，把军机面板中 raw JSON、英文兼容 id、内部方面 rawValue 和旧二战词的直出风险收敛为中文三国语义，同时让固定方面节点默认显示为“西北方面 / 东北方面 / 西南方面 / 东南方面”。

## 范围

- `AgentPanelView`：意图、摘要、命令结果、错误、防区指令诊断和底部审计区只显示玩家可读中文或中文兜底。
- `AgentDecisionRecord` / Legacy parser / 军师 directive decoder：未知执行者、未知 provider、格式错误和版本错误不把 raw id、JSON、UTF-8、schemaVersion 或 decoder 细节作为玩家文案。
- `FixedTheaterKind` / `TheaterSystem`：保留 `NorthWest/NorthEast/SouthWest/SouthEast` 作为 id / Codable rawValue，但 `TheaterNode.name` 使用中文方面名；临时缺失方面用势力简称加“临时方面”。
- 文档：同步 README、核心流程、流程图、prompt 索引和 update_log。

## 非目标

- 不改 `TheaterId`、`FrontZoneId`、`RegionId`、`WarDirectiveRecord`、`AgentDecisionRecord` 或 Codable schema。
- 不删除 `rawJSON` / `debugJSONDisplay` 等审计字段。
- 不改变动态方面推进、道路、粮道、交战、武将塑形、命令校验或规则数值。
- 不做本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 实现要求

1. 基于最新 `origin/main` 和干净 `main` 实现。
2. `AgentPanelView` 增加面板级安全展示 helper，对 raw JSON、明显机器字段、英文兼容 id、旧二战词和固定方面 rawValue 使用中文兜底。
3. 面板底部从默认展示原始 JSON 改为中文审计摘要；原始字段继续留在记录结构里供审计和兼容。
4. `FixedTheaterKind` 提供中文展示名，`TheaterSystem.makeInitialFixedTheaters` 和缺失节点补建使用展示名。
5. 更新文档，明确 rawValue 保留但玩家可见 name 已中文化。

- 文档尾随空白和冲突标记扫描。
- prompt 索引差集检查。
- `git diff --check`。

## 验收标准

- AI 面板不再默认展开 raw JSON。
- 命令错误、mapping 失败和 directive diagnostics 不把明显 raw id / JSON / 英文兼容字段直出给玩家。
- 固定方面节点显示中文方面名；底层 rawValue 和 id 兼容不变。
- 文档与源码口径一致，说明未跑本机重测试并等待云端 CI。
