# v2.4 Legacy Agent D 错误文案三国化提示词

本文记录 Legacy Agent D parser / mapper 缺字段和解析错误仍暴露英文诊断的小步实现提示词。执行前仍必须阅读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和总提示词 `md/prompt/v2.0-三国迁移/codex-v2.0-三国aiagent迁移总提示词.md`。

## 目标

- 将 `AgentDecisionParserError` 和 `AgentCommandMappingError` 的 `LocalizedError.errorDescription` 改为中文三国语义。
- 让 Legacy `LocalLLMDecisionProvider` / Agent D 回归路径启用时，AI 面板、命令诊断或错误日志不再显示 `Move order`、`Attack order`、`Malformed agent decision JSON` 等英文文案。

## 非目标

- 不修改 `AgentDecisionEnvelope`、`AgentOrder`、`schemaVersion`、`agentId`、`turn`、`toRegionId`、`targetDivisionId` 等机器字段或 Codable schema。
- 不修改 parser / mapper 行为、命令生成、命令校验、规则执行、MockAI、Local LLM prompt 或 WarCommandExecutor。
- 不恢复 Legacy Agent D 为默认战争 AI 主路径。

## 实现步骤

1. 修改 `WWIIHexV0/Agents/AgentDecisionParser.swift` 的 `errorDescription` 文案。
2. 修改 `WWIIHexV0/Agents/AgentCommandMapper.swift` 的 `errorDescription` 文案。
3. 同步更新 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 轻量检查

- `swiftc -parse WWIIHexV0/Agents/AgentDecisionParser.swift WWIIHexV0/Agents/AgentCommandMapper.swift`
- 固定字符串扫描确认旧英文错误文案无命中。
- 本轮改动文件尾随空白和冲突标记扫描。
- prompt 索引一致性检查。
- `git diff --check`

## 验收标准

- Parser / mapper 常见错误使用中文军机、军令、军队、目标郡县和目标军队措辞。
- enum case、字段名、schema、parser / mapper 控制流和命令管线无变化。
- 本机只做允许的轻量检查，重验证等待 `origin/main` 云端 CI artifact。
