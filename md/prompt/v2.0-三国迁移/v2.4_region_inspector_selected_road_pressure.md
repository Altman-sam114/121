# v2.4 郡县检查器当前官道受压显示

## 目标

继续推进武将、道路和交战可见审计闭环，把郡县检查器中选中地格的“当前官道”从简单是/否改为玩家视角可见的道路态势，说明该地格是据官道、离官道，还是据官道但受可见敌军压迫。
同时收口同一郡县检查器摘要里的武将展示名 fallback，缺少姓名快照时显示“未命名武将”，不把 raw `generalId` 暴露到本郡武将、敌军接战、官道压迫或非敌对关系摘要。

## 范围

- 修改 `WWIIHexV0/SpriteKit/MapDisplayAdapter.swift` 的 `RegionInspectorState` 只读派生字段。
- 修改 `WWIIHexV0/UI/RegionInspectorView.swift` 的当前官道展示文本。
- 同步 `md/prompt/README.md` 与 `update_log.md`。

## 非目标

- 不改变 `MovementRules`、`GeneralInfluence`、`WarCommandExecutor`、`RuleEngine`、道路修缮、粮道、补给、攻击合法性或 AI 军令落点。
- 不改变 JSON schema、Xcode project、测试文件或旧阿登 fallback。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 实现步骤

1. 在 `RegionInspectorState` 增加面向 UI 的 `selectedHexRoadStatusSummary`。
2. 在 `MapDisplayAdapter.inspectorState(...)` 中复用当前玩家视角可见 hostile 军队集合，只对选中 hex 生成道路态势摘要。
3. 若选中 hex 无官道，显示“离官道”；若有官道且没有可见 hostile 邻近，显示“据官道，未受可见敌军压迫”；若有可见 hostile 邻近，显示压迫来源、距离和敌将展示名。
4. `RegionInspectorView` 的“当前官道”优先显示该摘要，旧布尔字段只作 fallback。
5. `MapDisplayAdapter.generalDisplayName(for:)` 不再 fallback 到 raw `generalId`，缺展示名时统一返回“未命名武将”。

- 本轮改动文件尾随空白扫描。
- 本轮改动文件冲突标记扫描。
- prompt 索引差集检查。
- `git diff --check`。

## 验收标准

- 郡县检查器选中地格时，“当前官道”能说明据官道 / 离官道 / 受可见敌军压迫。
- 压迫判断只使用当前玩家视角可见且外交 hostile / atWar 的军队，不泄漏隐藏敌军，不把非敌对军队算作压迫。
- 郡县检查器相关武将展示名缺失时显示“未命名武将”，不显示 raw `generalId`。
- 不改变任何道路、粮道、移动、交战或命令执行规则。
