# v2.4 武将战术塑形兼容层提示词

> 本阶段承接总提示词 `codex-v2.0-三国aiagent迁移总提示词.md` 中“武将影响 tactic 选择或小幅战斗修正”的要求。执行前必须先读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和相关源码。

## 目标

- 让 `GeneralAgent` 不只调整投入强度和预备队，还能根据防区武将的风格、忠诚、满意度和技能快照塑形 `ZoneDirective.tactic`。
- 攻势指令只能得到攻势 tactic，守势指令只能得到守势 tactic；避免后续 `WarCommandExecutor` 因 tactic/category 不匹配生成空结果。
- 武将战术塑形必须继续走 `DirectiveEnvelope -> ZoneDirective -> WarCommandExecutor -> RuleEngine`，不能由 Agent 直接改 `GameState`。

## 范围

- 主要文件：`WWIIHexV0/Agents/GeneralAgent.swift`。
- 文档文件：`README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md`、`update_log.md`。
- 可新增本阶段提示词文件；不新增 JSON schema、不新增武将数据字段、不修改 Xcode project。

## 实现要求

- `GeneralAgent.plan` 在军师层之后、执行器之前读取 `FrontZone.generalAssignment` 和 `GeneralRegistry`。
- 攻势 tactic 选择参考：
  - 低忠诚或低满意度：优先收束为 `feint`，无法使用时回退 `standardAttack`。
  - 有会师点或跨防区协同：优先 `pincerMovement`。
  - `fortress_operations` / `siegecraft` / `set_piece_attack`：优先 `fireCoverage`，但必须有器械或远程单位条件。
  - `cavalry_charge` / `rapid_exploitation` / `armor_expert` 且风格进取：优先 `blitzkrieg` 或 `spearhead`。
  - `breakthrough` / `counterattack` / `offensive_planning`：可塑形为 `breakthrough`。
  - 谨慎风格可收束为 `feint`。
- 守势 tactic 选择参考：
  - 高压力且无纵深预备队：可 `lastStand`。
  - 低忠诚/满意度或谨慎、防守、参谋、预备队技能：优先 `defenseInDepth`，不可用则 `holdPosition`。
  - 进取风格、无压力且有反击 region：可 `elasticDefense`。
  - `fortress_operations` / `discipline`：可保持 `holdPosition`。
- 候选 tactic 必须用现有 `TacticConditionChecker` 做轻量可用性过滤。
- 如果原 tactic 不存在或类别不匹配，必须回退到对应默认 tactic：攻势 `standardAttack`、守势 `holdPosition`。
- `GeneralDecisionRecord` 应记录塑形后的 tactic；rationale 应说明武将姓名、忠诚/满意度、风格、技能摘要和最终战术。

## 禁止项

- 不允许 Agent 直接修改 `HexTile.controller`、`Division.coord`、资源、道路、生产队列、外交关系、`hexToTheater` 或 `hexToFrontZone`。
- 不允许绕过 `WarCommandExecutor` 或 `RuleEngine`。
- 不新增本机 Xcode build/test、模拟器、Probe、Smoke、Stage Regression、Dynamic Theater Regression、Full 或性能测试。
- 不改变 `TacticName` rawValue / Codable schema。
