# v2.4 RegionSupplyRules 战略粮道 hostile controller 外交化

## 目标

- 继续围绕武将、道路、交战和粮道迁移，把 `RegionSupplyRules` 中战略郡县粮道通行与撤退安全郡县的敌对 controller 判定，从旧二元 `Faction.isHostile(to:)` 改为 `DiplomacyState.isHostile(between:and:)`。
- 让中立、停战、同盟或共同作战势力的 region controller 不再仅因旧阵营二元关系阻断战略粮道或撤退安全郡县。

## 范围

- 只改 `WWIIHexV0/Rules/RegionSupplyRules.swift` 内部 hostile controller gate。
- 同步更新 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md`、总提示词和 `update_log.md`。
- 保持 `RegionSupplyRules` public API、`RegionRuleSystem` 调用方式和 region graph/pathfinder 不变。

## 禁止项

- 不实现完整借道、同盟通行、共同作战堆叠、共享补给源或多势力 turn order。
- 不把 region 变成战术权威；移动、攻击、占领、视野和补给落点仍以 hex 为准。
- 不改变 hex `SupplyRules`、`OccupationRules`、`MovementRules`、`CombatRules` 或 `WarCommandExecutor` 的规则语义。
- 不运行 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构依据

- `RegionSupplyRules` 是战略聚合层分析，读取 `RegionNode.controller` 作为从 hex controller 派生出的战略快照。
- hostile 分类应优先使用 `DiplomacyState`，缺外交建档时由 `DiplomacyState.isHostile` 内部 fallback 到旧 `Faction.isHostile(to:)`。
- 战略粮道可否通过敌对 controller region 的既有例外仍保留：若该 region 内已有己方单位，则允许通行。

## 实现步骤

1. 在 `isSafeRetreatRegion(_:for:in:)` 中，把 `!region.controller.isHostile(to: faction)` 替换为 `!state.diplomacyState.isHostile(between: region.controller, and: faction)`。
2. 在 `canSupplyPass(through:for:in:)` 中，把 `region.controller.isHostile(to: faction)` 替换为同一外交 hostile 判定。
3. 只读确认 `RegionRuleSystem` 不需要 API 变更。
4. 更新文档清单，把 `RegionSupplyRules` 纳入已外交化 hostile controller gate。
5. 新增本阶段提示词并更新 prompt 索引。
