# v2.4 AppContainer 武将军令与军队选择交互日志三国化提示词

## 目标

把 `AppContainer` 中玩家可见的交互日志继续迁移到三国语义，重点覆盖武将军令提交/拒绝、规则拒绝摘要、基础命令执行状态、军队查看/选择和地格/郡县选择提示。用户本轮特别强调“三国要注重武将，做好道路、交战”，因此本切片只处理玩家读得见、会影响武将军令理解的日志文字。

## 范围

- `WWIIHexV0/App/AppContainer.swift`
  - `submit(_:)` 的命令执行/拒绝交互日志。
  - `holdSelected()`、`allowRetreatSelected()`、`resupplySelected()` 的无可行动己方军队拒绝文案。
  - `orderSelectedGeneralHoldLine()`、`orderSelectedGeneralAttackRegion()` 和 `submitPlayerDirective(...)` 的武将军令拒绝、诊断、提交和执行摘要文案。
  - `handleDivisionTap(_:)` 和 `selectionMessage(for:)` 的查看军队、选择军队、选择地格、选择郡县文案。
- 文档同步：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`

## 非目标

- 不改 `Command`、`ZoneDirective`、`WarCommandExecutor`、`CommandValidator`、`RuleEngine` 的执行逻辑。
- 不改 `WarDirectiveRecord`、`PlayerPlannedOperation`、Codable 字段、rawValue、JSON schema 或调试 JSON。
- 不迁移 `Division` 类型名、`FrontZoneId`、`CountryId`、`RegionId` 等内部兼容 id。
- 不新增道路、交战、武将技能或 AI 决策规则。
- 不运行 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 当前架构依据

- 玩家和 AI 命令必须进入 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine`。
- `AppContainer` 负责 UI 交互状态、玩家命令提交、武将面板宏观军令提交和 `interactionLog` 事件追加。
- `submitPlayerDirective(...)` 会先经 `GeneralAgent.plan` 做武将战术塑形，再进入 `WarCommandExecutor`，本轮只能改日志展示，不改变这条执行顺序。
- `frontZoneDisplayName(_:)` 仅作为私有显示 helper，优先使用 `FrontZone.name`，缺失或等于 raw id 时用势力简称和前两个郡县名生成 `曹军防区：官渡、许昌` 一类展示名。

## 实现步骤

1. 将基础命令日志从 `Command accepted/rejected` 改为“命令已执行/命令被拒绝”。
2. 将无选中军队的 `Hold rejected` / `Allow retreat rejected` / `Resupply rejected` 改成固守、允许撤退、补给的中文失败原因。
3. 将 `General order rejected/submitted/executed` 改为“武将军令失败/已提交/已执行”，并把 `command(s)`、`division(s)`、`Player directive generated no executable commands` 改为中文。
4. 将 `Inspecting unit`、`Selected unit`、`Selected hex`、`Selected region` 改为查看军队、已选择军队、已选择地格、已选择郡县。
5. 扫描本切片关键词，确认目标英文不再出现在 `AppContainer.swift` 的玩家交互日志中。
6. 更新 README、flow、flowchart、prompt 索引和 update_log，说明本轮只完成显示层中文化。
