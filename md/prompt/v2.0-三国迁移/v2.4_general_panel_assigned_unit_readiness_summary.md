# v2.4 武将面板麾下军队备战摘要提示词

## 1. 目标

继续强化“武将、道路、交战”的玩家决策闭环：在 `GeneralCommandPanelView` 的“麾下军队”列表中显示每支军队的兵力、粮草、军令和行动状态短摘要，让玩家在下达固守或进攻前能快速判断该武将麾下军队是否适合继续行动或接战。

## 2. 范围

- 仅修改 `WWIIHexV0/UI/GeneralCommandPanelView.swift` 的只读展示。
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改 `AppContainer` row model，不新增 `GameState` 依赖，不修改 `MovementRules`、`CombatRules`、`SupplyRules`、`CommandValidator`、`RuleEngine` 或 `CommandExecutor`。

## 3. 实现要求

- “麾下军队”每行继续使用 `Division.thematicDisplayName` 作为军队名。
- 短摘要复用现有字段和短文案：
  - 兵力：`strength/maxStrength`
  - 粮草：`SupplyState.shortDisplayName`
  - 军令：`RetreatMode.shortDisplayCode`
  - 行动状态：溃散 / 撤退 / 可动 / 已动
- 若麾下军队超过当前展示上限，必须显示“另有 N 支麾下军队”，避免静默截断。
- 该摘要只提示当前状态，不计算道路加成、伤害、胜率、路径安全或攻击合法性。

## 4. 禁止项

- 不在 SwiftUI View 内调用 `MovementRules` / `CombatRules` 做 per-unit 规则预判。
- 不改变 `Division.canAct`、粮草、撤退、攻击、移动或回合推进规则。
- 不修改 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine` 统一执行链路。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 武将面板“麾下军队”列表能显示每支已展示军队的兵力、粮草、军令和行动状态短摘要。
- 超过展示上限时能显示剩余数量。
- 改动不影响真实移动、攻击、道路、粮草、撤退、补给、命令执行或 AI 战术塑形规则。
- 文档明确这是只读备战摘要，不是 per-unit 道路/交战预判或规则变更。
