# v2.3 战术审计显示三国化小步记录

完成日期：2026-07-05

## 目标

沿 `codex-v2.0-三国aiagent迁移总提示词.md` 的 v2.3 验收项，完成“战术名称在 UI 和 `WarDirectiveRecord` 中三国化”的最小可验证小步：

- 保留 `TacticName` 的 Codable rawValue，避免破坏既有 JSON schema。
- 补齐 `TacticName.displayName` 与三国战术口径。
- AI 面板和元帅 rationale 不再把 `blitzkrieg`、`fireCoverage` 等 rawValue 当作人读文本。
- 不改 `ZoneDirective -> WarCommandExecutor -> RuleEngine` 执行逻辑。

## 实现范围

- `WWIIHexV0/Commands/WarDirective.swift`
  - `fireCoverage` 显示为“箭雨/器械压制”。
  - `guerrillaWarfare` 显示为“奇袭/袭扰”。
  - `holdPosition` 显示为“固守”。
  - `elasticDefense` 显示为“诱敌/退守”。
  - tactic/category mismatch 诊断使用中文 `displayName`。
- `WWIIHexV0/Agents/ZoneCommanderAgent.swift`
  - 模拟元帅 rationale 使用 `tactic.displayName`。
  - `MarshalFrontSummary.statusDisplayName` 将内部状态字符串映射为粮道告急、承压、占优、寡势、对峙。
  - 兵力比显示改为 Swift `FormatStyle`，不再使用 `String(format:)`。

## 非目标

- 不修改 tactic rawValue。
- 不新增围城、士气、兵种克制或多势力 turn order。
- 不更改 AI 分类阈值、战术执行 profile 或规则伤害算法。

## 验收要点

- `AgentPanelView` 读取 `WarDirectiveRecord.tactic.displayName` 时显示三国战术名。
- 模拟元帅 rationale 不再输出旧战术 rawValue。
- Swift parse 和文本检查通过后 push 到 `origin/main`，等待 `ci-results` 云端重验证。
