# v2.4 粮道、撤退与修路日志展示名提示词

## 目标

继续推进三国迁移中“武将、道路、交战”的可见审计闭环，把 `SupplyRules`、`EconomyRules` 和 `GeneralInfluence` 中仍残留的旧军队名、裸 hex 坐标和 ASCII `base->effective` 文案收口为中文三国语义。

## 范围

- `WWIIHexV0/Rules/SupplyRules.swift`
- `WWIIHexV0/Rules/EconomyRules.swift`
- `WWIIHexV0/Rules/GeneralInfluence.swift`
- 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md`、`update_log.md`

## 非目标

- 不改补给路径、粮道通行、撤退目的地、围城判定、损耗数值、修路选点、生产部署筛选、武将道路机动数值或交战规则。
- 不改 `Division`、`HexCoord`、`RegionId`、`EconomyResources`、`ProductionKind`、`Command` 或 Codable/rawValue。
- 不新增运行时 UI、调试开关或 Agent 面板 JSON 过滤；`AgentPanelView` 与 `TheaterSystem` 英文方面名另拆后续切片。
- 不跑本机 Xcode build/test、模拟器、Probe、Smoke、Stage Regression、Dynamic Theater Regression、Full 或性能测试。

## 实现步骤

1. `SupplyRules` 事件日志中的军队名改用 `Division.thematicDisplayName`。
2. `SupplyRules.resolveRetreat` 撤退成功日志用地名优先的 hex 展示摘要：城池/关隘/郡县 + 官道/地形 + 坐标，坐标只作次级定位。
3. `EconomyRules.improveRoad` 修路结果用同一地名优先摘要，不再只列裸 `q,r`。
4. `EconomyRules.deploymentSummary` 的生产部署位置用郡县展示名和地名优先 hex 摘要，空名或等于 raw id 时显示中文占位。
5. `GeneralMovementInfluenceSummary.logFragment` 将 `上限 base->effective` 改为中文“上限由 X 提至 Y”。
6. 更新阶段文档与日志，明确本轮只改可见文本，不改规则。
