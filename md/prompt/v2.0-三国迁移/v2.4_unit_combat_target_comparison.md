# v2.4 军队接战目标对比提示词

> 本阶段承接 `v2.4_unit_combat_preview.md` 和 `v2.4_unit_road_mobility_preview.md`。执行前必须先读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和相关源码。

## 目标

- 继续强化“武将、道路、交战”体验，让玩家选中军队时不只看到射程内首个敌军，而是看到可攻击目标的简短对比。
- 复用现有 `CombatRules.attackDamage`、`counterAttackDamage`、`combatAuditSummary` 和 `GeneralInfluence`，显示最多三名射程内敌军的预计伤害、反击风险和距离。
- 首选目标保留武将影响与交战审计，帮助玩家判断哪个目标更值得打。

## 范围

- 主要文件：
  - `WWIIHexV0/App/AppContainer.swift`
- 文档文件：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改 `CombatRules`、`GeneralInfluence`、`MovementRules`、`WarCommandExecutor`、`RuleEngine`、JSON schema、UI 入参或 Xcode project。

## 实现要求

- `AppContainer.selectedUnitCombatPreviewNotes` 继续保持只读派生属性，只读取当前 `GameState`。
- 收集当前选中军队射程内、未溃散且敌对的目标，统一使用 `Faction.isHostile(to:)` 判断敌对。
- 对每个目标计算预计伤害、是否反击、反击伤害、武将攻防影响和交战审计摘要。
- 排序建议：预计伤害高者优先；伤害相同时反击伤害低者优先；再按距离近、名称稳定排序。
- 展示最多三名目标；首选目标之后再追加武将影响与交战审计，避免军队详情面板过长。

- 不允许把预览结果写入 `eventLog`、`WarDirectiveRecord` 或任何持久状态。
- 不允许改变伤害公式、反击规则、武将加成规则、道路规则、命令验证或执行链路。
- 不允许新增本机 Xcode build/test、模拟器、Probe、Smoke、Stage Regression、Dynamic Theater Regression、Full 或性能测试。
