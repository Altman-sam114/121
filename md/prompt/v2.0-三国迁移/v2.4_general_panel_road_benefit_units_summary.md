# v2.4 武将面板官道受益军队摘要提示词

## 1. 目标

继续强化“武将、道路、交战”的可读链路：在 `GeneralCommandPanelView` 的“道路与交战”摘要中补充具体哪些麾下军队获得了官道机动加成，让玩家不只看到数量和最高加成，也能知道哪支军队真正受益。

## 2. 范围

- 仅修改 `AppContainer.selectedGeneralInfluenceNotes` 的只读摘要生成。
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改 `GeneralInfluence`、`MovementRules`、`CombatRules`、`SupplyRules`、`CommandValidator`、`RuleEngine`、`CommandExecutor` 或 `GeneralCommandPanelView` 参数结构。

## 3. 实现要求

- 继续复用 `GeneralInfluence.movementSummary(for:in:)`，不重新实现道路机动规则。
- 当存在 `roadBonus > 0` 的军队时，在既有道路汇总后追加“道路受益”摘要。
- 摘要最多列出前三支受益军队，按加成从高到低、再按 id 排序；超过三支时显示剩余数量。
- 无道路加成时不追加“道路受益”行，保持原有“未获得官道机动加成”文案。
- 摘要只读展示当前状态，不代表完整路径安全、真实可达路径、敌控区绕行、同盟借道或未来回合机动。

## 4. 禁止项

- 不修改道路机动计算、移动上限、敌控区、粮道、补给、交战或命令执行规则。
- 不在 UI 层直接改 `GameState`。
- 不新增 per-unit 路径搜索或完整可达性计算。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 武将面板“道路与交战”摘要能在有官道机动加成时显示具体受益军队和加成值。
- 无官道加成时不新增空噪声行。
- 改动不影响真实移动、道路、敌控区、补给、交战、命令执行或 AI 战术塑形规则。
- 文档明确这是只读官道受益摘要，不是路径安全或可达性制度变更。
