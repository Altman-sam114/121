# v2.1 三国势力数据与外交 profile 小步记录

完成日期：2026-07-04

## 1. 本轮目标

在上一轮 `v2.1_neutral_faction_foundation.md` 的中立兼容基础上，继续推进 v2.1“势力、多方外交和三国数据基础”：

- 让 `Faction` 可以 Codable 表达首批三国势力。
- 让 MapEditor 能选择和导出三国势力 rawValue。
- 让 `DiplomacyState.initial` 能为这些势力生成基础 country / bloc profile。
- 不重写当前旧二元回合循环，避免一轮内扩大到 turn order、玩家选择和 AI 多势力调度。

## 2. 实现摘要

代码变化：

- `Faction` 新增：`cao`、`yuan`、`liuBei`、`sun`、`liuBiao`、`maTeng`、`han`。
- 保留 `germany` / `allies` 兼容 rawValue，并继续显示为曹操势力 / 袁绍势力。
- 新增 `Faction.activeTurnCases`，当前仍是 `.germany`、`.allies`。
- 兼容 `Faction.allCases` 暂时保持为 active turn set，避免旧经济、战区统计、回合循环误把新势力当作默认行动方。
- 新增 `Faction.scenarioCases`，包含兼容双方、三国势力和 `neutral`，供 MapEditor 与场景数据表达使用。
- `MapEditorView` 的势力选择器改用 `Faction.scenarioCases`。
- `MapEditorExporter` 导出的 scenario `factions` 改用 `Faction.scenarioCases.map(\.rawValue)`。
- `TheaterSystem`、`FrontLineManager`、`WarDeploymentManager` 和 `RegionRuleSystem` 的控制比例、主控方和可见 region 候选改用 `Faction.scenarioCases`，避免三国 controller 被旧二元 `allCases` 漏算。
- `DiplomacyState.initial` 改为按传入 faction 列表生成 profile：
  - 旧 `germany/allies` 保留原 Axis / Allied Coalition 兼容数据。
  - 新增曹操、袁绍、刘备、孙氏、刘表、马腾、汉室、中立郡县的基础 country / bloc。
  - 默认关系中，曹袁为 `atWar`，旧 germany/allies 为 `atWar`，汉室/中立和其他新增势力默认 `neutral`。
- `RulerAgent`、`MarshalAgentConfig`、`AppContainer`、`TurnManager`、`VictoryState`、`TerrainStyle` 补齐新势力分支。

## 3. 兼容边界

- 本轮让多势力“可以被数据表达”，不是完整多势力战役。
- 新增三国势力不会自动进入当前二元 turn cycle。
- `Faction.isHostile(to:)` 仍是无状态 fallback；规则层后续应逐步接 `DiplomacyState` / `PowerRelation` 的真实关系判断。
- 默认 JSON 仍是阿登兼容数据；v2.2 仍需新增官渡剧本并切换默认入口。

## 4. 后续建议

- 设计 `TurnOrder` / `ScenarioFactionSetup`，明确玩家可选势力、AI 控制势力和中立势力。
- 将规则层敌对判断从 `Faction.isHostile(to:)` 继续迁移到 `DiplomacyState`。
- 新增 `guandu_200_scenario.json` 和 `guandu_200_regions.json`，使用 `cao/yuan/liuBei/sun/liuBiao/maTeng/han/neutral`。
- 同步 MapEditor 术语，从德/盟兼容显示改成三国势力列表和郡县/方面/军队术语。
