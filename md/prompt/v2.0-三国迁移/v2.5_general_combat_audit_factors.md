# v2.5 武将交战审计因子提示词

## 目标

继续推进“武将、道路、交战”可解释性，把已经参与交战计算的武将攻防修正写入 `CombatRules.combatAuditSummary` 的审计因素，让玩家在军队详情、接战预判、攻击/反击日志中能直接看到“攻方武将 +N / 守方武将 +N”。

## 范围

- 允许修改 `WWIIHexV0/Rules/CombatRules.swift`。
- 只在 `attackProfile` / `defenseProfile` 的 `factors` 中追加非零武将攻防修正说明。
- 允许更新 `md/prompt/README.md` 和 `update_log.md`。

## 非目标

- 不改变 `GeneralInfluence.attackBonus` / `defenseBonus` 的数值规则。
- 不改变伤害、反击、撤退、围城、地形、河流、死守、兵种克制或命令执行链。
- 不重构 `AppContainer`、`UnitInspectorView`、`GeneralCommandPanelView` 或日志管线。
- 不删除现有“武将修正”预览文本；本轮允许解释性重复。
- 不跑本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构依据

- 武将规则影响通过 `GeneralInfluence -> CombatRules` 生效。
- `attackProfile` 已把 `generalInfluence.attackBonus(...)` 计入有效攻击。
- `defenseProfile` 已把 `generalInfluence.defenseBonus(...)` 计入有效防御。
- `CombatAuditSummary.logFragment` 会把 `attackFactors + defenseFactors` 展示给日志和 UI 预览；补充 factor 不改变实际规则结果。

## 实现步骤

1. 在 `CombatRules` 内增加私有 `signedBonus(_:)` helper，用于生成 `+1` / `-1` 文案。
2. `attackProfile` 中把武将攻击修正存为 `generalAttackBonus`，非 0 时追加 `攻方武将 +N`。
3. `defenseProfile` 中把武将防御修正存为 `generalDefenseBonus`，非 0 时追加 `守方武将 +N`。
4. 保持原有效攻防计算结果不变。
5. 更新阶段 prompt 索引和 `update_log.md`。

## 验收标准

- 有非零武将攻击修正时，交战审计因素包含“攻方武将 +N / -N”。
- 有非零武将防御修正时，交战审计因素包含“守方武将 +N / -N”。
- 有效攻击/防御计算公式与本轮前一致。
- 没有改动命令管线、UI 布局、JSON schema、Xcode project 或测试策略。
