# v2.5 静态 fallback 地图展示名三国化提示词

## 目标

继续推进 v2.5 玩家可见文案收口，把 `MapState.ardennesV0()` 这个代码内置 fallback 地图中仍暴露的 Bastogne / St. Vith / Houffalize / Bastogne Fortress 展示名改为三国语义，同时保留旧 fallback 胜利判定兼容。

## 范围

- 允许修改 `WWIIHexV0/Core/MapState.swift` 中 `ardennesV0()` 的 `cityName`、`fortressName` 和 `Objective.name`。
- 允许在 `MapState` 增加 objective id 查询 helper，或在规则层直接通过 `objective(id:)` 读取坐标。
- 允许修改 `WWIIHexV0/Rules/VictoryRules.swift`，让旧 fallback 胜利条件按稳定 objective id 判断。
- 允许更新 `md/prompt/README.md` 和 `update_log.md`。

## 非目标

- 不改 `Objective.id`、`SupplySource.id`、`Faction.germany/allies` rawValue、旧胜利 reason rawValue 或 Codable schema。
- 不改 `ardennes_v0_scenario.json`、`ardennes_v02_regions.json`、测试 fixture 或默认官渡数据。
- 不改 `RegionVictoryRules` 的旧 region fallback，除非本轮同时改 region 数据名并能保持兼容。
- 不做 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full 本机重测试。

## 当前架构依据

- 默认新局优先加载官渡 JSON；`MapState.ardennesV0()` 只作为缺数据时的静态 fallback。
- Hex 仍是战术权威；objective 只通过 coord 读取 hex controller。
- 旧 fallback 胜利规则此前通过 `controllerOfObjective(named:)` 读取英文名，若直接改 objective 展示名会破坏兼容，因此必须改为 id 查询。

## 实现步骤

1. 在 `MapState` 增加 `controllerOfObjective(id:)`，用 objective id 找 coord，再读 tile controller。
2. 将静态 fallback 地图展示名改为：
   - `bastogne`：白马津
   - `st_vith`：延津
   - `bastogne_fortress`：乌巢粮屯
   - `houffalize` 地格标签：官渡
3. 保持 objective id 不变：`bastogne`、`st_vith`、`bastogne_fortress`。
4. 将 `VictoryRules` 的旧 fallback Bastogne / St. Vith 判定改为 `controllerOfObjective(id:)`。
5. 更新 prompt 索引和 `update_log.md`。
