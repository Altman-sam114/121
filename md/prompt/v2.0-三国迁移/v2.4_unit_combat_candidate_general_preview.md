# v2.4 军队接战候选敌将预判兼容层

## 目标

在军队详情的“接战预判”候选目标行中显示敌方武将身份，让玩家比较首选和候选目标时，不只看到兵力伤害和风险，也能看到每个候选目标背后的敌将。

## 范围

- 修改 `WWIIHexV0/App/AppContainer.swift` 的 `combatTargetPreviewLine` 派生文本。
- 复用每个候选 `CombatTargetPreview.influence` 中的 `defenderGeneralName/id`。
- 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 禁止项

- 不改 `GeneralInfluence` 的武将查找、道路加成、攻击加成或防御加成。
- 不改 `CombatRules` 伤害、反击、撤退、死守、围城或兵种克制公式。
- 不改 `CommandExecutor`、`CommandValidator`、`WarCommandExecutor`、`RuleEngine` 或真实命令执行结果。
- 不新增武将任免、头像、士气、单挑、详细技能展开或 JSON schema。

## 当前架构依据

- `selectedUnitCombatPreviewNotes` 已为每个候选目标计算 `GeneralCombatInfluenceSummary`。
- 首选目标已有“交战武将”总览；候选目标行仍是只读 UI 文案，不能执行攻击。
- 候选目标排序仍按预计伤害、反击风险、距离和名称排序，本轮只补充敌将身份。

## 实现步骤

1. 让 `combatTargetPreviewLine` 接收当前候选的 `GeneralCombatInfluenceSummary`。
2. 若候选防守方存在 `defenderGeneralName/id`，在候选行追加 `敌将 ...`。
3. 若候选防守方没有武将分配，不追加额外文本，避免候选行过长。
4. 保留既有首选理由、接战官道、交战武将总览、强度结算、风险、态势、武将影响和交战审计。

## 验收标准

- 选中军队且射程内有多名敌军时，最多三条候选目标行能在敌方有武将分配时显示 `敌将 ...`。
- 首选目标仍显示已有“交战武将”总览和非零“武将影响”摘要。
- 真实移动、攻击、日志结算、命令校验和动态战略同步行为不变。

## 风险提示

- 本轮没有运行 UI 烟测；候选行新增敌将文本在窄屏、长武将名和 Dynamic Type 下的换行仍需云端 CI、后续 Agent C artifact 复判或人工授权运行检查。
- 摘要只展示候选目标防守方武将身份，不实现技能详情、头像、士气、单挑或忠诚变化解释。
