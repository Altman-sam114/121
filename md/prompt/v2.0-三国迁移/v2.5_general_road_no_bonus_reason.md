# v2.5 武将道路无加成原因同源化提示词

## 目标

继续推进“武将、道路、交战”体验，把道路机动未触发原因从 UI 侧复制判断收回到 `GeneralInfluence` 的只读摘要中。单位详情和武将面板都应消费同一份规则解释，避免忠诚/满意阈值、道路技能表和郡县官道判断在 `AppContainer` 中漂移。

## 范围

- 修改 `WWIIHexV0/Rules/GeneralInfluence.swift`。
- 修改 `WWIIHexV0/App/AppContainer.swift`。
- 更新 `md/flow/flow.md`、`md/prompt/README.md` 和 `update_log.md`。

## 非目标

- 不改变道路机动加成数值、触发阈值、技能列表或移动范围计算。
- 不改变 `MovementRules`、`WarCommandExecutor`、`CommandValidator`、`RuleEngine`、道路修缮、粮道、交战或占领规则。
- 不改 `UnitInspectorView`、`GeneralCommandPanelView`、SpriteKit、JSON schema、Xcode project 或测试文件。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Full。

## 架构依据

- 武将道路机动规则权威是 `GeneralInfluence -> MovementRules`。
- `AppContainer.selectedUnitMobilityPreviewNotes` 和 `selectedGeneralInfluenceNotes` 是只读展示派生，不应维护独立道路技能表或忠诚/满意阈值。
- SwiftUI View 只消费 notes 数组，不直接重算道路规则。

## 实现要求

1. 新增 `GeneralRoadMobilityNoBonusReason`，表达未分配武将、忠诚/满意不足、缺少道路技能、所在郡县无可借官道等只读原因。
2. 扩展 `GeneralMovementInfluenceSummary`，加入 `noBonusReason` 和中文 `noBonusFragment`。
3. `GeneralInfluence.movementSummary` 使用真实道路机动判断生成原因；`roadMobilityBonus` 复用同一判断，避免规则和解释分叉。
4. `AppContainer.selectedUnitMobilityPreviewNotes` 消费 `summary.noBonusFragment`。
5. `AppContainer.selectedGeneralInfluenceNotes` 的无加成原因从 movement summaries 聚合，不再本地复制道路技能表、忠诚阈值或郡县道路判断。
6. 删除 `AppContainer` 中重复的 `generalRoadNetworkAvailable` / `hasGeneralRoadNetworkSkill` helper。

## 验收标准

- 单位详情在无道路加成时仍显示明确中文原因。
- 武将面板道路受益无加成原因来自 `GeneralMovementInfluenceSummary.noBonusReason` 聚合。
- `AppContainer` 不再维护道路技能 raw id 列表或忠诚/满意阈值。
- 道路机动数值、路径、宏观军令、命令执行和 UI View 结构保持不变。
