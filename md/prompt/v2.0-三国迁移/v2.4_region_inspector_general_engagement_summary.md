# v2.4 郡县检查器武将与接战摘要提示词

## 1. 目标

继续强化“武将、道路、交战”的郡县态势闭环：在 `RegionInspectorView` 中补充本郡己方军队对应武将，以及当前可见敌军相对选中地格或郡县代表格的距离、射程、兵力和敌将摘要。

## 2. 范围

- 仅修改只读 UI 派生与展示：
  - `MapDisplayAdapter.inspectorState`
  - `RegionInspectorState`
  - `RegionInspectorView`
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改真实移动、攻击、敌控区、道路修缮、粮道、补给、外交或命令执行规则。

## 3. 实现要求

- `RegionInspectorState` 新增本郡武将摘要和可见敌军接战摘要。
- 本郡武将摘要从己方可见军队和现有 `GeneralAssignment` 快照派生，优先显示武将中文名，缺失时使用 `generalId`。
- 敌军接战摘要只来自当前郡县内可见、未毁灭且 `Faction.isHostile(to:)` 判定 hostile 的军队。
- 距离锚点优先使用当前选中 hex；没有选中 hex 时使用郡县 `representativeHex`。
- 摘要只显示距离、是否入敌军当前射程、兵力和敌将名，不计算伤害、胜率或真实攻击合法性。
- 不直接调用或改写 `CombatRules`、`MovementRules`、`CommandValidator` 或 `RuleEngine`。

## 4. 禁止项

- 不让不可见敌军进入郡县接战摘要。
- 不把接战摘要写成完整胜率、完整路径安全、真实攻击许可或敌方回合反制判断。
- 不修改 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine` 统一执行链路。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 郡县检查器能显示本郡己方军队对应武将。
- 郡县检查器能显示最多三名可见 hostile 敌军的距离、射程态势、兵力和敌将名。
- 可见性与敌对判断不泄漏隐藏敌军，不把中立/友好势力误写为敌军。
- 真实移动、交战、道路、粮道、补给和命令执行规则不变。
- 文档明确这是只读态势摘要，不是完整胜率、路径安全或攻击合法性制度。
