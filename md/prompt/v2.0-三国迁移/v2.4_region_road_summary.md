# v2.4 郡县官道摘要提示词

> 本阶段承接 `v2.4_unit_road_mobility_preview.md` 和 `v2.4_unit_combat_target_comparison.md`。执行前必须先读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和相关源码。

## 目标

- 继续强化“武将、道路、交战”体验，把道路信息从单支军队详情扩展到选中郡县面板。
- 让玩家查看郡县时能看到当前官道覆盖数量和当前地格是否在官道上，便于判断修路、行军和粮道价值。
- 保持道路摘要为只读 UI 派生，不改变修路、移动、补给或经济规则。

## 范围

- 主要文件：
  - `WWIIHexV0/SpriteKit/MapDisplayAdapter.swift`
  - `WWIIHexV0/UI/RegionInspectorView.swift`
- 文档文件：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改 `MovementRules`、`SupplyRules`、`EconomyRules`、`Command.improveRoad`、`CommandValidator`、`RuleEngine`、JSON schema 或 Xcode project。

## 实现要求

- 在 `RegionInspectorState` 增加只读字段，表达当前郡县官道格数、可通行格数和当前选中 hex 是否有官道。
- 在 `MapDisplayAdapter.inspectorState` 中从 `RegionNode.displayHexes` 与 `MapState.tile(at:)` 派生这些字段。
- 在 `RegionInspectorView` 使用现有 `LabeledContent` 展示“当前官道”和“官道覆盖”，保持紧凑、可换行。
- 该摘要只读取 `GameState.map` 和 region 数据，不写入 `GameState`，不触发命令。

## 禁止项

- 不允许 UI 直接修路、移动军队、改补给或修改 `GameState`。
- 不允许改变道路移动成本、武将道路加成条件、补给通行、经济收入或太守修路执行逻辑。
- 不允许把官道摘要写入 `eventLog`、`WarDirectiveRecord` 或任何持久状态。
- 不允许新增本机 Xcode build/test、模拟器、Probe、Smoke、Stage Regression、Dynamic Theater Regression、Full 或性能测试。
