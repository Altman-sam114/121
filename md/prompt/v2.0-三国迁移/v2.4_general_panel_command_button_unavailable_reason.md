# v2.4 武将面板命令按钮不可用原因提示

## 1. 目标

继续强化“武将、道路、交战”的玩家操作可读性：当 `GeneralCommandPanelView` 的“固守战线”或“进攻郡县”按钮因阶段、观察模式、防区或目标条件不足而灰掉时，显示只读原因和下一步提示，让玩家知道该选己方防区、己方军队、敌方前线郡县，还是等待玩家军令阶段。

## 2. 范围

- 仅修改只读 UI 派生与展示：
  - `AppContainer.canOrderSelectedGeneralHoldLine`
  - `AppContainer.canOrderSelectedGeneralAttackRegion`
  - `AppContainer.selectedGeneralHoldLineUnavailableReason`
  - `AppContainer.selectedGeneralAttackRegionUnavailableReason`
  - `GeneralCommandPanelView`
  - `RootGameView` 两处面板传参
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改真实移动、攻击、道路、粮道、补给、外交或命令执行规则。

## 3. 实现要求

- 在 `AppContainer` 中集中生成两个只读原因：
  - `selectedGeneralHoldLineUnavailableReason`
  - `selectedGeneralAttackRegionUnavailableReason`
- `canOrderSelectedGeneralHoldLine` 和 `canOrderSelectedGeneralAttackRegion` 应由对应原因是否为 `nil` 决定，避免按钮条件与提示条件分叉。
- 原因覆盖：
  - 观察模式不能下令。
  - 当前不是玩家行动势力或玩家军令阶段。
  - 固守缺少己方防区、己方军队或相邻敌方郡县来推断防区。
  - 进攻缺少敌方前线郡县。
  - 选中郡县未映射防区。
  - 目标是己方或非敌对防区。
  - 目标附近没有可用己方相邻防区。
- `GeneralCommandPanelView` 只消费原因字符串，用 `info.circle` 或同类只读提示显示，不在 View 内重新推导规则。

## 4. 禁止项

- 不修改 `CommandValidator`、`RuleEngine`、`CommandExecutor`、`WarCommandExecutor`。
- 不修改 `GeneralAgent` 战术塑形。
- 不修改 `WarDeploymentState` / `WarDeploymentManager` 的动态防区归属。
- 不改变 `ZoneDirective`、`Command`、`PlayerCommandState` schema。
- 不让灰态提示替代真实执行前校验；点击执行后的 guard 和规则拒绝仍保留。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 按钮灰掉时能显示对应固守/进攻不可用原因或下一步提示。
- 按钮可用性与原因一致：原因为 `nil` 时可点，有原因时灰掉。
- `GeneralCommandPanelView` 不直接推导防区、阶段、敌我或目标规则。
- 真实 `ZoneDirective -> WarCommandExecutor -> CommandValidator -> RuleEngine` 执行链路不变。
- 文档明确这是只读操作提示，不是新的战术、外交或路径合法性制度。
