# v2.0 三国迁移审计与兼容合同

完成日期：2026-07-04

## 1. 本轮目标

本轮按 `codex-v2.0-三国aiagent迁移总提示词.md` 执行 v2.0 起步任务：先建立三国迁移合同、做硬编码审计、收束玩家可见主 UI 术语，不做大规模类型重命名和规则迁移。

本轮实际落地：

- README 和核心流程文档改为“三国棋策 Agent”迁移口径。
- 在不改变 Codable/rawValue 的前提下，把主 UI 显示词迁移为三国语义：势力、军队、武将、郡县、方面、防区、钱粮、军械、粮草。
- `Faction.germany/allies` 仍保留源码兼容名，但 `displayName` 当前显示为曹操势力 / 袁绍势力。
- `Division` 仍保留源码兼容名，但主面板通过 `thematicDisplayName` 显示为军队/步卒营/骑兵军/器械营。
- `ProductionKind`、`SupplyState`、`BaseTerrain`、`TacticName`、`DirectiveType`、`Command` 等显示名做三国化。
- SpriteKit 可见标签中 `FORT`、`SUP A/G`、空棋盘标题改为三国语义。

## 2. 非目标

本轮明确不做：

- 不扩展 `Faction` 到曹、袁、刘、孙、刘表、马腾、汉室/中立等多势力。
- 不移除或重写 `Faction.opponent`。
- 不迁移默认 JSON 到官渡剧本。
- 不重命名 `Division`、`Theater`、`FrontZone` 等源码类型。
- 不改 `WarCommandExecutor` / `RuleEngine` 的执行语义。
- 不新增真实 LLM、真实三国头像或商业素材。
- 不跑 Xcode build/test、模拟器、Probe、Smoke、Full Regression。

## 3. 兼容合同

v2.0 后续开发必须遵守：

- 源码兼容名和玩家显示名分离。显示层可三国化，rawValue 和 Codable 字段暂不破坏旧数据。
- `HexTile.controller` 和 `Division.coord` 仍是战术权威。
- `RegionNode` 仍是从 hex 聚合的战略层，不能替代 hex 移动/攻击/占领。
- `regionToTheater` 仍只作初始/基础方面映射；运行时推进看 `hexToTheater`。
- `hexToFrontZone` 仍是部署层动态归属权威。
- 玩家、AI、未来聊天命令仍必须经 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine`。
- UI、SpriteKit、Agent 不得直接改 `GameState` 的占领、单位坐标、动态方面或防区归属。

## 4. 审计摘要

执行过的只读审计命令：

```sh
git branch --show-current
git status --short
rg -n "germany|allies|Ardennes|Panzer|Division|Bastogne|MockAI|Marshal|Ruler|Faction\.opponent" WWIIHexV0 MapEditor README.md md
rg -n "enum Faction|struct Division|ComponentType|EconomyResources|DiplomacyState|GeneralData|ZoneDirective|WarCommandExecutor|RuleEngine" WWIIHexV0
```

主要发现：

- `Faction` 仍只有 `germany/allies`，且 `opponent` 是二元敌我假设。
- `DataLoader`、`general_agents.json`、阿登 JSON、测试和历史文档中仍大量使用 germany/allies/Ardennes/Bastogne/guderian。
- `Division` / `ComponentType` 仍绑定二战兵种：tank、motorizedInfantry、infantry、artillery。
- `EconomyResources.manpower/industry/supplies` 当前只做显示映射，数据结构还未三国化。
- `MapEditor` 默认 faction 和导出仍是 germany/allies；v2.2 前不能声称编辑器已三国化。
- README、`md/flow/*` 已更新迁移状态，但旧版本历史段落仍保留二战记录，作为历史上下文不视为主 UI 残留。

## 6. 后续拆分

v2.1 建议优先处理：

- 扩展或桥接多势力：曹、袁、刘、孙、刘表、马腾、汉室/中立。
- 弃用 `Faction.opponent`，敌我关系改由 `DiplomacyState` / `PowerRelation` 判断。
- 修复中立 region/hex fallback 到 allies 的旧问题。

v2.2 建议优先处理：

- 新建 `guandu_200_scenario.json` 和 `guandu_200_regions.json`。
- DataLoader 支持默认加载三国剧本。
- MapEditor 术语和导出字段迁移为郡县、方面、军队、武将。

v2.3+ 建议优先处理：

- 三国兵种、粮草、围城、士气和战术规则。
- 君主、军师、太守、武将 Agent schema 与结构化 directive。
