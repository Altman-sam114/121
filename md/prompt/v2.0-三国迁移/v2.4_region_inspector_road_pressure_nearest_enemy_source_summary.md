# v2.4 郡县检查器官道受压最近敌军来源摘要提示词

## 1. 目标

继续围绕“武将、道路、交战”补强郡县检查器：在已有官道覆盖和受压数量之外，显示造成本郡官道受压的最近可见敌军来源，让玩家能判断哪支敌军、哪名敌将正在压迫道路。

## 2. 范围

- 仅修改只读 UI 派生与展示：
  - `MapDisplayAdapter.inspectorState`
  - `RegionInspectorState`
  - `RegionInspectorView`
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改真实移动、攻击、敌控区、道路修缮、粮道、补给、外交或命令执行规则。

## 3. 实现要求

- `RegionInspectorState` 新增只读 `roadPressureSourceSummaries`。
- 摘要来源只能使用当前 viewer 可见、未毁灭、且 `Faction.isHostile(to:)` 判定 hostile 的敌军。
- 本郡 `hasRoad == true` 的 hex 被可见 hostile 军队占据或邻近时，算作道路受压来源候选。
- 同一支敌军只输出一条最近受压官道摘要，最多显示三条。
- 摘要应包含敌军名、受压官道坐标、距官道、距选中地格或郡县代表格的锚点距离；若能从当前防区分配读到敌将，追加敌将名。
- 文案必须保持只读态势提示，不承诺路径安全、攻击合法性、胜率或敌方回合反制。

## 4. 禁止项

- 不让不可见敌军进入官道压迫来源摘要，避免泄漏隐藏军队。
- 不改 `MovementRules.isEnemyZoneOfControl`、`CombatRules`、`SupplyRules`、`CommandValidator`、`RuleEngine` 或 `CommandExecutor`。
- 不把中立或友好势力按旧二元 `!= faction` 写成敌军；必须使用 `Faction.isHostile(to:)`。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 郡县检查器新增“官道压迫”行，能显示最多三条最近可见敌军来源。
- 官道受压数量和来源摘要使用同一 hostile / visible / not destroyed 口径。
- 摘要能关联道路坐标、距离和敌将，不泄漏不可见敌军。
- 真实移动、交战、道路、粮道、补给、外交和命令执行规则不变。
- 文档明确这是只读态势摘要，不是完整 ZOC、路径安全或攻击许可制度。
