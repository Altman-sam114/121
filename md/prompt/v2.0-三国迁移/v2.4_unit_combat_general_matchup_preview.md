# v2.4 军队接战武将对阵预判兼容层

## 目标

在军队详情的“接战预判”中，为首选目标增加一条只读“交战武将”摘要，让玩家即使在武将攻防加成为 0 时，也能看到这次首选接战涉及的我方武将和敌方武将。

## 范围

- 修改 `WWIIHexV0/App/AppContainer.swift` 的 `selectedUnitCombatPreviewNotes` 派生文本。
- 复用现有 `GeneralCombatInfluenceSummary` 中的 `attackerGeneralName/id` 与 `defenderGeneralName/id`。
- 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 禁止项

- 不改 `GeneralInfluence` 的武将查找、攻击加成、防御加成或道路加成。
- 不改 `CombatRules` 伤害、反击、撤退、死守、围城或兵种克制公式。
- 不改 `CommandExecutor`、`CommandValidator`、`RuleEngine` 或真实命令执行结果。
- 不新增武将状态、任免 UI、头像资产、JSON schema、士气、单挑或完整技能解释。

## 当前架构依据

- `GeneralCombatInfluenceSummary` 已由 `CombatRules.generalInfluenceSummary` 生成，和真实攻防修正使用同一套武将分配查找。
- `GeneralCombatInfluenceSummary.logFragment` 只在攻防修正非 0 时输出；本轮新增的“交战武将”只补充参战双方身份，不替代“武将影响”。
- `UnitInspectorView` 只渲染 `AppContainer` 传入的文本，不执行攻击。

## 实现步骤

1. 在 `selectedUnitCombatPreviewNotes` 中读取首选目标的 `leadingPreview.influence`。
2. 若我方或敌方至少一侧有武将 name/id，追加 `交战武将：我方 ...，敌方 ...`。
3. 缺少一侧武将时显示“未任命”，避免玩家误以为 UI 漏读。
4. 保留既有首选理由、接战官道、三目标候选行、武将影响和交战审计。
5. 文档记录本轮只是 UI 预判解释层，不改变真实移动和交战。

## 验收标准

- 选中军队且射程内有敌军时，若首选接战任一侧有武将分配，“接战预判”显示交战武将摘要。
- 武将影响非 0 时仍保留已有“武将影响”加成摘要。
- 真实移动、攻击、日志结算、命令校验和动态战略同步行为不变。

## 风险提示

- 本轮没有运行 UI 烟测；新增“交战武将”文本在窄屏、长武将名和 Dynamic Type 下的换行仍需云端 CI、后续 Agent C artifact 复判或人工授权运行检查。
- 摘要只展示当前首选接战双方的武将快照，不实现任免、头像、技能详情、士气、单挑或忠诚变化解释。
