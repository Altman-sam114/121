# v2.4 军队未接敌时官道接战距离预判兼容层

## 目标

当选中军队当前射程内没有敌军时，细化“官道接近”摘要：不仅显示本回合可达且更接近敌军的官道位，还要说明这些官道位抵达后是否已经进入该军队射程，或仍差多少格才能入射程。

## 范围

- 修改 `WWIIHexV0/App/AppContainer.swift` 的 `combatOutOfRangeRoadApproachText` 派生文本。
- 继续复用 `MovementRules.movementRange`、`MovementRules.isEnemyZoneOfControl` 和 `HexCoord.distance`。
- 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 禁止项

- 不改 `MovementRules` 的移动范围、道路机动、敌控区、移动成本或行动消耗。
- 不改 `CombatRules` 的射程、伤害、反击、撤退、死守、围城或兵种克制公式。
- 不改 `GeneralInfluence` 的道路加成或攻防修正。
- 不改 `CommandExecutor`、`CommandValidator`、`WarCommandExecutor`、`RuleEngine` 或真实命令执行结果。
- 不承诺移动后同回合可攻击；本轮只说明抵达官道位置后的距离态势。

## 当前架构依据

- `selectedUnitCombatPreviewNotes` 在射程内存在敌军时走现有多目标接战预判。
- 当前射程内无敌军但地图上存在敌军时，`combatOutOfRangePreviewNotes` 会选择最近敌军并调用 `combatOutOfRangeRoadApproachText`。
- `combatOutOfRangeRoadApproachText` 已经只读统计可达、更接近目标的官道位、安全官道位和敌控区官道位，不执行移动命令，也不修改 `GameState`。

## 实现步骤

1. 在 `combatOutOfRangeRoadApproachText` 中复用已有 `reachableRoadCoords` 和 `nearestRoadDistance`。
2. 统计 `reachableRoadCoords` 中距离目标不大于 `attacker.range` 的官道位数量。
3. 若数量大于 0，追加 `... 个抵达后入射程`。
4. 若数量为 0，按 `nearestRoadDistance - attacker.range` 追加 `仍差 ... 格入射程`。
5. 保留既有可达更近官道位、最近距离、安全数量、敌控区数量、我方/目标临官道提示。

## 验收标准

- 选中军队且当前射程内没有敌军但地图上存在敌军时，若有可达更近官道位，`官道接近` 会显示抵达官道后是否入射程。
- 若没有任何可达更近官道位进入射程，`官道接近` 会显示最近官道位仍差多少格入射程。
- 射程内有敌军时既有三目标接战预判保持不变。
- 真实移动、攻击、日志结算、命令校验和动态战略同步行为不变。

## 风险提示

- 本轮没有运行 UI 烟测；新增“抵达后入射程 / 仍差 ... 格入射程”文本在窄屏、长军队名和 Dynamic Type 下的换行仍需云端 CI、后续 Agent C artifact 复判或人工授权运行检查。
- 摘要只做当前位置到最近敌军的只读距离判断，不模拟移动后同回合攻击、跨回合追击、完整路径安全、同盟借道或真实胜率。
