# v2.4 - 地格、战报与外交标签展示安全提示词

## 目标

继续推进三国迁移玩家可见文本收敛，把裸坐标、战报 raw record id、外交面板英文 `Agent` 标签和地图计划军令 raw `generalId` fallback 改为中文三国语义展示。

## 范围

- 军队详情、玩家选择地格日志、移动命令结果、行军日志和动态方面推进事件：使用城池、关隘、郡县、官道或地形优先的地格展示，坐标只作括号内次级定位。
- 战报 metadata：有 `relatedRecordId` 时显示“军机审计”，不直接展示 raw id。
- 外交面板：君主和外交官记录字段名显示“执行者”。
- 地图计划军令标签：缺武将展示名时显示“未命名武将”，不 fallback 到 raw `generalId`。

## 非目标

- 不改变 `HexCoord`、`GameLogEntry.relatedRecordId`、`PlayerPlannedOperation.createdByGeneralId`、Codable schema 或 rawValue。
- 不改变移动、占领、动态方面推进、道路、粮道、交战、外交或武将塑形规则。
- 不做本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 实现步骤

1. 基于最新 `origin/main` 和干净 `main`。
2. 在 `MapDisplayAdapter` 为 `UnitInspectorStrategicState` 生成地格展示名，`UnitInspectorView` 只读展示。
3. 在 `AppContainer`、`RuleEngine`、`CommandExecutor` 和 `WarCommandExecutor` 的可见消息边界使用同一地名/地形优先格式。
4. `EventLogView` 隐藏 raw `relatedRecordId`，改显示中文审计关联。
5. `DiplomacyPanelView` 和 `BoardScene` 收敛英文/ raw fallback。
6. 同步 README、flow、flowchart、prompt 索引和 update_log。

## 轻量检查

- `swiftc -parse` 覆盖本轮改动 Swift 文件。
- 定向扫描裸坐标主文案、`relatedRecordId` 直出、`LabeledContent("Agent")` 和 `return generalId`。
- 文档尾随空白、冲突标记、prompt 索引差集和 `git diff --check`。

## 验收标准

- 玩家可见主文案不再以裸 `q,r` 作为地格主展示。
- 战报 metadata 不显示 raw record id。
- 外交面板不显示英文 `Agent` 字段名。
- 地图计划军令标签不显示 raw `generalId`。
- 底层记录字段、schema 和规则行为保持不变。
