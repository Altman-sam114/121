# v2.4 军队可见官道受压审计兼容层提示词

## 1. 目标

继续强化“武将、道路、交战”的玩家决策闭环：军队详情和武将面板的官道受压、可达官道安全数、接战官道压制位和未接敌官道接近摘要，只能受玩家视角可见 hostile 军队影响，避免隐藏敌军通过道路受压数量、最近安全官道距离或可达格变化泄漏。

## 2. 范围

- 仅修改只读 UI 派生：
  - `WWIIHexV0/App/AppContainer.swift`
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改 `MovementRules`、真实移动、攻击、道路、粮道、补给、外交、AI 指令执行或 `RuleEngine`。

## 3. 实现要求

- 在 `AppContainer` 中新增只读 helper，用玩家视角可见、未毁灭且 `Faction.isHostile(to:)` 的军队判断 UI 文案里的敌控区。
- `selectedUnitMobilityPreviewNotes` 的当前位置官道受压、可达官道安全/受压数量和最近安全官道距离使用该可见口径。
- `selectedUnitCombatPreviewNotes` 的接战官道压制位、未接敌接近候选、官道接近安全/受压数量使用该可见口径。
- `GeneralCommandPanelView` 麾下军队行的官道受压摘要使用同一可见口径。
- 只读 `movementRange` 预判可使用剔除玩家不可见 hostile 的临时 `GameState`，避免隐藏敌军 ZOC 改变可达官道和接近候选摘要。
- 观察模式继续由 `MapDisplayAdapter` 的 revealAll 行为兼容全图查看。

## 4. 禁止项

- 不改 `MovementRules.isEnemyZoneOfControl` 的真实规则口径。
- 不改 `CommandValidator`、`WarCommandExecutor`、`RuleEngine`、`CommandExecutor` 或真实命令执行。
- 不新增持久字段，不修改 Codable schema，不改 Xcode project。
- 不让不可见敌军进入军队详情或武将面板的道路受压、可达官道安全数、最近安全官道距离、接战官道压制位或未接敌官道接近摘要。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 军队详情“道路机动”的当前官道受压、可达官道安全/受压数量和最近安全官道距离只受玩家视角可见 hostile 军队影响。
- 军队详情“接战预判”的接战官道压制位、未接敌接近候选和官道接近摘要只用玩家视角可见 hostile 计算受压/安全数量。
- 武将面板麾下军队行的官道受压摘要只受玩家视角可见 hostile 军队影响。
- 真实移动、攻击、伤害、反击、道路、粮道、补给、外交和命令执行规则不变。
