# v2.3 兵种克制最小规则

完成日期：2026-07-05

## 目标

把 v2.3 总提示词中“兵种克制：骑兵在平原机动优势，弓弩有 range，器械对城池/关隘有效，山地限制骑兵”的首版规则补齐到现有 `Division` / `CombatRules` / `MovementRules`，不新增 schema，不改变玩家和 AI 的统一命令入口。

## 当前源码依据

- `ComponentType` 已支持 `cavalry`、`archer`、`siegeEngine`、`naval`、`guard`，并保留旧 `tank`、`artillery` rawValue 作 fallback。
- `Division.range` 已由 component stats 计算，`archer` 与 `siegeEngine` 默认 range 为 2。
- `Division.isArmor` 已把旧 `tank` 和三国 `cavalry` 统一为骑兵/机动力单位。
- `MovementRules.tacticalTerrainPenalty` 已对 `isArmor` 单位进入 forest / hill / city / fortress / mountain 追加困难地形移动成本。
- `CombatRules.effectiveAttack` 已对 `isArmor` 攻击 plain 加成，并在困难地形降低攻击。

## 本轮实现

- 新增 `Division.isSiegeCapable`，将旧 `artillery` 和三国 `siegeEngine` 统一视为攻城能力单位。
- 保留 `Division.isArtillery` 兼容名，内部转接到 `isSiegeCapable`，避免打断旧 AI / UI / 经济规则。
- `CombatRules.effectiveAttack` 对 `isSiegeCapable` 攻击 city / fortress terrain、`cityName` 或 `fortressName` hex 时增加攻坚修正。
- 攻击、移动、补给和占领仍由 `Command -> RuleEngine -> CommandExecutor -> CombatRules / MovementRules` 执行。

## 非目标

- 不新增兵种克制矩阵。
- 不改 JSON schema 或单位模板结构。
- 不实现水战、兵种阵型、武将技能或士气字段。
- 不改 AI prompt schema 或真实 LLM 接入。
- 不跑 Xcode / XCTest / 模拟器 / Probe / Smoke / Full。

## 验收标准

- 弓弩和器械的 range 继续来自 component stats。
- 骑兵/旧装甲的平原优势和困难地形限制仍保留。
- 攻城器械/旧炮兵攻击城池或关隘时有可解释加成。
- 旧 `isArtillery` 调用点不需要同步重构。

## 风险

- 本轮仅做首版数值修正，未通过本机运行时战斗测试确认具体手感。
- 旧 `artillery` fallback 也获得攻城加成，这是为了兼容旧阿登数据和当前“器械”显示语义；后续若要区分野战弓炮与攻城器械，需要新增更细 component 或模板字段。
