# v2.4 WarCommandExecutor 武将官道机动落地提示词

## 目标

继续围绕“武将、道路、交战”推进三国迁移，让 `WarCommandExecutor` 在宏观 `ZoneDirective` 选兵和行军落点时读取 `MovementRules` 的有效机动与真实寻路结果，使 `GeneralInfluence` 的官道机动加成不只停留在 UI 预览和移动日志，而能影响 AI / 玩家武将宏观军令的可达判断。

## 范围

- `WWIIHexV0/Commands/WarCommandExecutor.swift`
  - 复用 `MovementRules`。
  - 攻击单位排序中的 movement 字段使用 `MovementRules.effectiveMovementLimit(for:in:)`。
  - 防守/接近目标继续复用同一个 `MovementRules.movementRange(for:in:)`。
  - `tacticalDestination(in:for:state:)` 判断目标郡县候选 hex 是否可达时，改用 `MovementRules.shortestPath(for:to:in:)`，不再用直线距离和 `division.movement`。
- 文档同步：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`

## 非目标

- 不改 `MovementRules` 的移动成本、敌控区、道路、河流、装甲/骑兵地形惩罚或寻路算法。
- 不改 `GeneralInfluence` 的武将技能、忠诚满意、道路机动加成数值或触发条件。
- 不改 `CommandValidator`、`CommandExecutor`、`RuleEngine`、`Command`、`ZoneDirective`、`WarDirectiveRecord` 或 JSON / Codable schema。
- 不改攻击伤害、反击、撤退、补给、粮道、围城、动态战区推进或外交 hostile 规则。
- 不运行 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 当前架构依据

- `GeneralInfluence.effectiveMovementLimit(for:in:)` 已把防区武将、官道网络和技能转换为有效机动上限。
- `MovementRules.movementRange` 和 `shortestPath` 已使用 `effectiveMovementLimit`，并同时遵守道路成本、河流、困难地形惩罚、敌控区停止和单位占位规则。
- `WarCommandExecutor` 是 `ZoneDirective -> Command` 的翻译层；它只能生成 `.move` / `.attack` / `.hold` 等底层命令，真实状态修改仍由 `RuleEngine` 执行。

## 实现步骤

1. 在 `WarCommandExecutor` 内复用一个私有 `MovementRules` 实例。
2. 将 `attackSortKey` 的 movement 排序指标从 `division.movement` 改为 `movementRules.effectiveMovementLimit(for:in:)`。
3. 将 `defensiveDestination` 和 `approachDestination` 中临时创建的 `MovementRules()` 改为复用实例。
4. 将 `tacticalDestination` 对目标候选 hex 的可达判断从 `division.coord.distance(to:) <= division.movement` 改为 `movementRules.shortestPath(...) != nil`。
5. 扫描确认 `WarCommandExecutor` 中不再用 `MovementRules()` 临时实例，也不再用基础 movement 过滤目标候选 hex。
6. 更新文档和阶段记录，明确本轮只改变执行器的选兵/落点判断，不改变真实移动规则。

## 验收标准

- `WarCommandExecutor` 的攻击单位 movement 排序使用 `MovementRules.effectiveMovementLimit`。
- `tacticalDestination` 的目标 hex 可达判断使用 `MovementRules.shortestPath`，因此会遵守武将官道机动、道路、敌控区、地形和占位。
- `WarCommandExecutor` 仍只生成底层命令，真实移动、攻击、占领和动态战区推进仍由 `RuleEngine` / `CommandExecutor` / `StrategicStateSynchronizer` 执行。
- 文档、prompt 索引和 `update_log.md` 同步记录本轮范围、轻量检查和未跑重测试原因。
