# v2.5 MapEditor 兼容势力标签三国化

## 目标

继续推进 v2.5 发布级 UI / MapEditor 收口，把地图编辑器中兼容 `Faction.germany/allies` 的玩家可见标签从旧二战语义改成三国势力语义，并把编辑器势力选择标题从“阵营”收口到“三国势力”口径，减少默认编辑器界面的“德军 / 盟军 / 阵营”残留。

## 范围

- 只修改 `MapEditor/MapEditorView.swift` 的显示文本。
- `.germany` 显示为“曹军”，`.allies` 显示为“袁军”。
- “补给阵营”显示为“补给势力”，“阵营”显示为“势力”。
- 同步 `md/prompt/README.md` 与 `update_log.md`。

## 禁止项

- 不修改 `Faction` enum、rawValue、Codable、JSON、DataLoader、MapEditor 数据 schema 或导出格式。
- 不修改 `WWIIHexV0/Data/*.json`、`WWIIHexV0.xcodeproj/project.pbxproj`、规则、AI、命令管线或测试文件。
- 不处理旧阿登 fallback 数据、历史文档、`DataLoader` 英文校验错误或 `generals.json` 兼容武将数据。
- 不运行 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构依据

- 当前源码仍保留 `Faction.germany/allies` 作为兼容 rawValue，但默认三国剧本中它们在可见层对应曹操 / 袁绍语义。
- MapEditor 负责稀疏 hex、郡县、方面、军队和势力归属编辑；本轮只改编辑器可见标签，不改变数据身份。
- `Faction.displayName` 已在主游戏 UI 中显示为曹操势力 / 袁绍势力；MapEditor 应跟随同一展示方向。
