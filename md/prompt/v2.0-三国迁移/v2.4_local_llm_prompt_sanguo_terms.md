# v2.4 Local LLM 提示词三国语义兼容层

## 目标

清理 Legacy `LocalLLMDecisionProvider` 使用的 `AgentPromptBuilder` 中仍直接写入的二战题材提示词，避免本地 LLM 路径启用时继续把模型锚定到 `WWII hex strategy prototype`。提示词应强调三国棋策、武将、军队、郡县、官道、粮草、围城压力和可见交战机会。

## 范围

- 只修改 `WWIIHexV0/Agents/AgentPromptBuilder.swift` 的解释性 prompt 文案和 section label。
- 保留 `AgentDecisionEnvelope` / `AgentOrder` 的 JSON schema、字段名、枚举 rawValue 和 parser 约束。
- 保留 `LocalLLMDecisionProvider`、`AgentDecisionParser`、`AgentCommandMapper`、`MockAIClient`、`TurnManager` 和规则执行链不变。

## 非目标

- 不启用真实 LLM。
- 不修改 `schemaVersion`、`agentId`、`turn`、`intent`、`orders`、`type`、`divisionId`、`toRegionId`、`targetDivisionId`、`stance`、`reason` 等机器字段。
- 不修改 `move`、`attack`、`hold`、`resupply` 这些命令 rawValue。
- 不改 `guderian`、`germany`、`allies`、region id、division id 或任何 Codable/rawValue。
- 不迁移 `MockAIClient` 的 Bastogne fallback 算法。

## 架构依据

- Legacy Agent D / Local LLM 路径默认不是当前战争 AI 主路径，但仍是保留的回归参考和未来可插拔入口。
- 玩家、AI 和后续聊天命令最终仍必须走 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine`。
- Prompt 可以指导 LLM 使用三国语义，但输出仍必须是 parser 可接受的 JSON。

## 实现步骤

1. 将 system prompt 从二战原型定位改为 `三国棋策 Agent` / Three Kingdoms hex strategy。
2. 在系统约束中追加：把兼容 `division` 理解为武将统领的军队，决策要尊重可见官道、粮草/补给、郡县控制和可达交战机会。
3. 将 user prompt 的任务说明改为给“军队”下达作战命令，并要求 intent / reason 使用武将、郡县、官道、粮道、围城压力和可见交战措辞。
4. 将展示 section label 从 `Friendly divisions` / `Known enemy divisions` / `Supply` 改为 `Friendly armies` / `Known hostile armies` / `Grain and supply`。
5. 保持 JSON schema block 的 key 和命令 rawValue 不变，只调整 placeholder 文案。

- `AgentPromptBuilder` 不再把本地 LLM 定位为 WWII 原型。
- prompt 明确引导三国武将、道路、粮草和交战语义。
- JSON schema、字段名、命令 rawValue、parser / mapper 行为无变化。
