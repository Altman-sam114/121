# v2.4 Agent 记录展示名兼容层

## 目标

减少 AI 面板、交互日志和上下文摘要中的 raw agent id / provider 残留，让兼容 `guderian` id 以张辽展示，让 `MockAI` 以兼容武将 AI 展示；底层 id、provider、JSON schema 和解析校验保持不变。

## 范围

- `WWIIHexV0/Agents/AgentDecisionRecord.swift`
- `WWIIHexV0/Turn/TurnManager.swift`
- `WWIIHexV0/UI/AgentPanelView.swift`
- `WWIIHexV0/App/AppContainer.swift`
- 同步 `README.md`、`md/flow/flow.md`、`md/prompt/README.md` 和 `update_log.md`

## 禁止项

- 不改 `AgentDecisionEnvelope.agentId`、`AgentDecisionRecord.agentId`、`provider`、Codable schema 或 parser 校验。
- 不改 `MockAIClient` 的目标选择、命令生成、Bastogne fallback 或算法。
- 不改 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine` 执行链。
- 不跑本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构依据

- `guderian` 仍是 Legacy Agent D 的兼容 id，旧阿登校验和 AgentDecision parser 需要保留该 id。
- 发布 UI 不应把兼容 id 当作人物名展示；展示层可通过 computed helper 把 raw id/provider 映射为三国可读名称。
- 调试 JSON 可作为 UI 文本做展示脱敏，但不改变 `AgentDecisionRecord.rawJSON` 原始存储。

## 实现步骤

1. 在 `AgentDecisionRecord` 增加 `agentDisplayName`、`providerDisplayName` 和 `debugJSONDisplay` computed helper。
2. `TurnManager.contextSummary`、上游 Agent 事件和 diagnostics 改为中文摘要，并使用 agent 展示名。
3. `AgentPanelView` 的 Agent、来源、摘要和调试 JSON 使用展示字段，不直接显示 `guderian` / `MockAI`。
4. `AppContainer.runAIIfNeeded` 的 AI 回合结果消息和交互日志改为中文，并使用 provider 展示名。
5. 更新文档，明确 raw id/schema 仍保留兼容。

## 验收标准

- AI 面板主字段不再直接展示 `guderian` 或 `MockAI`。
- AI 回合摘要和交互日志使用中文与三国展示名。
- `AgentDecisionRecord` 的 raw id、provider、raw JSON 存储和解析校验保持兼容。

## 遗留风险

- `MockAIClient` 的 legacy 决策文本仍可能在意图、理由和调试 JSON 中出现 Bastogne / armor / artillery；后续按 `v2.4_mockai_rationale_sanguo_localization.md` 单独迁移。
