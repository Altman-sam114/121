# v2.4 武将道路与交战日志中文化兼容层

本文记录“武将道路/交战日志摘要仍是英文片段”的小步实现提示词与交付边界。执行前仍必须阅读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和总提示词 `md/prompt/v2.0-三国迁移/codex-v2.0-三国aiagent迁移总提示词.md`。

## 目标

- 沿续“玩家可见中文优先”和“注重武将、道路、交战”的 v2.4 方向，先把新增武将影响日志片段三国化。
- `GeneralCombatInfluenceSummary.logFragment` 输出中文 `武将影响：张辽 攻击 +1，沮授 防御 +1` 这类片段。
- `GeneralMovementInfluenceSummary.logFragment` 输出中文 `武将道路：曹操 机动 +2 (上限 3->5)` 这类片段。
- 保持道路机动、攻防修正、移动和交战执行管线完全不变。

## 非目标

- 不重写全部事件日志，不一次性替换所有英文移动/攻击/撤退文案。
- 不改 `CombatRules` 伤害公式、`MovementRules` 路径搜索、`CommandExecutor` 状态写入顺序或 `RuleEngine` 入口。
- 不新增完整战报面板、头像、技能树、士气、单挑或道路建设制度。
- 不跑本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构边界

```text
GeneralAssignment 快照
  -> GeneralInfluence movement/combat summary
  -> logFragment 中文审计片段
  -> CommandExecutor movementLog / combatLog
  -> GameState.eventLog
```

- 日志片段是审计输出，不是状态权威。
- 仍只在道路加成或攻防修正非 0 时输出，避免日志噪声。
- 武将显示名仍优先使用 `generalDisplayName`，缺失时回退到武将 id。

## 验收标准

- 武将道路机动和交战影响日志片段中文优先。
- 计算字段、规则执行和状态写入不变。
- 玩家与 AI 移动/攻击仍走 `Command / RuleEngine / CommandExecutor`。

## 风险

- 本轮只中文化武将影响片段，移动主体、攻击主体、撤退和毁灭等历史事件日志仍有英文残留。
- 未做运行时 UI 烟测，具体面板显示宽度和可读性等待云端 CI、后续 Agent C artifact 复判或人工授权运行检查。
