# v2.4 郡县检查器官道受压摘要提示词

## 1. 目标

继续强化“武将、道路、交战”的只读态势反馈：在 `RegionInspectorView` 的“官道覆盖”摘要中追加当前郡县官道受压数量，让玩家查看郡县时能同时判断修路、行军、粮道和接战风险。

## 2. 范围

- 仅修改只读 UI 派生与展示：
  - `MapDisplayAdapter.inspectorState`
  - `RegionInspectorState`
  - `RegionInspectorView.roadSummary`
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改移动、攻击、敌控区、道路修缮、粮道、补给、外交或命令执行规则。

## 3. 实现要求

- `RegionInspectorState` 新增只读 `pressuredRoadHexCount`。
- 统计口径：
  - 只统计当前 region 内 `hasRoad == true` 的 hex。
  - 压迫来源只来自当前 viewer 可见、未毁灭、且 `Faction.isHostile(to:)` 判定 hostile 的军队。
  - hostile 军队邻近或占据官道格时，该官道格计入受压。
- 不直接裸用 `MovementRules.isEnemyZoneOfControl` 扫描全局军队，避免隐藏敌军位置通过郡县面板泄漏。
- UI 文案使用“受压”，不要把该只读摘要写成完整规则 ZOC 或真实胜率判断。

## 4. 禁止项

- 不改 `MovementRules.isEnemyZoneOfControl` 当前规则。
- 不改 `CombatRules`、`SupplyRules`、`CommandValidator`、`RuleEngine`、`CommandExecutor`。
- 不让不可见敌军影响玩家可见的郡县官道受压摘要。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 郡县检查器“官道覆盖”能显示覆盖数和受压数。
- 受压来源只来自可见且未毁灭的 hostile 军队，不泄漏不可见敌军。
- 真实移动、交战、道路、粮道、补给和命令执行规则不变。
- 文档明确这是只读态势摘要，不是完整外交、借道或敌控区制度变更。
