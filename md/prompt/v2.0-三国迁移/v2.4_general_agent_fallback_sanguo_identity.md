# v2.4 兼容武将 Agent 身份三国化

## 目标

将 Legacy Agent D 默认武将配置从二战人物显示名迁移为三国武将语义，让 AI 面板、审计记录和兼容 fallback 不再显示 Heinz Guderian；保持旧 `guderian` id、raw faction、JSON schema 和回归兼容不变。

## 范围

- `WWIIHexV0/Data/general_agents.json`
- `WWIIHexV0/Agents/AgentConfiguration.swift`
- `WWIIHexV0/Agents/GameAgent.swift`
- `WWIIHexV0/Agents/MockAIClient.swift`
- 同步 `README.md`、`md/flow/flow.md`、`md/prompt/README.md` 和 `update_log.md`

## 禁止项

- 不改 `guderian` id，不改 `GeneralAgentDefinition` schema，不改 `assignedDivisionIds`。
- 不改 MockAI 启发式、目标排序、Bastogne fallback、命令 schema 或规则执行。
- 不改 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine` 执行链。
- 不跑本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构依据

- `DataLoader.validate` 对旧阿登场景仍要求 `general_agents.json` 中存在 `guderian`，且其 `assignedDivisionIds` 覆盖德军初始单位；因此本轮只能改显示名和 personality 文案，不能替换 id。
- `GameAgent.guderian(from:state:)` 仍是 Legacy Agent D 启动兼容入口；默认主战争 AI 路径已经走 Marshal / ZoneDirective / GeneralAgent 编排。
- 三国武将影响道路和交战的主路径来自 `GeneralRegistry` / `GeneralAgent` / `GeneralInfluence`；本轮只清理兼容 agent 身份，不新增武将技能规则。

## 实现步骤

1. 将 `general_agents.json` 中 `guderian` 的 `name` 改为三国武将显示名。
2. 将其 `personalityPrompt` 改为骑军突击、官道机动、兵力集中、快速合围和攻城支援语义。
3. 将 `GameAgent.guderianFallback` 的 fallback `name` 和 prompt 同步为同一三国武将语义。
4. 清理源码注释中把 Legacy Agent D 绑定到 Guderian 的过期描述。
5. 更新 README、flow、prompt 索引和 update log，明确旧 id 仍是兼容 id。

## 验收标准

- 默认兼容武将显示名不再是 Heinz Guderian。
- 兼容 personality 文案体现武将、官道、骑军突击、合围和攻城支援。
- `guderian` id、rawValue、旧单位分配、MockAI 算法和命令执行链保持不变。

- `MockAIClient` 的 legacy 决策启发式仍会向 Bastogne fallback，并保留 armor / artillery 等旧战术文案；后续按 `v2.4_mockai_rationale_sanguo_localization.md` 单独迁移。
