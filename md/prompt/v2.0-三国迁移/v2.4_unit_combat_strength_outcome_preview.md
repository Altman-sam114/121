# v2.4 军队接战剩余兵力预判提示词

> 本阶段承接 `v2.4_unit_combat_target_comparison.md`。执行前必须先读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和相关源码。

## 目标

- 继续强化“交战”可读性，让玩家选中军队时不只看到预计伤害和反击风险，还能直接看到战后敌我剩余兵力。
- 复用现有 `CombatRules.attackDamage`、`counterAttackDamage` 和 `Division.strength/maxStrength`，在最多三名射程内敌军的接战预判行里显示敌军预计剩余兵力；若目标会反击，同时显示我方预计剩余兵力。
- 保持首选目标的武将影响与交战审计摘要，方便判断武将、地形、围城、河流和兵种因素。

## 实现要求

- `AppContainer.selectedUnitCombatPreviewNotes` 继续保持只读派生属性，只读取当前 `GameState`。
- 目标剩余兵力按 `max(0, target.strength - damage.strengthDamage) / target.maxStrength` 显示。
- 有反击时，我方剩余兵力按 `max(0, attacker.strength - counterDamage.strengthDamage) / attacker.maxStrength` 显示；无反击时继续显示“无反击”。
- 继续保留现有目标排序：预计伤害高者优先，伤害相同时反击伤害低者优先，再按距离和名称稳定排序。
- 不把预判写入日志、审计记录或持久状态。
