# v2.4 军队可达官道距离预判兼容层提示词

## 1. 目标

继续强化“武将、道路、交战”的军队详情只读反馈：`selectedUnitMobilityPreviewNotes` 已显示基础/有效机动、武将官道加成、当前位置官道状态和本回合可达官道安全/受压数量。本轮补齐“最近可达官道距离”和“最近安全官道距离”，让玩家判断当前军队本回合接入官道的远近与风险。

## 2. 范围

- 仅修改 `WWIIHexV0/App/AppContainer.swift` 的 `reachableRoadAccessNote` 只读文案。
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改 `MovementRules`、`CombatRules`、`GeneralInfluence`、`WarCommandExecutor`、`CommandValidator`、`RuleEngine` 或真实执行规则。
- 不修改 UI 入参、JSON schema、Codable 字段或 Xcode project。

## 3. 实现要求

- 继续复用 `MovementRules.movementRange` 得到本回合可达 hex。
- 只在可达官道集合内计算当前军队 hex 到官道 hex 的 hex 距离。
- 在已有“本回合可入 N 格、安全/受压数量”基础上追加最近可达官道距离。
- 若存在未受 `MovementRules.isEnemyZoneOfControl` 压迫的可达官道，追加最近安全官道距离。
- 没有可达官道时保留既有“尚不能接入官道”或当前已在官道时不重复提示。

## 4. 禁止项

- 不改变移动范围、道路加成、敌控区、补给、粮道、交战、撤退或武将加成规则。
- 不新增完整路径规划、跨回合接道计划、路径安全、胜率、隐藏敌军推断或敌方回合模拟。
- 不把 hex 距离包装成真实移动成本；真实路径和行动结果仍以 `WarCommandExecutor -> RuleEngine` 为准。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 存在可达官道时，“道路机动”预判显示最近可达官道距离。
- 存在未受敌控区压迫的可达官道时，同一摘要显示最近安全官道距离。
- 可达官道总数、安全/受压数量和“尚不能接入官道”旧语义保持不变。
- 真实道路、移动、敌控区、补给、交战和命令执行规则保持不变。
- 文档明确本轮只是只读 UI 派生文案改进，运行时 UI 表现仍需云端 CI、后续 Agent C artifact 复判或人工授权运行检查。
