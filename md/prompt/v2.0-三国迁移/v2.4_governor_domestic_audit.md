# v2.4 太守内政建议审计兼容层

本文记录“君主、军师、太守、武将 AI Agent 分层”中的太守内政小步实现提示词与交付边界。执行前仍必须阅读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和总提示词 `md/prompt/v2.0-三国迁移/codex-v2.0-三国aiagent迁移总提示词.md`。

## 目标

- 新增 deterministic `GovernorAgent`，承接君主姿态并读取经济总账、郡县、道路、补给和生产队列。
- 新增 `GovernorDecisionRecord`，把太守内政建议写入 `GameState.governorRecords`。
- `TurnManager` 在君主姿态塑形之后、军师目标编排之前调用太守层，并把建议追加到 `DirectiveEnvelope.theaterContext` 与 AI raw JSON。
- `AgentPanelView` 显示太守 agent、内政重点、重点郡县、建议生产和 rationale。

## 非目标

- 不自动执行生产、不直接扣资源、不直接改人口/军械/粮草库存。
- 不新增完整内政 directive validator / executor，不实现修路、屯田、治安的状态变更。
- 不实现太守任命 UI、城市建设树、税率、民心或完整郡县治理系统。
- 不跑本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构边界

```text
RulerAgent.adjust
  -> GovernorAgent.plan
  -> GovernorDecisionRecord + DirectiveEnvelope.theaterContext
  -> StrategistAgent.plan
  -> GeneralAgent.plan
  -> WarCommandExecutor
  -> RuleEngine
```

- `GovernorAgent` 只读 `GameState` 并输出 Codable 记录。
- 太守建议可以影响后续 Agent 的上下文解释，但本轮不生成底层 `Command.queueProduction`。
- 后续若让太守真正排产，必须经 `Command.queueProduction -> CommandValidator -> EconomyRules.queueProduction`。
