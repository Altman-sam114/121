# v2.4 外交面板展示名收敛提示词

## 目标

- 继续推进三国多势力、道路和交战边界可读性，把 `DiplomacyPanelView` 中玩家可见的国家、集团、关系对象、君主重点防区和外交对象从 raw id fallback 改为中文展示名。
- 外交面板应优先显示 `CountryProfile.name`、`DiplomaticBloc.name`、`Faction.displayName` 和 `RootGameView` 生成的防区展示名；缺资料时使用“未知势力 / 未知集团 / 未知防区”等中文占位。

## 非目标

- 不改变 `CountryId`、`DiplomaticBlocId`、`FrontZoneId`、Codable、JSON schema、外交关系状态或命令执行。
- 不改变 `DiplomacyState.applyProposal`、敌对判定、借道/同盟通行、道路、粮道、移动或交战规则。
- 不隐藏调试 JSON，不迁移 Legacy Agent D schema 字段。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构依据

```text
DiplomacyState.countries / blocs / relations / rulerRecords / diplomatRecords
  -> DiplomacyPanelView 只读展示
  -> 玩家判断外交关系、借道、敌对和交战边界
```

- 外交面板是只读 UI，不应推导或修改外交规则。
- 玩家和 AI 的真实外交变化仍只能由 `Command.proposeDiplomacy -> CommandValidator -> CommandExecutor -> RuleEngine` 完成。
- 防区展示名由 `RootGameView` 的现有 `agentPanelFrontZoneDisplayNames` 生成，避免面板自行暴露 `FrontZoneId.rawValue`。

## 实现范围

- `DiplomacyPanelView` 增加 `frontZoneDisplayNames` 入参，默认空字典以兼容 preview / 旧调用。
- 国家行使用 `country.name`，空名或等于 raw id 时 fallback 到 `country.faction.displayName`。
- 国家所属集团和集团列表使用 `DiplomaticBloc.name`，空名或等于 raw id 时 fallback 到 `Faction.shortDisplayName + "集团"`。
- 关系列表使用国家展示名，不直接拼 `firstCountryId.rawValue` / `secondCountryId.rawValue`。
- 君主 / 外交官记录使用 `AgentDecisionRecord.displayName(forAgentId:)`；重点防区和外交对象使用展示名 helper。
- `RootGameView` 将现有防区展示名字典传给外交面板。
