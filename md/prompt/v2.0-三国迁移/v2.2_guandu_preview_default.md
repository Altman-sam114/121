# v2.2 官渡默认剧本预览小步记录

完成日期：2026-07-04

## 1. 本轮目标

沿 `codex-v2.0-三国aiagent迁移总提示词.md` 的 v2.2 方向，先完成“默认新局加载三国剧本路径”的最小可验证小步：

- 新增 `guandu_200_scenario.json` 和 `guandu_200_regions.json`。
- `DataLoader.loadInitialGameState()` 优先加载官渡预览数据，失败时再 fallback 到旧阿登数据。
- MapEditor 默认资源桥读写官渡预览数据，避免编辑器默认地图和游戏默认入口分叉。
- 保留当前旧 `.germany/.allies` 回合兼容，避免一轮内把 v2.1 未完成的 turn order 强行扩大。

## 2. 实现摘要

代码和数据变化：

- 新增 `WWIIHexV0/Data/guandu_200_scenario.json`：
  - `id = guandu_200`，展示名为“官渡前夜 200 迁移预览”。
  - 40 个 hex，8 个 region 覆盖，目标点包括邺城、官渡、许昌、汝南、襄阳、寿春、洛阳残垣。
  - 当前可行动双方仍用旧 rawValue：`allies` 作为袁绍兼容玩家方，`germany` 作为曹操兼容 AI 方。
  - `factions` 列表包含 `cao/yuan/liuBei/sun/liuBiao/maTeng/han/neutral`，供外交 profile 和后续多势力迁移承接。
- 新增 `WWIIHexV0/Data/guandu_200_regions.json`：
  - 8 个郡县/区域：邺城、黎阳渡、官渡、许昌、汝南、襄阳、洛阳残垣、寿春。
  - 使用 `theaterId` 种子，保留 `regionToTheater` 只作初始/基础战区归属的语义。
  - 汉室、刘表、孙氏等作为数据 controller/owner 出现在默认地图，但不进入当前旧二元回合。
- `DataLoader.loadInitialGameState()` 默认先尝试 `guandu_200_scenario` + `guandu_200_regions`。
- `MapEditorGameResourceBridge` 默认资源名改为 `guandu_200_scenario` + `guandu_200_regions`。
- `WWIIHexV0.xcodeproj/project.pbxproj` 将两个新 JSON 加入 iOS/macOS 主资源。

## 3. 兼容边界

- 本轮不是完整 80-160 hex 官渡首发地图，只是可加载的默认路径预览。
- 当前 `GamePhase` / `CommandValidator` 仍只允许旧 `.germany/.allies` 行动，所以官渡预览用 legacy rawValue 承载曹/袁可行动双方。
- 其他三国势力当前只作为地图和外交数据基础，不是 playable turn order。
- 旧阿登 JSON 仍保留，作为 fallback 和历史回归参考。

## 4. 后续建议

- 设计 `TurnOrder` / `ScenarioFactionSetup`，让 `cao/yuan/...` rawValue 可以真正进入回合循环。
- 将 `guandu_200` 扩到 80-160 hex、20-40 region，并补齐下邳、江夏等总提示词建议目标。
- 迁移 `unit_templates.json` 为三国兵种模板，避免官渡数据继续复用 Panzer/Division 模板 id。
- 继续把 MapEditor UI 术语从“部队/阵营”推进到军队/武将/势力，并支持武将初始分配。
