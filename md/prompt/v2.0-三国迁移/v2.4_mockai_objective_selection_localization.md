# v2.4 Legacy MockAI 目标选择去阿登硬编码

## 目标

继续收敛旧二战题材残留，把 Legacy Agent D `MockAIClient` 的进军目标选择从硬编码 `Bastogne` 改为读取当前 `AgentContext.objectives` 的剧本目标，避免阿登 fallback 名称进入 AI 记录理由。

## 范围

- 修改 `WWIIHexV0/Agents/MockAIClient.swift`。
- 保留 Legacy Agent D 管线，只改目标选择 helper 和注释。
- AI 移动理由仍使用当前 objective 的展示名；官渡默认剧本会显示许昌、邺城等当前目标名。
- 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 非目标

- 不恢复 Legacy Agent D 为默认战争 AI 主路径。
- 不改变 `AgentContext` schema、`AgentOrder` schema、`Command` / `ZoneDirective` / `WarCommandExecutor` / `RuleEngine` 管线。
- 不修改 `DataLoader` 阿登 fallback、`GameState.initial()`、`MapState.ardennesV0()` 或胜负规则。
- 不删除旧阿登数据、旧测试或兼容 rawValue。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 实现步骤

1. 将 `MockAIClient.decide(context:)` 中 `Bastogne` 优先选择改为 `primaryObjective(context:)`。
2. `primaryObjective` 优先选择有 regionId 且不由当前 faction 控制的目标，其次选择有 regionId 的目标，最后 fallback 到第一个 objective。
3. 更新文档，说明 Legacy MockAI 已不再硬编码阿登目标；更大的阿登 fallback 默认路径另列为后续风险。

## 验收标准

- `MockAIClient` 不再用 `objective.name == "Bastogne"` 选择进军目标。
- 默认官渡或后续三国剧本的 Legacy MockAI fallback 会按当前 objective 列表选择目标。
- 本轮不触碰规则执行、地图、数据 schema 或默认战争 AI 主路径。
