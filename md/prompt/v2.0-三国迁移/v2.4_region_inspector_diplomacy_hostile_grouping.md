# v2.4 郡县检查器外交敌对分组兼容层提示词

## 1. 目标

继续强化三国多势力、武将、道路和交战边界：郡县检查器已经能展示外交关系摘要，但“可见敌军 / 可见非敌对军队”和官道受压来源仍按静态 `Faction.isHostile(to:)` 分组，可能与 `DiplomacyState` 中的中立、停战、同盟或共同作战状态不一致。本轮只把郡县检查器的只读分组接到外交关系 helper，先不改变真实攻击、移动、道路、粮道、部署或 AI 执行规则。

## 2. 范围

- 修改只读外交 helper 和郡县检查器派生：
  - `WWIIHexV0/Core/DiplomacyState.swift`
  - `WWIIHexV0/SpriteKit/MapDisplayAdapter.swift`
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`

## 3. 实现要求

- 在 `DiplomacyState` 增加 faction-to-faction 只读 helper：
  - 能从双方 primary country 读取 `DiplomaticRelation.status`。
  - `.hostile` / `.atWar` 判为敌对；`.neutral` / `.allied` / `.coBelligerent` 判为非敌对。
  - 缺国家或关系建档时 fallback 到当前 `Faction.isHostile(to:)`，保持旧数据兼容。
- `MapDisplayAdapter.inspectorState` 的 `visibleEnemyDivisions`、`visibleNonHostileDivisions`、官道受压计数和官道压力来源使用该外交 helper。
- `nonHostileRelationSummary` 继续展示 `DiplomaticRelation.status/tension`；分类和展示必须使用同一外交状态来源。

## 4. 禁止项

- 不修改 `Faction.isHostile(to:)` 语义。
- 不修改 `CommandValidator.validateAttack`、`RuleEngine`、`MovementRules`、`SupplyRules`、`WarDeploymentManager` 或真实命令执行。
- 不实现完整借道、同盟通行、共同作战堆叠、补给共享或外交宣战制度。
- 不新增持久字段，不修改 JSON schema，不改 Xcode project。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 郡县检查器“可见敌军”和“可见非敌对军队”分组与 `DiplomaticRelation.status` 一致。
- 郡县官道受压计数和最近来源只来自外交关系判定为 hostile / atWar 的可见军队。
- 缺外交建档的旧数据仍能 fallback 到 `Faction.isHostile(to:)`。
- 真实攻击、移动、道路、粮道、部署、补给和 AI 执行规则不变。
