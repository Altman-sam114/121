# v2.5 Region 胜负 fallback 目标 id 同源提示词

## 目标

继续推进三国迁移，把 `RegionVictoryRules` 的旧 fallback 胜负分析从展示城市名查找改为 objective id 查找，让 region 只读胜负分析和 `VictoryRules` 使用同一套稳定目标锚点，避免旧英文地名或后续三国展示名改动影响胜负判断。

## 范围

- 修改 `WWIIHexV0/Rules/RegionVictoryRules.swift`。
- 更新 `md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 非目标

- 不改变官渡 `ScenarioVictoryCondition` 主路径。
- 不改变 `VictoryReason` rawValue、旧存档字段、objective id、region id、JSON schema 或测试 fixture。
- 不改变 `VictoryRules`、HUD、RegionRuleSystem API、地图数据或 UI 布局。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试 / app 启动 / 全量构建。

## 架构依据

- 默认官渡剧本通过 `VictoryState.scenarioConditions` 的 `controlObjective` 条件判断胜负。
- `VictoryRules` 的旧 fallback 已通过 `MapState.controllerOfObjective(id:)` 读取 `bastogne` / `st_vith`，不依赖展示名。
- `MapState.ardennesV0()` 已保留旧 objective id，同时把 fallback 展示名改为三国语义。

## 实现要求

1. `RegionVictoryRules.assessVictory(in:)` 未命中剧本条件时，读取 `state.map.controllerOfObjective(id: "bastogne")` 和 `state.map.controllerOfObjective(id: "st_vith")`。
2. 移除 region 城市展示名查找 helper，避免 `"Bastogne"` / `"St. Vith"` 字符串继续作为规则锚点。
3. 保持旧 fallback 胜负结果、winner、reason 不变。
4. 只做只读 analysis 口径同源化，不修改 `GameState.victoryState`。

## 验收标准

- `RegionVictoryRules.swift` 不再包含 `"Bastogne"` / `"St. Vith"` 字符串。
- Region 只读胜负 fallback 通过 objective id 读取控制权。
- 官渡 scenario victory 主路径、旧 fallback reason rawValue 和命令/AI 管线不变。
