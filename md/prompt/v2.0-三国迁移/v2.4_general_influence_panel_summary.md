# v2.4 武将道路与交战面板摘要提示词

> 本阶段承接 `v2.4_general_road_combat_rules.md`、`v2.4_general_road_log_audit.md`、`v2.4_general_combat_log_audit.md` 和 `v2.4_player_general_tactic_shaping.md`。执行前必须先读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和相关源码。

## 目标

- 让玩家选中武将防区时，在 `GeneralCommandPanelView` 直接看到当前武将对道路机动和交战攻防的只读摘要。
- 复用 `GeneralInfluence` 的既有 `movementSummary` / `combatSummary`，避免 UI 复制道路或战斗规则。
- 保持移动、攻击、修路、占领、动态方面和防区归属的执行边界不变。

## 范围

- 主要文件：
  - `WWIIHexV0/App/AppContainer.swift`
  - `WWIIHexV0/UI/GeneralCommandPanelView.swift`
  - `WWIIHexV0/UI/RootGameView.swift`
- 文档文件：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改 Xcode project、不新增资源、不改 `GeneralInfluence`、`MovementRules`、`CombatRules`、`WarCommandExecutor` 或 `RuleEngine`。

## 实现要求

- `AppContainer` 增加只读派生属性，基于当前选中 `FrontZone`、`GeneralAssignment`、麾下军队和 `GameState` 生成 1-2 条中文摘要。
- 道路摘要应来自 `GeneralInfluence.movementSummary(for:in:)`，说明有多少支麾下军队获得官道机动加成和最高加成。
- 交战摘要应来自 `GeneralInfluence.combatSummary(attacker:defender:in:)`，只统计当前已进入射程的敌我配对；未接敌时显示后续接战再计算。
- `GeneralCommandPanelView` 新增只读展示区，显示“道路与交战”摘要；不得在 View 内执行命令或直接改状态。
- `RootGameView` 在两个现有 `GeneralCommandPanelView` 调用点传入摘要。

## 禁止项

- 不允许 UI 直接修改 hex controller、单位坐标、动态方面、防区、资源、道路或生产队列。
- 不允许把道路或交战规则复制到 SwiftUI View 中。
- 不允许新增本机 Xcode build/test、模拟器、Probe、Smoke、Stage Regression、Dynamic Theater Regression、Full 或性能测试。
