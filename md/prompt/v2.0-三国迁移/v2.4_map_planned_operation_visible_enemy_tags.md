# v2.4 地图计划军令可见近敌短标签提示词

## 1. 目标

继续强化“武将、道路、交战”的玩家宏观军令反馈：地图上的计划军令标签不仅显示武将、攻守战术和官道/受压，还要显示最近可见 hostile 敌军短标签；同时把计划军令面板摘要与地图短标签的受压/近敌口径统一为当前 viewer 可见敌军，避免隐藏敌军通过只读预览泄漏。

## 2. 范围

- 仅修改只读 UI 派生与展示：
  - `WWIIHexV0/SpriteKit/BoardScene.swift`
  - `WWIIHexV0/App/AppContainer.swift`
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改 `PlayerPlannedOperation` schema，不修改真实移动、攻击、道路、粮道、补给、外交、AI 指令执行或 `RuleEngine`。

## 3. 实现要求

- `BoardScene` 绘制计划军令时，继续只读 `PlayerCommandState.plannedOperations`。
- 地图计划军令的官道受压判断必须只统计当前 operation faction 可见、未毁灭、且 `Faction.isHostile(to:)` 判定 hostile 的敌军。
- 地图计划军令标签追加最近可见 hostile 敌军短标签，例如 `近袁骑2`，保持短文本，避免压住地图。
- `AppContainer.selectedGeneralPlannedOperationRows` 的受压和近敌摘要同样改为可见 hostile 口径，与提交前目标预览一致。
- 地图标签背景可做小幅尺寸/字号适配，但不得改成交互面板或大段说明。

## 4. 禁止项

- 不改 `Command` / `ZoneDirective` / `WarCommandExecutor` / `CommandValidator` / `RuleEngine`。
- 不改 `MovementRules`、`CombatRules`、`SupplyRules`、`GeneralInfluence` 的规则数值或公式。
- 不新增持久字段，不修改 `PlayerPlannedOperation` Codable schema。
- 不让不可见敌军进入地图计划标签或武将面板计划摘要。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 地图计划军令标签在有可见 hostile 敌军时显示最近敌军短标签，且不显示隐藏敌军。
- 地图计划军令受压标签和武将面板计划军令摘要的受压/近敌口径都使用可见 hostile。
- 真实计划军令记录、命令执行、移动、攻击、道路、粮道、补给、外交和 AI 决策规则不变。
- 文档明确这是只读地图/面板反馈，不代表完整路径安全、胜率或攻击许可制度。
