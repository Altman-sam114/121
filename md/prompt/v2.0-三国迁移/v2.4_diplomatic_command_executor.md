# v2.4 外交命令执行兼容层

本文记录“君主、外交、太守、军师、武将 AI Agent 分层”中的外交提案执行小步实现提示词与交付边界。执行前仍必须阅读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和总提示词 `md/prompt/v2.0-三国迁移/codex-v2.0-三国aiagent迁移总提示词.md`。

## 目标

- 新增底层外交命令 `Command.proposeDiplomacy(sourceCountryId:targetCountryId:proposal:)`，让外交关系变化也进入统一命令管线。
- `CommandValidator` 校验阶段、源国家、目标国家、当前行动势力、既有关系和提案合法性。
- `CommandExecutor` 通过 `DiplomacyState.applyProposal` 最小更新 `DiplomaticRelation.status/tension`，并写入 diplomacy 事件日志。
- `TurnManager.applyDiplomatPlanning` 将有源国家和目标国家的 `DiplomatDecisionRecord` 转成 `Command.proposeDiplomacy`，经 `commandHandler.execute` 执行，并把结果写入 `AgentDecisionRecord.commandResults`。
- 文档明确：`DiplomatAgent` 仍只产出 Codable 提案和审计记录，关系变化只允许通过 `CommandValidator -> CommandExecutor -> RuleEngine`。

## 非目标

- 不新增 `DiplomaticStatus` raw value；不在本轮加入 vassal/truce/pass permission 等完整 schema。
- 不实现真实借道通行、贡赋资源转移、臣属体系、联盟共同胜利或完整外交 AI 谈判。
- 不改完整多势力 turn order，不让三国新增势力进入完整 AI 回合循环。
- 不让外交 Agent、UI 或 MapEditor 直接修改 `DiplomacyState.relations`。
- 不跑本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构边界

```text
RulerAgent.adjust
  -> DiplomatAgent.plan
  -> DiplomatDecisionRecord
  -> Command.proposeDiplomacy
  -> CommandValidator
  -> CommandExecutor / DiplomacyState.applyProposal
  -> GovernorAgent.plan
  -> StrategistAgent.plan
  -> GeneralAgent.plan
  -> WarCommandExecutor
  -> RuleEngine
```

- `DiplomatAgent` 不持有写权限；只读 `GameState`，输出 Codable `DiplomatDecisionRecord` 和调整后的 `DirectiveEnvelope`。
- `TurnManager` 只在记录有源国家和目标国家时生成外交命令；缺目标时仅写诊断，不伪造执行。
- 当前提案映射到既有状态：同盟推进到共同作战/同盟，停战降为中立，讨伐檄文提高敌对/交战紧张度，借道/称臣/奉表只做状态或紧张度级兼容。
- 外交预命令结果写入 `AgentDecisionRecord.commandResults`，不混入单条 `WarDirectiveRecord`。
