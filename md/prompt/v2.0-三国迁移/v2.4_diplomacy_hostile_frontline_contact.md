# v2.4 外交敌对动态前线接触兼容层提示词

## 1. 目标

继续推进三国多势力、武将、道路和交战迁移：道路 ZOC、粮道/围城、安全补员、部署接触、攻击入口、Legacy AI 摘要已逐步接入 `DiplomacyState` hostile / atWar 口径。本轮只迁移 `FrontLineManager` 的动态前线接触和 front line 对手标签，避免中立、停战、同盟或共同作战势力只因动态 theater 相邻就生成敌对前线。

## 2. 范围

- `WWIIHexV0/Rules/FrontLineManager.swift`
  - `makeInitialState` / `tick` / `update` 增加可选 `DiplomacyState` 参数，默认 nil 时保留旧 `Faction.isHostile(to:)` fallback。
  - 动态相邻 theater 是否形成 enemy region 接触改用外交 hostile。
  - `FrontLine.factionB` 选择优先使用外交 hostile 对手。
- 生产运行时调用显式传入 `state.diplomacyState`：
  - `WWIIHexV0/Data/DataLoader.swift`
  - `WWIIHexV0/Core/StrategicStateBootstrapper.swift`
  - `WWIIHexV0/Rules/StrategicStateSynchronizer.swift`
  - `WWIIHexV0/Commands/WarCommandExecutor.swift`

## 3. 实现要求

- 使用 `diplomacyState?.isHostile(between:and:) ?? lhs.isHostile(to: rhs)`，保留历史测试和旧调用兼容。
- `DataLoader.loadGameState(scenarioName:regionName:)` 必须先构造 `DiplomacyState.initial`，再构造 `frontLineState`。
- 不改变 `FrontLine` / `FrontSegment` / `FrontLineState` Codable schema。
- 不改变 `HexTile.controller`、`RegionNode.controller`、`hexToTheater`、`hexToFrontZone` 权威边界。
- `makeSegment` 内压力、补给影响、包围候选继续使用 region controller / 邻接拓扑语义，不改成外交关系。

## 4. 禁止项

- 不迁移 `ZoneCommanderAgent`、`StrategistAgent` 或 `RulerAgent` 的剩余 hostile 派生。
- 不实现完整借道、同盟通行、共同作战堆叠、补给共享或外交宣战制度。
- 不修改 JSON schema、Xcode project、测试文件或 Probe。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 动态 front line 不再因外交非敌对 theater 相邻而生成敌对 front segment。
- `FrontLine.factionB` 不再优先把外交非敌对势力标成敌对前线对手。
- 生产运行时前线初始化、刷新和占领同步路径都传入真实 `DiplomacyState`。
- `FrontLine` / `FrontSegment` / `FrontLineState` schema 不变。
- 本机只跑轻量检查，重验证交给 GitHub Actions。
