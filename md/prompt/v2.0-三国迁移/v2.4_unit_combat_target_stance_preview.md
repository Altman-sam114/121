# v2.4 军队接战目标态势预判兼容层

## 1. 目标

在现有“接战预判”的三目标对比基础上，让玩家不只看到预计伤害、余兵和反击，也能直接看到目标所处地形、是否据城/据关、是否临官道、粮道状态和撤退姿态。该增量继续强化“三国要注重武将，做好道路、交战”：战前判断应让玩家知道为什么某个目标难打、值得打或需要先断粮。

## 2. 范围

- 修改 `WWIIHexV0/App/AppContainer.swift` 的 `selectedUnitCombatPreviewNotes` 目标行私有派生逻辑。
- 复用 `HexTile.baseTerrain.displayName`、`cityName`、`fortressName`、`hasRoad`、`SupplyState.shortDisplayName` 和 `RetreatMode.shortDisplayCode`。
- 在每个目标预判行追加短“态势”片段。
- 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 3. 非目标和禁止项

- 不修改 `CombatRules` 的攻击、防御、地形、河流、围城、侧击、器械攻城或反击计算。
- 不修改 `CommandExecutor` 的攻击、反击、撤退、死守、歼灭、日志结算或状态写入。
- 不修改 `SupplyRules`、粮道判定、围城判定、道路、地图控制权或动态战区。
- 不新增胜率、AI 推荐、目标评分、士气、疲劳、单挑或跨回合围攻计划。
- 不让 UI 预判写入 `GameState`、事件日志、审计记录或持久数据。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 4. 架构依据

- Hex 仍是战术权威，目标态势来自目标所在 `HexTile` 和目标 `Division` 自身状态。
- `UnitInspectorView` 只读展示 `AppContainer.selectedUnitCombatPreviewNotes`，不能绕过规则系统。
- 真实攻击仍必须走 `Command.attack -> RuleEngine -> CommandExecutor`。
- 交战审计仍由 `CombatRules.combatAuditSummary` 负责，目标态势只是给三目标短行补充上下文。

## 5. 实现步骤

1. 新增私有 `combatTargetStanceText(for:)`。
2. 从目标所在 hex 读取地形中文名。
3. 若目标所在 hex 有 `cityName`，追加“据城”；若有 `fortressName`，追加“据关”；若 `hasRoad`，追加“临官道”。
4. 追加目标粮道短名和撤退姿态短码。
5. 将态势片段拼入 `combatTargetPreviewLine`，保持原有伤害、余兵、风险、反击和距离信息。
6. 更新核心文档、流程图、阶段索引和版本日志。

## 7. 文档更新要求

- `README.md`：同步 v2.4 武将道路/交战规则摘要。
- `md/flow/flow.md`：同步当前核心链路说明。
- `md/flow/flowchart.md`：同步接战预判节点文字。
- `md/prompt/README.md`：登记本阶段 prompt。
- `update_log.md`：追加 v2.4 版本记录，写明轻量检查和未跑重测试原因。

## 8. 验收标准

- 每个接战目标行保留预计伤害、结算后敌我剩余兵力、风险、反击和距离。
- 每个接战目标行新增目标态势摘要，至少包含目标地形、粮道短名和撤退姿态短码。
- 位于城池、关隘或官道的目标能额外显示据城/据关/临官道。
- 真实战斗、反击、撤退、死守、粮道、围城、道路和地形规则不变。
- 轻量检查通过；重测试交给云端 CI 或后续 Agent C artifact 复判。

## 9. 风险

- 本轮没有运行 UI 烟测，新增长文本在窄屏、长军队名和 Dynamic Type 下的实际换行仍需后续检查。
- 态势摘要只展示当前目标所在格和目标自身状态，不解释完整伤害公式；详细因素仍以首选目标交战审计为准。
- 完整目标评分、AI 推荐、胜率和多回合围攻计划仍待后续版本。
