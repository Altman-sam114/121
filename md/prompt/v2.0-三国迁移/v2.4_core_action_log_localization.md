# v2.4 核心行动日志中文化兼容层

本文记录“移动、交战、姿态和回合推进主体日志仍有英文残留”的小步实现提示词与交付边界。执行前仍必须阅读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和总提示词 `md/prompt/v2.0-三国迁移/codex-v2.0-三国aiagent迁移总提示词.md`。

## 目标

- 沿续“玩家可见中文优先”和“注重武将、道路、交战”的 v2.4 方向，中文化最常见的核心行动日志。
- `CommandExecutor` 的移动、攻击、反击、自动撤退、额外兵力损失、歼灭、死守、允许撤退、回合推进和动态方面推进日志改为中文。
- `WarCommandExecutor` 中同源的动态方面推进和前线变化审计日志改为中文。
- 保持所有规则计算、状态写入、占领同步、动态方面推进和 AI 指令执行管线不变。

## 非目标

- 不一次性替换所有历史英文文案、Legacy Agent D 文案、测试基线或调试输出。
- 不改 `CombatRules`、`MovementRules`、`SupplyRules`、`TheaterSystem`、`WarDeploymentManager` 的规则逻辑。
- 不改 `Command` / `ZoneDirective` / `WarCommandExecutor` / `RuleEngine` 统一入口。
- 不跑本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构边界

```text
Command / ZoneDirective
  -> RuleEngine / WarCommandExecutor
  -> CommandExecutor
  -> 中文 EventLog 审计文本
```

- 日志是审计输出，不是状态权威。
- Hex controller、Division coord、hexToTheater、hexToFrontZone 的写入逻辑不变。
- 武将道路和攻防摘要仍由 `GeneralInfluence` 生成，本轮只让主体行动文本与这些摘要同一中文口径。

## 验收标准

- 核心移动、攻击、反击、姿态、回合推进和动态方面事件日志中文优先。
- 武将道路与交战摘要能自然拼入中文主体日志。
- 玩家与 AI 命令仍走 `Command / ZoneDirective -> WarCommandExecutor / RuleEngine`，没有绕过规则系统。

## 风险

- 本轮仍不是全项目文案清零；历史 README、旧测试、Legacy Agent D、MockAI rationale 和部分 debug 审计文本可能仍保留英文。
- 未做运行时 UI 烟测，日志宽度、换行和实际面板可读性仍待云端 CI、后续 Agent C artifact 复判或人工授权运行检查。
