# v2.4 武将面板计划军令官道与受压摘要提示词

## 1. 目标

继续强化“三国要注重武将，做好道路、交战”的可见性：让 `GeneralCommandPanelView` 的计划军令列表不仅显示指令类型、最终战术和目标，还显示执行武将、目标名称、源/目标代表 hex 的官道状态与敌军压迫摘要，使面板反馈与地图计划军令标识保持一致。

## 2. 范围

- 在 `AppContainer` 增加只读计划军令摘要派生，读取 `PlayerPlannedOperation`、`WarDeploymentState.frontZones`、`GeneralRegistry`、`MapState` 和当前 `Division` 位置。
- `GeneralCommandPanelView` 只消费摘要行，不在 View 内直接推导道路、敌军压迫或 region 名称。
- `RootGameView` 使用新的摘要行传参。
- 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 3. 禁止项

- 不修改 `PlayerPlannedOperation` Codable schema。
- 不修改 `MovementRules`、`CombatRules`、`GeneralInfluence`、`WarCommandExecutor`、`CommandValidator`、`RuleEngine` 或真实移动/攻击/补给规则。
- 不把面板摘要写成完整路径安全、真实胜率、移动后同回合攻击或敌方回合反制模拟。
- 不绕过 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine`。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 4. 实现要求

- 摘要行包含：
  - 武将姓名，优先 `FrontZone.generalAssignment.generalDisplayName`，否则回退 `createdByGeneralId` 或 registry 名称。
  - `DirectiveType.displayName`。
  - `TacticName.displayName`，缺失时显示未定战术。
  - 目标 region 名称，缺失时回退 source region 或 zone 名称。
  - 官道短摘要：进攻显示双道、源道、目道或无道；防御显示据道或离道。
  - 压迫短摘要：源目受压、源受压、目受压或据点受压。
- 压迫判断必须使用 `Faction.isHostile(to:)`，只统计未毁灭且距离源/目标代表 hex 小于等于 1 的敌对军队。
- 摘要只读，不修改 `GameState`。

## 6. 验收标准

- 武将面板计划军令列表能展示武将、最终战术、目标名、官道状态和源目受压摘要。
- 地图计划军令标识和面板计划军令摘要使用同一套源/目标代表 hex 口径。
- 规则执行、状态 schema、移动、道路成本、交战、补给和微操锁逻辑不变。
- 文档记录该摘要的只读边界和未模拟完整路径的风险。
