# v2.4 RegionCommand / 军师指令诊断三国化提示词

## 目标

继续推进三国迁移可见文本收口，把旧 region 命令兼容层、region-to-hex adapter 和军师指令 JSON 解码诊断中的英文 / raw id 文案改为中文三国语义，避免 AI 面板、错误记录或交互日志继续暴露 `RegionMove`、`Division`、`FrontZone`、`Theater directive` 等内部表达。

## 范围

- `WWIIHexV0/Commands/RegionCommand.swift`
- `WWIIHexV0/Commands/CommandIntentAdapter.swift`
- `WWIIHexV0/Commands/WarDirective.swift`
- `WWIIHexV0/Turn/TurnManager.swift`
- 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md`、`update_log.md`

## 非目标

- 不改 `RegionCommand`、`Command`、`ZoneDirective`、`TheaterDirectiveEnvelope`、`TheaterDirective` 的 Codable schema。
- 不改任何 rawValue、JSON key、id、命令执行、校验、移动、攻击、道路、粮道、外交、生产或动态战区规则。
- 不引入 state-dependent region command 展示名解析；无 `GameState` 的 `displayName` 只显示安全类别，不拼 raw id。
- 不跑本机 Xcode build/test、模拟器、Probe、Smoke、Stage Regression、Dynamic Theater Regression、Full 或性能测试。

## 实现步骤

1. 将 `RegionCommand.displayName` 改为中文类别：郡县进军、郡县交战、郡县固守、郡县补给。
2. 将 `CommandIntentAdapterError.errorDescription` 改为中文，保留必要 hex 坐标，但不拼 `RegionId` 或军队 id。
3. 将 `DirectiveTarget` 解码失败说明和 `TheaterDirectiveDecoderError.errorDescription` 改为中文军师指令语义；缺失防区/方面/郡县时使用类型名，不展示 raw id。
4. 将 `TurnManager` 中外交和太守命令成功诊断从 `command.displayName` 改为 `CommandResult.message`，复用 `RuleEngine` 已生成的 state-aware 中文摘要。
5. 更新阶段索引、核心流程文档和更新日志，明确本轮只改可见诊断，不改规则。
