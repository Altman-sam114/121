# v2.4 武将姓名快照兼容层

本文记录“武将交战日志不能只显示 id”的小步实现提示词与交付边界。执行前仍必须阅读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和总提示词 `md/prompt/v2.0-三国迁移/codex-v2.0-三国aiagent迁移总提示词.md`。

## 目标

- 在 `GeneralAssignment` 中新增可选 `generalDisplayName` 快照，保持 Codable 向后兼容。
- `GeneralData.defaultAssignment` 和 `GeneralDispatcher` 刷新 assignment 时写入 `localizedName`。
- `GeneralCombatInfluenceSummary` 生成交战日志时优先显示 `generalDisplayName`，缺失时回退到武将 id。
- 保持 `GeneralInfluence` 仍只读 `GameState`，不依赖 UI 或外部 registry。

## 非目标

- 不重构战报面板。
- 不新增头像、美术资源、完整技能树、单挑、士气或疲劳。
- 不改变 `CombatRules` 伤害公式、撤退或反击规则。
- 不改变 JSON schema；旧存档缺少 `generalDisplayName` 时必须可解码。
- 不跑本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构边界

```text
GeneralData.localizedName
  -> GeneralAssignment.generalDisplayName
  -> GeneralInfluence.combatSummary
  -> GeneralCombatInfluenceSummary.logFragment
  -> CommandExecutor.combatLog
```

- `generalDisplayName` 是审计快照，不是规则权威。
- 武将 id 仍是 assignment 的 identity；显示名缺失时按 id fallback。
- UI 可继续从 `GeneralRegistry` 读取完整武将档案；规则层不引入 registry 依赖。
