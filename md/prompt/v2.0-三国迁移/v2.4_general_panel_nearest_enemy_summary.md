# v2.4 武将面板道路与交战近敌摘要提示词

## 1. 目标

继续强化“武将、道路、交战”的可读性：在 `GeneralCommandPanelView` 的道路与交战小节中，让选中武将防区不仅显示道路机动和接敌攻防修正，还能显示当前最近敌军对象、距离以及对应的麾下军队，帮助玩家判断这个武将防区正面对谁、离接战还有多远。

## 2. 范围

- 仅修改只读 UI 派生与展示：
  - `AppContainer.selectedGeneralInfluenceNotes`
  - `GeneralCommandPanelView`
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改 `PlayerPlannedOperation` Codable schema。

## 3. 实现要求

- `AppContainer.selectedGeneralInfluenceNotes` 在计算当前防区麾下军队与敌对军队时，按 `Faction.isHostile(to:)` 过滤敌军。
- 最近敌军摘要应包含：
  - 最近敌军的三国化军队名。
  - 当前 hex 距离。
  - 与该敌军距离最近的麾下军队名。
- 当前已接敌时，把近敌摘要追加到接敌攻防修正行。
- 当前未接敌时，把近敌摘要追加到“未接敌”行，并继续提示进入射程后才计算武将攻防。
- `GeneralCommandPanelView` 可以适度放宽道路与交战摘要行数，降低近敌对象文本被截断的概率。

## 4. 禁止项

- 不改 `MovementRules`、`CombatRules`、`GeneralInfluence`、`WarCommandExecutor`、`CommandValidator`、`RuleEngine`。
- 不改变真实移动成本、道路加成、敌控区、攻击、反击、撤退、补给、围城或粮道规则。
- 不把最近敌军摘要写成路径模拟、胜率预测、敌方回合反制或移动后同回合攻击判断。
- 不让 SwiftUI View 直接推导敌军、道路或交战规则；只读派生放在 `AppContainer`。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 武将面板“道路与交战”摘要能显示当前最近敌军对象、距离和对应麾下军队。
- 旧的道路机动、接敌攻防修正摘要仍存在。
- 规则执行、状态 schema、真实道路/交战/补给逻辑不变。
- 文档明确该摘要是只读当前静态距离，不是路径安全或胜率模拟。
