# v2.4 命令结果展示名三国化兼容层

## 目标

继续推进三国迁移中“武将、道路、交战”相关的可见审计闭环，把 `RuleEngine` 命令结果、玩家交互日志、AI 面板命令结果标题和核心移动/交战日志里的 raw id / 旧单位名收敛为三国语义展示名。

本轮聚焦命令结果展示，不改变命令 schema。

## 范围

- `RuleEngine` 的成功 `CommandResult.message`。
- `AppContainer.submit(_:)` 的玩家交互日志。
- `CommandResultSummary` 给 AI 面板使用的命令结果标题展示。
- `CommandExecutor` 的移动、攻击、反击、外交 fallback 和动态方面推进日志。
- `WarCommandExecutor` 的动态方面推进日志位置展示。
- 同步更新 README、核心流程文档、流程图、prompt 索引和 update log。

## 非目标

- 不修改 `Command.displayName` 本体，避免影响 Codable、历史测试和旧 AI 判断。
- 不改 `Command`、`CommandResult`、`CommandResultSummary` 的 Codable 字段。
- 不改变移动、攻击、占领、修路、外交、生产、胜负、补给、围城、撤退、伤害或 AI 选兵规则。
- 不处理 `RegionCommand`、`CommandIntentAdapter`、`WarDirective` 解码诊断、`SupplyRules` 全量日志或 `VictoryRules` fallback；这些另列后续切片。

## 架构依据

- 玩家、AI、武将面板和后续聊天命令必须经 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine`。
- `Command.displayName` 当前仍保留兼容身份；玩家可见结果应由 state-aware 展示层或结果消息生成。
- `Division.thematicDisplayName` 是当前军队三国展示名入口。
- Region / Country 展示应优先读 `RegionNode.name` / `CountryProfile.name`，空名或等于 raw id 时使用中文占位或势力展示名。

## 实现要求

1. `RuleEngine` 成功消息使用 state-aware helper：
   - 移动：军队展示名 + 坐标 + 郡县名。
   - 交战：攻守双方军队展示名。
   - 固守/撤退/补给：军队展示名。
   - 修路：郡县展示名。
   - 外交：国家/势力展示名。
2. `AppContainer.submit(_:)` 不再把 `command.displayName` 拼进玩家交互日志。
3. `AgentPanelView` 的命令结果标题使用 `CommandResultSummary.commandDisplayNameForDisplay`，只显示“进军命令 / 交战命令 / 修缮道路命令”等类别。
4. `CommandExecutor` 移动、攻击、反击日志使用 `Division.thematicDisplayName`。
5. 动态方面推进日志保留 hex 坐标，但追加郡县展示名。

## 验收标准

- `RuleEngine` 成功结果不再直接拼 `command.displayName`。
- 玩家交互日志不再重复展示 raw id 版命令名。
- AI 面板命令结果标题不再直接展示 `commandDisplayName` 的 raw id 文案。
- 移动、攻击、反击日志使用三国军队展示名。
- 动态方面推进日志带郡县名，坐标只作辅助。

## 遗留风险

- `Command.displayName` 本体仍保留 raw id 兼容输出，供旧测试、旧 AI 判断和调试字段使用。
- `RegionCommand`、`CommandIntentAdapter`、`WarDirective` 解码诊断、`SupplyRules` 和部分经济/胜负 fallback 仍有后续可见文案收敛空间。
- 本轮不跑本机重测试，运行时 UI 呈现等待云端 CI、Agent C artifact 复判或人工授权检查。
