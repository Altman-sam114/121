# v2.4 武将道路机动日志审计兼容层

本文记录“武将道路机动已经影响规则，但移动日志不可见”的小步实现提示词与交付边界。执行前仍必须阅读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和总提示词 `md/prompt/v2.0-三国迁移/codex-v2.0-三国aiagent迁移总提示词.md`。

## 目标

- 沿续“注重武将、做好道路、交战”的 v2.4 方向，让武将道路机动不只影响移动上限，也能被事件日志审计。
- 新增只读 `GeneralMovementInfluenceSummary`，复用 `GeneralInfluence.roadMobilityBonus` 的同一计算来源。
- `MovementRules` 暴露道路机动摘要给执行层。
- `CommandExecutor` 在移动日志中只在道路加成非 0 时追加武将姓名、加成和有效移动上限。

## 非目标

- 不新增完整修路、借道、屯田或道路建设制度。
- 不改变移动路径搜索、敌控区、占领、动态战区推进或补给规则。
- 不让武将层、UI 或日志直接修改 `GameState`。
- 不跑本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构边界

```text
GeneralAssignment(generalDisplayName / skills / loyalty / satisfaction)
  -> GeneralInfluence.roadMobilityBonus
  -> GeneralMovementInfluenceSummary
  -> MovementRules.generalInfluenceSummary
  -> CommandExecutor.movementLog
```

- 真实移动上限仍来自 `MovementRules.effectiveMovementLimit`。
- 移动合法性仍由 `CommandValidator.validateMove` 判断。
- 移动执行仍由 `Command.move -> RuleEngine -> CommandExecutor` 写 `Division.coord`。
- 日志摘要只解释已有加成，不引入第二套移动规则。

## 实现步骤

1. 在 `GeneralInfluence.swift` 新增 `GeneralMovementInfluenceSummary`，字段包含武将 id、武将显示名、基础移动、有效移动和道路加成。
2. 将 `roadMobilityBonus` 的核心逻辑收束到同一个私有 helper，供移动上限和摘要复用。
3. 在 `MovementRules.swift` 暴露 `generalInfluenceSummary(for:in:)`。
4. 在 `CommandExecutor.executeMove` 执行移动前读取摘要，移动后通过 `movementLog` 追加日志片段。
5. 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 验收标准

- 有道路机动加成时，移动事件日志能显示对应武将和移动上限变化。
- 无加成时移动日志不新增噪声。
- 道路机动摘要和真实移动上限使用同一套 `GeneralInfluence` 计算。
- 玩家与 AI 移动仍走 `Command / RuleEngine / CommandExecutor`，没有绕过规则系统。

## 风险

- 本轮不做运行时 UI 烟测，日志展示是否符合玩家预期仍等待云端 CI、后续 Agent C artifact 复判或人工授权运行检查。
- 旧存档若武将 assignment 没有 `generalDisplayName`，移动日志仍会回退到武将 id。
- 当前道路加成仍是 v2.4 最小规则，不等于完整道路建设、粮道可视化或借道制度。
