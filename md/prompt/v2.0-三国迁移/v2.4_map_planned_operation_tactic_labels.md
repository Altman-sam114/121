# v2.4 地图计划军令武将战术标识兼容层

## 目标

让玩家通过武将面板提交的宏观军令，不只在面板列表里显示最终 tactic，也能在 SpriteKit 地图计划层显示为带武将与短战术信息的可读态势标识，强化“武将下令、道路推进、交战目标”之间的地图反馈。

## 范围

- 修改 `WWIIHexV0/SpriteKit/BoardScene.swift` 的 `drawPlannedOperations` 只读渲染。
- 复用既有 `PlayerCommandState.plannedOperations`、`PlayerPlannedOperation.tactic`、`FrontZone.generalAssignment.generalDisplayName`、`DirectiveType` 与 `TacticName`。
- 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 禁止项

- 不改 `PlayerCommandState` schema，不新增存档字段。
- 不改 `GeneralAgent` 战术塑形算法。
- 不改 `WarCommandExecutor`、`CommandValidator`、`RuleEngine` 或真实移动/攻击/道路/补给/交战结算。
- 不让 `BoardScene` 直接修改 `GameState`、hex controller、单位坐标、动态方面、防区、资源、道路或生产队列。
- 不新增本机 Xcode build/test、模拟器、Probe、Smoke、Stage Regression、Dynamic Theater Regression、Full 或性能测试。

## 当前架构依据

- 玩家宏观军令由 `GeneralCommandPanelView -> AppContainer.submitPlayerDirective -> GeneralAgent.plan -> WarCommandExecutor -> RuleEngine` 执行。
- 执行后 `AppContainer` 会记录 `WarDirectiveRecord` 与 `PlayerPlannedOperation`，其中 `PlayerPlannedOperation.tactic` 已保存塑形后的最终 tactic。
- `BoardScene` 已读取当前回合、当前查看势力的 `plannedOperations`，攻击命令画源 region 到目标 region 的箭头，防御命令画源点圆环。
- `BoardScene` 是只读地图渲染层，适合补充端点、箭头标签和战术短名，但不应承载规则判断。

## 实现步骤

1. 在攻击计划箭头上补充源点和目标点标记。
2. 在攻击计划箭头中点附近显示短标签，内容优先使用防区武将显示名、攻/守字样和短战术名。
3. 在防御计划圆环上方显示同样风格的固守标签。
4. 为过长武将名和过长 tactic 做短名收束，避免地图标签过宽。
5. 保持计划图层在 `frontLine` 图层隐藏、在其他图层只读显示的既有行为。

## 验收标准

- 玩家当前回合的进攻计划在地图上显示源点、目标点、进攻箭头和武将/短战术标签。
- 玩家当前回合的防御计划在地图上显示固守圆环和武将/短战术标签。
- 标签来自既有 `PlayerPlannedOperation` 与 `FrontZone.generalAssignment`，无武将名时安全退回攻守/战术标签。
- 真实命令执行、道路、交战、补给、战略同步、微操锁和面板计划军令列表行为不变。

## 风险提示

- 本轮不做运行时 UI 烟测；多个计划箭头相互靠近、长武将名、长战术名和小屏缩放下的实际重叠情况仍需云端 CI、后续 Agent C artifact 复判或人工授权运行检查。
- 地图标签只展示已记录的计划态势，不模拟完整路径、真实胜率、移动后同回合攻击、敌方下回合反制或外交借道。
