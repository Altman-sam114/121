# v2.4 武将军衔与头像占位显示中文化

## 目标

- 继续推进武将 UI 三国化，避免 `GeneralCommandPanelView` 和 `GeneralProfileView` 在旧 fallback 将军数据下直接显示 `Field Marshal`、`Generaloberst`、`Brigadier General` 等二战/英文军衔。
- 将武将头像占位的 VoiceOver 文案从英文或“占位”调试语义改为玩家可理解的中文“头像”。

## 范围

- `GeneralData` 增加只读展示属性，把旧英文 rank 映射为三国语义显示名。
- `GeneralCommandPanelView` 和 `GeneralProfileView` 只读使用 `rankDisplayName`。
- 同步更新 README、核心流程文档、prompt 索引和 `update_log.md`。

## 禁止项

- 不修改 `generals.json` / `sanguo_generals.json` raw data。
- 不改变 `GeneralData.rank` Codable 字段、JSON schema、武将分配、战术塑形或规则计算。
- 不重构武将面板布局，不引入新资源或头像系统。
- 不运行 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构依据

- 旧 `generals.json` 仍作为 fallback 和回归参考保留，内部 rank 可以继续是英文 raw data。
- 玩家可见层应通过展示属性消化旧数据，不影响 Codable / 调试 / 历史回归。
- 武将面板和档案弹窗都读取 `GeneralData`，因此展示映射放在 `GeneralData` 的只读 computed property 中，避免两个 UI 文件各自维护映射。

## 实现步骤

1. 在 `GeneralData` 增加 `rankDisplayName`。
2. 映射旧英文 rank：
   - `Field Marshal` -> `大将军`
   - `Generaloberst` -> `上将军`
   - `Brigadier General` -> `偏将军`
   - `General` -> `将军`
   - 空 rank -> `武将`
   - 其他 rank 原样显示，保留三国 JSON 已有中文官职。
3. `GeneralCommandPanelView` 和 `GeneralProfileView` 改用 `rankDisplayName`。
4. 将头像占位 accessibility label 改为 `\(general.localizedName) 头像`。
5. 更新阶段 prompt、prompt 索引、README、核心流程文档和更新日志。

## 轻量检查

- 针对改动 Swift 文件做可行的轻量 parse；若 SwiftUI 或跨文件依赖导致单文件 parse 不适用，记录未跑原因，交给云端 workflow。
- `rg -n "general\\.rank([^A-Za-z_]|$)|portrait placeholder|头像占位" WWIIHexV0/UI WWIIHexV0/Agents`
- `rg -n "Field Marshal|Generaloberst|Brigadier General" WWIIHexV0/UI WWIIHexV0/Agents`
- 本轮改动文件尾随空白扫描。
- 本轮改动文件冲突标记扫描。
- `md/prompt/v2.0-三国迁移` 目录与 `md/prompt/README.md` 索引差集检查。
- `git diff --check`

## 验收标准

- 武将面板和武将档案不再直接读取 `general.rank`。
- 旧英文 fallback rank 在玩家可见 UI 中显示为三国语义军衔。
- 头像 VoiceOver 文案不再包含英文 `portrait placeholder`。
- JSON raw data、Codable 字段和规则行为不变。
