# v2.4 武将面板可见敌军接敌摘要兼容层提示词

## 1. 目标

继续强化“武将、道路、交战”的玩家决策闭环：武将面板“道路与交战”的最近敌军、当前接敌配对和攻防修正摘要只能基于玩家视角可见 hostile 军队生成，避免不可见敌军通过武将面板名称、距离或接敌修正泄漏。

## 2. 范围

- 仅修改只读 UI 派生：
  - `WWIIHexV0/App/AppContainer.swift`
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改 `GeneralCommandPanelView` API，不修改真实攻击、移动、道路、粮道、补给、外交、AI 指令执行或 `RuleEngine`。

## 3. 实现要求

- `AppContainer.selectedGeneralInfluenceNotes` 收集敌军集合时，继续使用 `Faction.isHostile(to:)` 和未毁灭条件，并追加 `mapDisplayAdapter.isDivisionVisible(target, viewerFaction: playerFaction)`。
- 可见性必须使用玩家视角 `playerFaction`；不要用防区 faction 或敌军 faction 推导另一套视野。
- `nearestEnemyText`、`engagementPairingText`、攻防 bonus 范围只消费过滤后的可见 hostile 集合。
- 观察模式继续由 `mapDisplayAdapter` 的 revealAll 行为兼容全图查看。

## 4. 禁止项

- 不改 `MovementRules`、`CombatRules`、`GeneralInfluence`、`SupplyRules`、`CommandValidator`、`WarCommandExecutor`、`RuleEngine` 或 `CommandExecutor`。
- 不新增持久字段，不修改 Codable schema，不改 Xcode project。
- 不让不可见敌军进入武将面板最近敌军、接敌配对、攻防修正范围或麾下军队接敌摘要。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 武将面板“道路与交战”的近敌对象、距离、当前接敌配对和攻防修正范围只来自玩家视角可见 hostile 军队。
- 无可见 hostile 时，不显示隐藏敌军名、距离或接敌配对。
- 观察模式仍可通过 `mapDisplayAdapter` revealAll 看到全图摘要。
- 真实移动、攻击、伤害、反击、道路、粮道、补给、外交和命令执行规则不变。
