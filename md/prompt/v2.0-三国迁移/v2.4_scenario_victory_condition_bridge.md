# v2.4 官渡剧本胜负条件兼容层提示词

## 目标

继续推进 v2.0 三国迁移，把默认官渡剧本 JSON 中已经声明的 `victoryConditions` 接入运行时胜负判断，避免玩家 HUD 只能依赖旧阿登 `Bastogne / St. Vith` 胜负规则。

## 范围

- `DataLoader` 从 `ScenarioDefinition.victoryConditions` 提取 active 的 `controlObjective` 条件，写入 `VictoryState` 的轻量剧本条件快照。
- `VictoryRules` 在旧阿登 fallback 规则前，优先用 objective id 找到对应 hex，并按 `HexTile.controller` 判断该势力是否控制目标。
- HUD 胜负文本显示势力胜利和中文胜利原因。
- `MapState` 可按 objective id 查询目标点，避免用可见名称匹配官渡目标。
- 同步 README、`md/flow/*`、`update_log.md` 和本提示词索引。

## 非目标

- 不实现完整多目标计分、限回合目标、歼灭模板目标、任务面板或胜利条件编辑器 UI。
- 不重命名旧 `VictoryReason` rawValue、`eliminatedGermanDivisions`、`germanBastogneHeldSinceTurn` 等兼容字段。
- 不删除旧阿登胜负规则；它仍作为 fallback 和历史测试兼容层。
- 不改变占领、道路、粮道、交战、外交、命令管线或 AI 决策。

## 架构依据

- Hex 仍是战术权威，胜负目标控制权必须读取 objective 所在 `HexTile.controller`。
- Region / Theater / FrontZone 都是派生或聚合层，不作为本轮胜负控制权权威。
- 玩家、AI 和聊天命令仍必须经 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine` 修改控制权；胜负规则只读状态并写 `VictoryState`。
- 官渡 JSON 仍用 `.germany/.allies` 作为曹操/袁绍兼容 rawValue；显示层使用 `Faction.displayName`。

## 验收标准

- 官渡默认剧本加载后，`VictoryState.scenarioConditions` 包含 active `controlObjective` 条件。
- 袁绍控制许昌或曹操控制邺城时，`VictoryRules` 可设置对应 winner，并记录中文原因。
- HUD 胜负栏显示“势力胜利：原因”。
- 旧阿登规则、旧 Codable 字段和历史测试期待的 enum rawValue 保持兼容。
