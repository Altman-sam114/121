# v2.4 太守生产命令执行兼容层

本文记录“君主、外交、太守、军师、武将 AI Agent 分层”中的太守建议生产执行小步实现提示词与交付边界。执行前仍必须阅读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和总提示词 `md/prompt/v2.0-三国迁移/codex-v2.0-三国aiagent迁移总提示词.md`。

## 目标

- 复用既有 `Command.queueProduction(kind:)`，让太守推荐生产也进入统一命令管线。
- `TurnManager.applyGovernorPlanning` 将 `GovernorDecisionRecord.recommendedProductionKind` 转换为 `Command.queueProduction`，经 `commandHandler.execute` 执行。
- `CommandValidator.validateProduction` 继续负责阶段和资源校验，`CommandExecutor.executeQueueProduction` 继续负责扣资源和追加 `ProductionOrder`。
- `AgentDecisionRecord.commandResults` 显示太守推荐生产是否成功或被规则层拒绝。
- 文档明确：`GovernorAgent` 仍只产出 Codable 建议和审计记录，生产队列变化只允许通过 `Command.queueProduction -> RuleEngine`。

## 非目标

- 不新增完整内政 directive schema。
- 不实现真实修路、屯田、治安、民心或郡县状态变化。
- 不直接修改 `EconomyState`、`ProductionOrder`、道路或 region 字段。
- 不改变玩家手动生产按钮的既有命令入口。
- 不跑本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构边界

```text
RulerAgent.adjust
  -> DiplomatAgent.plan
  -> Command.proposeDiplomacy
  -> GovernorAgent.plan
  -> GovernorDecisionRecord
  -> Command.queueProduction
  -> CommandValidator
  -> CommandExecutor / EconomyRules.queueProduction
  -> StrategistAgent.plan
  -> GeneralAgent.plan
  -> WarCommandExecutor
  -> RuleEngine
```

- `GovernorAgent` 不持有写权限；只读 `GameState`，输出 Codable `GovernorDecisionRecord` 和调整后的 `DirectiveEnvelope`。
- `TurnManager` 只在记录有 `recommendedProductionKind` 时生成生产命令；没有建议时仅写诊断。
- 生产预命令结果写入 `AgentDecisionRecord.commandResults`，不混入单条 `WarDirectiveRecord`。
- 当前内政执行只覆盖已有生产队列；其它内政重点仍是审计和后续版本依据。
