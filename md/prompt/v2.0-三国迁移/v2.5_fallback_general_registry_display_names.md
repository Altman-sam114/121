# v2.5 fallback 武将 registry 展示名三国化提示词

## 目标

继续推进 v2.5 武将数据和玩家可见文案收口，把旧 fallback `WWIIHexV0/Data/generals.json` 中的二战将领姓名、头像名、军衔和传记替换为三国兼容武将展示资料，避免缺少 `sanguo_generals.json` 时武将面板、军队详情或军机审计透出 Montgomery、Weygand、de Gaulle、Guderian、Leeb、Bock 等旧题材名称。

## 范围

- 只允许修改 `WWIIHexV0/Data/generals.json` 的展示字段：`name`、`localizedName`、`rank`、`portrait`、`biography`。
- 建议映射：
  - `general_montgomery` -> 袁绍
  - `general_weygand` -> 沮授
  - `general_de_gaulle` -> 张郃
  - `general_guderian` -> 张辽
  - `general_leeb` -> 荀攸
  - `general_bock` -> 曹操
- 更新 `md/prompt/README.md` 和 `update_log.md`。

## 非目标

- 不修改 `id`、`faction`、`commandStyle`、`skills`、`preferredTheaterIds`、`preferredRegionIds`、忠诚/满意数值或 JSON schema。
- 不修改默认 `sanguo_generals.json`。
- 不修改 `ardennes_v02_regions.json` 中的 `assignedGeneralId` 兼容引用。
- 不修改 `GeneralRegistry`、`GeneralDispatcher`、`GeneralAgent`、规则、AI、MapEditor 或命令管线。
- 不清理历史 prompt、旧记录和测试样例中的旧二战名称。

## 架构依据

- `DataLoader.loadGeneralRegistry()` 优先加载 `sanguo_generals.json`，缺文件时才 fallback 到旧 `generals.json`。
- `GeneralData.defaultAssignment(...)` 和 UI 面板读取 `localizedName` 作为玩家可见武将名；`name` 仍可作为兼容配置名使用。
- `GeneralData.rankDisplayName` 对旧英文军衔有兼容映射；本轮直接写入中文军衔，降低 fallback 可见残留。
- `GeneralDispatcher` 通过 `id` 和 faction / preferred region 进行兼容分配，本轮不改这些字段。
