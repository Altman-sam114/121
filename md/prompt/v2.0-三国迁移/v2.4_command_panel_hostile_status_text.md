# v2.4 命令面板敌对状态文案提示词

## 1. 目标

继续收口三国多势力敌我判断：在玩家命令面板中，选择非玩家军队时不要一律显示“敌军”，而是按 `Faction.isHostile(to:)` 区分敌对军队和非敌对军队，避免中立、友好或后续借道势力被误标为敌军。

## 2. 范围

- 仅修改只读 UI 状态文案：
  - `CommandPanelView.statusText`
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改按钮可用性、玩家下令权限或命令执行规则。

## 3. 实现要求

- 玩家选择敌对军队时继续显示不能下令的敌军提示。
- 玩家选择非己方但非敌对军队时显示“非敌对军队，只能查看”一类提示。
- `canCommandSelectedUnit` 仍只允许玩家当前回合指挥玩家势力、未行动军队。
- 不改变 `CommandPanelView` 的按钮、动作或命令派发。

## 4. 禁止项

- 不改 `Faction.isHostile(to:)` 当前最小实现。
- 不改 `CommandValidator`、`RuleEngine`、`CommandExecutor`、移动、攻击、补给或外交关系规则。
- 不让非敌对非己方军队变成可指挥对象。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 命令面板选择 hostile 军队时仍提示敌军不能下令。
- 命令面板选择非 hostile 的非玩家军队时不再误称敌军。
- 玩家下令权限、按钮可用性和真实规则执行不变。
