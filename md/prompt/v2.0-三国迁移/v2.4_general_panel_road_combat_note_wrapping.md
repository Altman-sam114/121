# v2.4 武将面板道路与交战摘要换行兼容层提示词

## 1. 目标

继续强化“武将、道路、交战”的可读链路：`AppContainer.selectedGeneralInfluenceNotes` 已经生成道路机动、官道受益军队、接敌攻防、当前接敌配对、最近敌军对象和麾下备战摘要，本轮只调整 `GeneralCommandPanelView` 的“道路与交战”展示约束，降低长摘要在窄屏、长军队名和 Dynamic Type 下被固定 3 行截断的概率。

## 2. 范围

- 仅修改 `WWIIHexV0/UI/GeneralCommandPanelView.swift` 的“道路与交战”摘要行展示。
- 可同步文档：
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改 `AppContainer.selectedGeneralInfluenceNotes`。
- 不修改 `GeneralInfluence`、`MovementRules`、`CombatRules`、`WarCommandExecutor`、`CommandValidator`、`RuleEngine` 或真实执行规则。

## 3. 实现要求

- 保留每条影响摘要的 SF Symbol 图标和原始文本。
- 取消固定 3 行截断，让文本按内容纵向展开。
- 使用 SwiftUI 只读布局修饰，不在 View 中推导道路、敌军、武将、伤害或风险。
- `RootGameView` 外层已有 `ScrollView`，长内容通过现有滚动承接，不新增滚动容器。

## 4. 禁止项

- 不改变移动范围、道路加成、敌控区、攻击、反击、撤退、补给、围城、粮道或武将加成规则。
- 不新增路径安全、攻击合法性、隐藏敌军推断、胜率或敌方回合模拟。
- 不改变 `GeneralCommandPanelView` public 入参。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 武将面板“道路与交战”摘要不再固定截断为 3 行，长摘要可按内容换行展开。
- App 层摘要、真实道路/交战/补给、命令执行和 Codable schema 保持不变。
- 文档明确本轮只是只读 UI 可读性改进，运行时 UI 高度与滚动体验仍需云端 CI、后续 Agent C artifact 复判或人工授权运行检查。
