# v2.4 外交敌对 ZOC / 粮道 / 安全补员兼容层提示词

## 1. 目标

继续推进三国多势力、武将、道路和交战迁移：攻击合法性和 UI 预览已接入 `DiplomacyState` hostile / atWar 口径，但道路敌控区、粮道单位阻断、围城邻接和安全补员邻接仍按静态 `Faction.isHostile(to:)` 判断单位敌对。本轮只迁移这些“单位敌对关系”判断，让中立、停战、同盟或共同作战单位不再误构成 ZOC、切粮、围城或阻止安全补员。

## 2. 范围

- `WWIIHexV0/Rules/MovementRules.swift`
  - `isEnemyZoneOfControl` 的敌对单位来源。
- `WWIIHexV0/Rules/SupplyRules.swift`
  - 粮道路径上的敌对单位阻断。
  - 围城邻接敌对单位判断。
  - 通过 `MovementRules.isEnemyZoneOfControl` 间接受影响的撤退安全格和粮道 ZOC。
- `WWIIHexV0/Rules/EconomyRules.swift`
  - 安全补员的相邻敌对单位判断。

## 3. 实现要求

- 使用 `state.diplomacyState.isHostile(between:and:)`，保留缺建档 fallback 到 `Faction.isHostile(to:)` 的兼容策略。
- 只改单位敌对关系，不改领土控制权和堆叠规则。
- 保留玩家视角、移动、补给、围城和补员的既有结构，只替换敌对来源。

## 4. 禁止项

- 不修改 `Faction.isHostile(to:)` 语义。
- 不允许同盟/中立单位同格或穿格；`MovementRules` 的非同 faction 堆叠阻挡仍保留。
- 不把 `HexTile.controller` / `RegionNode.controller` 的敌控或非己控判断改成外交关系。
- 不实现完整借道、同盟通行、共同作战堆叠、补给共享或外交宣战制度。
- 不修改部署层前线接触、动态战区推进、占领、生产、道路修缮或攻击伤害。
- 不新增持久字段，不修改 JSON schema，不改 Xcode project。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 外交非敌对单位不再构成移动 ZOC。
- 外交非敌对单位不再直接阻断粮道或触发围城邻接。
- 外交非敌对单位不再阻止安全补员邻接判定。
- 领土 controller、堆叠阻挡、借道、补给共享、部署接触和动态战区推进不变。
- 本机只跑轻量检查，重验证交给 GitHub Actions。
