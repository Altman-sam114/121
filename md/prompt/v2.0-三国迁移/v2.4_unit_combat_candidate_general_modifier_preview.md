# v2.4 军队接战候选武将修正预判兼容层

## 目标

在军队详情的“接战预判”候选目标行中显示每个候选目标的武将攻防修正，让玩家比较目标时能看出本次接战的预计伤害是否受到我方或敌方武将影响。

## 范围

- 修改 `WWIIHexV0/App/AppContainer.swift` 的 `combatTargetPreviewLine` 派生文本。
- 复用每个候选 `CombatTargetPreview.influence` 中的 `attackBonus` 与 `defenseBonus`。
- 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 禁止项

- 不改 `GeneralInfluence` 的武将查找、道路加成、攻击加成或防御加成。
- 不改 `CombatRules` 伤害、反击、撤退、死守、围城或兵种克制公式。
- 不改 `CommandExecutor`、`CommandValidator`、`WarCommandExecutor`、`RuleEngine` 或真实命令执行结果。
- 不新增武将任免、头像、士气、单挑、详细技能展开或 JSON schema。

## 当前架构依据

- `selectedUnitCombatPreviewNotes` 已为每个候选目标计算 `GeneralCombatInfluenceSummary`。
- 首选目标已有非零“武将影响”详情；候选目标行需要更紧凑，只显示非零攻防修正。
- 候选目标排序仍按预计伤害、反击风险、距离和名称排序，本轮只补充解释文本。

## 实现步骤

1. 在 `combatTargetPreviewLine` 中读取当前候选的 `GeneralCombatInfluenceSummary`。
2. 若 `attackBonus` 或 `defenseBonus` 非 0，候选行追加 `武将修正 攻+... 防+...`。
3. 若两项修正均为 0，不追加额外文本，避免候选行过长。
4. 保留既有首选理由、接战官道、交战武将总览、候选敌将、强度结算、风险、态势、武将影响和交战审计。

## 验收标准

- 选中军队且射程内有敌军时，最多三条候选目标行能在存在非零武将攻防修正时显示 `武将修正 ...`。
- 首选目标仍显示已有“交战武将”总览和非零“武将影响”摘要。
- 真实移动、攻击、日志结算、命令校验和动态战略同步行为不变。

## 风险提示

- 本轮没有运行 UI 烟测；候选行新增武将修正文案在窄屏、长武将名和 Dynamic Type 下的换行仍需云端 CI、后续 Agent C artifact 复判或人工授权运行检查。
- 摘要只展示候选目标当前攻防修正数值，不解释完整技能、忠诚、满意、单挑或士气变化。
