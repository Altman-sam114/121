# v2.4 非敌对借道不自动占领兼容层

## 目标

让玩家、AI 和武将宏观军令在移动到外交非敌对控制格时不自动翻转 hex controller，也不把该移动解释为突破推进动态方面；移动到 hostile / atWar 控制格或无控制者可占领格时，保持既有占领与后续战略同步。

## 范围

- `WWIIHexV0/Rules/OccupationRules.swift`
- `WWIIHexV0/Commands/WarCommandExecutor.swift`
- 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 和 `md/prompt/README.md`

## 禁止项

- 不实现完整借道条约、通行权费用、共同堆叠、补给共享、宣战流程或外交惩罚。
- 不改变 `Command` / `ZoneDirective` schema。
- 不改变 `CommandValidator` 的移动合法性、`MovementRules` 路径搜索、真实交战伤害、补给规则或 Region 聚合算法。
- 不把非敌对单位视为合法攻击目标。

## 架构依据

- Hex controller 是真实占领权威，Region controller 只从 hex 聚合。
- 玩家移动走 `CommandExecutor.executeMove`，AI / 武将宏观军令走 `WarCommandExecutor.run -> CommandHandler -> CommandExecutor`。
- 动态方面推进必须来自真实占领或敌对突破；非敌对经过道路、郡县或控制格不能自动推进 `hexToTheater` / `hexToFrontZone`。
- 外交 hostile / atWar 口径来自 `DiplomacyState.isHostile(between:and:)`，缺建档时保留 faction fallback。

## 实现步骤

1. 在 `OccupationRules.canOccupy` 中保留 capturable、非己方 controller 和无其他单位条件。
2. 当目标 hex 已有 controller 时，只允许占领外交 hostile / atWar 控制格；allied、coBelligerent、neutral 控制格不自动占领。
3. 保留无 controller 可占领格的既有占领行为。
4. 在 `WarCommandExecutor` 中记录命令执行前该移动是否可占领，并只用真实可占领/占领 region 触发动态方面推进，避免成功移动到非敌对控制格后仍因 affected region 推进动态归属。
5. 同步核心文档，明确本轮只做“非敌对不自动占领/不突破”的最小兼容层。

## 验收标准

- 移动到 hostile / atWar 控制格仍可占领，并继续触发占领后的战略同步。
- 移动到 allied / coBelligerent / neutral 控制格不改写 tile controller。
- AI / 武将移动到非敌对控制格不因 affected region 自动推进 `hexToTheater` 或 `hexToFrontZone`。
- 本轮不改变路径搜索、堆叠、补给、外交状态或攻击合法性。

## 遗留风险

- 当前仍没有完整借道制度；非敌对通行是否允许、是否消耗外交资源、是否影响补给和治安，后续另拆。
- 本轮不跑本机 Xcode / XCTest / 模拟器重测试；运行时行为等待 `origin/main` 云端 CI 和后续 Agent C artifact 复判。
