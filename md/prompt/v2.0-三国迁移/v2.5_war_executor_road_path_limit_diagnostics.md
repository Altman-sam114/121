# v2.5 宏观道路路径超限审计提示词

## 目标

继续推进“武将、道路、交战”体验，把 `WarCommandExecutor` 宏观军令道路审计里的“目标不可达”细化为本回合机动不足的“路径超限”和真实无路的“无通路”，让 AI 防区指令和玩家武将宏观军令能更清楚解释道路、地形、敌控区和占位对落点的影响。

## 范围

- 修改 `WWIIHexV0/Commands/WarCommandExecutor.swift`。
- 更新 `md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 非目标

- 不改变 `MovementRules` 路径搜索、移动成本、敌控区、占位阻挡、道路加成或移动合法性。
- 不改变 `WarCommandExecutor` 选兵、落点排序、fallback、攻击筛选、命令执行顺序或占领推进。
- 不改 `WarDirectiveRecord` Codable schema。
- 不改 `AgentPanelView`、`TurnManager`、`AppContainer` 或 SpriteKit 绘制。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试 / app 启动 / 全量构建。

## 架构依据

- `WarCommandExecutor.roadSelectionDiagnostic` 已在执行前生成道路审计，并通过 `WarCommandExecutionResult.diagnostics` 进入 AI / 玩家 `WarDirectiveRecord.diagnostics`。
- `MovementRules.shortestPath(for:to:in:)` 使用有效机动上限判断本回合可达。
- `MovementRules.shortestPathIgnoringMovement(for:to:in:)` 复用同一套地形、敌控区、占位和通行规则，只忽略本回合机动上限，适合只读区分“可通但超限”和“无可用路径”。

## 实现要求

1. 在 `WarCommandExecutor` 中为道路审计新增私有 helper，集中生成路径诊断文案。
2. 若本回合可达，显示 `路径 cost/effectiveMovement`。
3. 若忽略机动上限后可达，显示 `路径超限 cost/effectiveMovement，转入后备命令`。
4. 若目标是原地、越界或被其他军队占据，使用对应中文短诊断，避免把占位/边界问题误写成“无通路”。
5. 若忽略机动上限后仍不可达，显示 `无通路，转入后备命令`。
6. 本轮只增加诊断解释，不改变真实移动合法性、fallback 命令、隐藏敌军、同盟借道或完整跨回合行军制度。
