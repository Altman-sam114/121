# v2.4 军队接战无敌对空状态提示词

## 1. 目标

继续强化“武将、道路、交战”的可读链路：当选中军队当前没有任何 hostile 敌对军队可用于接战预判时，军队详情仍显示一条明确空状态，避免玩家把空白误解为面板失效或信息未加载。

## 2. 范围

- 仅修改 `AppContainer.selectedUnitCombatPreviewNotes` 的只读摘要生成。
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改 `UnitInspectorView` 参数结构，不新增 UI API。
- 不修改 `CombatRules`、`MovementRules`、`GeneralInfluence`、`SupplyRules`、`CommandValidator`、`RuleEngine`、`CommandExecutor` 或真实攻击规则。

## 3. 实现要求

- `selectedUnitCombatPreviewNotes` 继续使用 `target.faction.isHostile(to: division.faction)` 筛选敌对军队。
- 当筛选后没有任何未溃散 hostile 目标时，返回一条短文案，例如 `接战：当前无可预判敌对军队`。
- 当存在 hostile 目标但不在射程内时，保留既有未接敌/接近预判链路。
- 当存在射程内 hostile 目标时，保留既有多目标对比、接战官道、武将对阵、风险和审计摘要。
- 空状态只读展示当前可见规则语义，不代表完整外交制度、隐藏敌军、未来回合威胁或胜率判断。

## 4. 禁止项

- 不改敌对关系计算，不改 `Faction.isHostile(to:)`。
- 不新增攻击合法性、路径搜索、隐藏敌军推断或胜率计算。
- 不在 UI 层直接改 `GameState`。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 选中军队没有任何 hostile 敌对军队时，“接战预判”能显示明确空状态。
- 有敌对军队但未入射程时，仍显示既有最近敌军和接近预判。
- 有射程内敌对军队时，既有多目标交战预判不变。
- 改动不影响真实移动、道路、敌控区、补给、交战、命令执行或 AI 战术塑形规则。
