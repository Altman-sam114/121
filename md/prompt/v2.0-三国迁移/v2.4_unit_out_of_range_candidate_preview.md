# v2.4 军队未接敌时接近候选预判兼容层

## 目标

当选中军队当前射程内没有敌军，但地图上存在多支敌军时，在“接战预判”中补充最多三支最近敌军的接近候选摘要，帮助玩家判断接下来优先靠近哪支敌军、需要接近多少格、对方射程是多少，以及目标敌将是谁。

## 范围

- 修改 `WWIIHexV0/App/AppContainer.swift` 的 `combatOutOfRangePreviewNotes` 派生文本。
- 新增只读 helper，复用 `CombatRules.generalInfluenceSummary` 获取候选敌军武将身份。
- 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 禁止项

- 不改 `MovementRules` 的移动范围、敌控区、道路机动、移动成本或行动消耗。
- 不改 `CombatRules` 的射程、伤害、反击、撤退、死守、围城或兵种克制公式。
- 不改 `GeneralInfluence` 的道路加成或攻防修正。
- 不改 `CommandExecutor`、`CommandValidator`、`WarCommandExecutor`、`RuleEngine` 或真实命令执行结果。
- 不新增完整目标选择 AI、胜率计算、跨回合追击、路径安全或同盟借道判断。

## 当前架构依据

- `selectedUnitCombatPreviewNotes` 在射程内有敌军时走既有三目标接战预判。
- 当前射程内无敌军时，`combatOutOfRangePreviewNotes` 已显示最近敌军、接近武将、接近参考、接近态势、接近威胁、官道接近和官道入射程判断。
- `CombatRules.generalInfluenceSummary` 已可只读获取候选敌军对应的防守方武将姓名/id。

## 实现步骤

1. 在 `combatOutOfRangePreviewNotes` 中，当敌军数量大于 1 时追加 `接近候选` 摘要。
2. 候选按距离、名称、id 稳定排序，最多显示三支。
3. 每个候选显示目标名、当前距离、还需接近格数、敌方射程；若候选防守方有武将分配，显示 `敌将 ...`。
4. 保留既有最近敌军详细预判；射程内有敌军时既有三目标接战预判不变。

## 验收标准

- 选中军队且当前射程内没有敌军，但地图上存在多支敌军时，接战预判显示 `接近候选：...`。
- 接近候选最多显示三支最近敌军，并包含距离、需接近格数、敌方射程和可用敌将身份。
- 只有一支射程外敌军时不额外显示候选行，避免重复最近敌军摘要。
- 射程内有敌军时既有三目标接战预判保持不变。
- 真实移动、攻击、反击、补给、道路、日志结算、命令校验和动态战略同步行为不变。

## 风险提示

- 本轮没有运行 UI 烟测；新增“接近候选”文本在窄屏、长军队名、长武将名和 Dynamic Type 下的换行仍需云端 CI、后续 Agent C artifact 复判或人工授权运行检查。
- 摘要只读展示当前静态接近候选，不模拟敌方下回合 AI 行动、完整路径安全、同盟借道或真实胜率。
