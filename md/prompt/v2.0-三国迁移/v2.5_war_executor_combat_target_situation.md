# v2.5 宏观交战目标态势审计提示词

## 目标

继续推进“武将、道路、交战”体验，把 `WarCommandExecutor` 宏观军令生成 `.attack` 时已经可读取的目标兵力、地形、城关/官道、粮草和军令姿态，追加到 `WarDirectiveRecord.diagnostics` 的“交战审计”文案中，让 AI 防区指令和玩家武将宏观军令能更直接解释“打的是一个什么态势的目标”。

## 范围

- 修改 `WWIIHexV0/Commands/WarCommandExecutor.swift`。
- 更新 `md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 非目标

- 不改变 `CombatRules` 伤害、攻防修正、反击、撤退、围城、兵种克制或武将加成公式。
- 不改变 `WarCommandExecutor` 的目标筛选、排序、fallback、命令执行顺序或占领推进。
- 不改 `WarDirectiveRecord` Codable schema。
- 不改 `AgentPanelView`、`TurnManager`、`AppContainer` 或 SpriteKit 绘制。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试 / app 启动 / 全量构建。

## 架构依据

- `WarCommandExecutor.combatSelectionDiagnostic` 已在执行前生成交战审计，并通过 `WarCommandExecutionResult.diagnostics` 进入 AI / 玩家 `WarDirectiveRecord.diagnostics`。
- `AgentPanelView` 已按“交战审计”marker 将诊断归入“战术审计”分组。
- 目标态势只读自 `Division` 与 `HexTile`：兵力、地形、城池/关隘、官道、粮草、退守姿态，不执行命令、不修改状态。

## 实现要求

1. 在 `WarCommandExecutor` 中为交战诊断新增私有 helper，输出目标态势短句。
2. 目标态势至少包含：
   - `目标兵 strength/maxStrength`
   - 目标 hex 地形中文名
   - 有城池显示“据城”，有关隘显示“据关”
   - 有官道显示“临官道”
   - 粮草短码：`粮足 / 粮缺 / 粮断`
   - 姿态短码：`令退 / 令守`
3. 交战审计仍复用 `CombatRules.combatAuditSummary`，不复制或改写战斗公式。
4. 文案必须保持中文玩家可见，不包含 raw id、JSON、英文 schema 或下划线机器字段，避免 `AgentPanelView` 安全过滤兜底。
