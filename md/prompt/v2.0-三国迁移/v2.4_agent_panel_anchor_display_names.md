# v2.4 Agent 面板战略锚点展示名兼容层

## 目标

减少 AI 面板中外交对象、防区、目标郡县和指令摘要继续直接展示 `CountryId.rawValue` / `RegionId.rawValue` / `FrontZoneId.rawValue` 的情况，让君主、外交官、太守、军师、武将与防区指令审计优先显示当前三国状态中的国家、郡县名和防区名。

## 范围

- 只改 `AgentPanelView` 的展示层输入和格式化。
- 从 `RootGameView` 将当前 `GameState.diplomacyState.countries`、`GameState.map.regions` 与 `GameState.warDeploymentState.frontZones` 的只读显示名传入 AI 面板。
- 防区名为空或仍等于 raw id 时，用势力简称和前 1-2 个郡县名生成兜底展示名，例如 `曹军防区：官渡、许昌`。
- 保留 `AgentDecisionRecord`、`WarDirectiveRecord`、`GovernorDecisionRecord`、`StrategistDecisionRecord`、`GeneralDecisionRecord` 的 Codable 字段和 raw id。

## 非目标

- 不修改 `CountryId`、`RegionId`、`FrontZoneId`、`CountryProfile`、`RegionNode`、`FrontZone` schema。
- 不修改 AI 决策、parser、mapper、`ZoneDirective`、`WarCommandExecutor`、`RuleEngine` 或动态部署逻辑。
- 不迁移调试 JSON 的原始字段；调试 JSON 仍可用于回归审计。
- 不实现完整防区命名数据迁移或官渡大地图扩展。

## 架构依据

- Hex 仍是战术权威；Region / FrontZone 是从状态派生的战略展示和调度层。
- 玩家、AI 和聊天命令仍必须走 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine`。
- 本轮是 UI 审计显示兼容层，不改变命令执行结果或战略派生状态。

## 实现步骤

1. 在 `AgentPanelView` 初始化参数中加入 `[CountryId: String]`、`[RegionId: String]` 与 `[FrontZoneId: String]` 展示名映射，并提供空字典默认值保持调用兼容。
2. 将外交对象、君主重点防区、外交目标郡县、太守重点郡县、军师主防区/目标、武将摘要、防区指令 badge 和 directive summary 的 raw id 展示替换为 helper 输出。
3. 在 `RootGameView` 生成只读映射：
   - country 使用 `CountryProfile.name`，空名 fallback raw id。
   - region 使用 `RegionNode.name`，空名 fallback raw id。
   - front zone 优先使用 `FrontZone.name`，空名或 raw id 时 fallback 为 `Faction.shortDisplayName + 防区 + 郡县名摘要`。
4. 更新 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 验收标准

- AI 面板主要审计行不再把已知 region / zone 直接显示为 raw id。
- 已知郡县显示为 `RegionNode.name`。
- 已知国家显示为 `CountryProfile.name`。
- 防区显示为 `FrontZone.name` 或 `曹军防区：官渡、许昌` 这类三国可读 fallback。
- 原始记录、JSON schema、命令管线和规则执行无变化。
