# v2.4 AI 与执行器单位敌对筛选兼容层提示词

## 1. 目标

继续强化三国多势力、武将、道路和交战边界：AI 摘要、MockAI 威胁估算、ZoneCommander 目标/接触判断和 `WarCommandExecutor` 的具体单位目标筛选，遇到单位级敌军判断时必须使用 `Faction.isHostile(to:)`，避免把同阵营或 `.neutral` 中立单位只因“不是本方”就算作敌军、威胁或可攻击目标。

## 2. 范围

- 修改单位级敌对筛选：
  - `WWIIHexV0/Commands/WarCommandExecutor.swift`
  - `WWIIHexV0/Agents/RulerAgent.swift`
  - `WWIIHexV0/Agents/MockAICommander.swift`
  - `WWIIHexV0/Agents/ZoneCommanderAgent.swift`
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`

## 3. 实现要求

- `WarCommandExecutor.enemyStrength`、`hasEnemyPresence` 和 `visibleEnemyDivision` 只统计/选择 `division.faction.isHostile(to: faction)` 或 `target.faction.isHostile(to: zone.faction)` 的未毁灭单位。
- `RulerAgent` 的临近敌军强度摘要只统计 hostile 单位。
- `MockAICommander.visibleEnemyStrength` 只统计 hostile 单位。
- `ZoneCommanderAgent.visibleEnemyStrengthByRegion` 和 `hasEnemyPresence` 只统计 hostile 单位。
- 保留 region/controller 的非所属判断不动；那类判断表达未控制、争夺或候选区域，不等同于“敌军单位”。

## 4. 禁止项

- 不修改真实移动、攻击、道路、粮道、补给、外交、生产或回合流程规则。
- 不修改 `Faction.isHostile(to:)` 语义，不引入完整借道/同盟通行制度。
- 不把 `region.controller != faction`、`map.regions[...].controller != zone.faction` 等区域控制判断批量替换为 hostile，除非该判断直接是在筛选单位。
- 不新增持久字段，不修改 Codable schema，不改 Xcode project。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- AI 和执行器的单位级敌军强度、敌军存在、可见敌军目标筛选不再把同阵营或 `.neutral` 中立单位算作敌军。
- `WarCommandExecutor` 仍通过 `ZoneDirective -> Command -> RuleEngine` 执行具体命令，不绕过校验。
- 区域非所属/争夺判断保持原样，避免把“未控制区域”误改成“敌对区域”。
- 文档说明本轮只调整单位级 hostile 筛选，不代表基于外交关系的完整多势力非交战、借道或同盟通行制度完成。
