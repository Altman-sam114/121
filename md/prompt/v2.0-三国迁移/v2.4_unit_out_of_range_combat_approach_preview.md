# v2.4 军队未接敌时接近距离与官道预判兼容层

## 目标

当选中军队当前射程内没有敌军时，军队详情仍显示最近敌军、需接近格数和可达更近官道位，让玩家能在开战前判断道路接敌价值，而不是看到空白的“接战预判”。

## 范围

- 修改 `WWIIHexV0/App/AppContainer.swift` 的 `selectedUnitCombatPreviewNotes` 派生逻辑。
- 复用 `MovementRules.movementRange` 与 `MovementRules.isEnemyZoneOfControl` 计算只读官道接近摘要。
- 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 禁止项

- 不改 `MovementRules` 的移动范围、道路机动、敌控区、移动成本或行动消耗。
- 不改 `CombatRules` 伤害、反击、撤退、死守、围城或兵种克制公式。
- 不改 `CommandExecutor`、`CommandValidator`、`WarCommandExecutor`、`RuleEngine` 或真实命令执行结果。
- 不承诺移动后同回合可攻击；本轮只展示当前位置到最近敌军的接近信息。

## 当前架构依据

- `selectedUnitCombatPreviewNotes` 已集中生成军队详情接战预判文本，适合承接未接敌时的只读接近提示。
- `MovementRules.movementRange` 和 `isEnemyZoneOfControl` 已用于道路机动与接战官道预览；本轮继续复用它们，不重算独立规则。
- 玩家真实移动、攻击和敌控区限制仍由 `CommandValidator -> CommandExecutor -> RuleEngine` 执行。

## 实现步骤

1. 先统一收集当前选中军队的敌对军队与距离。
2. 射程内仍走既有三目标接战预判。
3. 若没有射程内敌军，但存在敌军，显示最近敌军、当前距离、军队射程和需接近格数。
4. 读取本回合可达格中能缩短到最近敌军距离的官道格，显示数量、最近距离、安全/受敌控区数量，以及我方/目标是否在官道。
5. 若全图没有敌军，不新增无意义提示。

## 验收标准

- 选中军队且当前射程内没有敌军但地图上存在敌军时，接战预判显示最近敌军、距离、射程和需接近格数。
- 若可达范围内存在能缩短敌距的官道格，显示官道接近位数量、最近距离和安全/受敌控区数量。
- 射程内有敌军时既有三目标接战预判保持不变。
- 真实移动、攻击、日志结算、命令校验和动态战略同步行为不变。

## 风险提示

- 本轮没有运行 UI 烟测；新增“接战距离/官道接近”文本在窄屏、长军队名和 Dynamic Type 下的换行仍需云端 CI、后续 Agent C artifact 复判或人工授权运行检查。
- 摘要只做当前位置的只读接近判断，不模拟移动后攻击、跨回合追击、完整路径安全、同盟借道或真实胜率。
