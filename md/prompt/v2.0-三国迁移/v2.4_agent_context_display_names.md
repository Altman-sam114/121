# v2.4 Agent 上下文展示名收敛

## 目标

- 继续推进三国迁移中玩家可见的 AI 审计文本，避免君主、外交官、太守、军师和模拟军机写入 `DirectiveEnvelope.theaterContext` 或 `DiplomacyState.summary` 时直接展示英文 fallback、`CountryId`、`RegionId` 或 `FrontZoneId` rawValue。
- 保持本轮聚焦显示层，不改变外交、道路、粮道、交战、生产、修路或武将战术塑形规则。

## 范围

- `RulerAgent`：君主上下文中的重点防区使用 `FrontZone.name` 或势力简称加郡县名 fallback。
- `DiplomatAgent`：外交摘要、理由和上下文使用国家展示名，缺名时回退势力展示名或中文占位。
- `GovernorAgent`：太守 rationale 和上下文中的重点郡县使用 `RegionNode.name` 或中文占位。
- `StrategistAgent`：军师 intent / rationale 中的主防区和目标郡县使用展示名。
- `MockAICommander`：模拟军机摘要和 fallback 指挥官名称中文化。
- `DiplomacyState`：外交 summary 与 legacy `.germany/.allies` profile 的玩家可见名称三国化，保留原 id / bloc id 兼容。

## 禁止项

- 不改 `CountryId`、`DiplomaticBlocId`、`RegionId`、`FrontZoneId`、`Faction` rawValue 或 Codable schema。
- 不改 `DiplomaticRelation.status`、tension、`Command.proposeDiplomacy`、`Command.improveRoad`、`Command.queueProduction` 或 `WarCommandExecutor` 执行逻辑。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Full。

## 轻量检查

- 对本轮改动 Swift 文件运行 `swiftc -parse`。
- 扫描旧英文 fallback 与 raw 展示残留：`mock directive(s)`、`no countries`、`hostile relation(s)`、`German Reich`、`Allied Coalition`、`preferredFrontZoneId?.rawValue` 等。
- 检查本轮改动文件尾随空白、冲突标记、prompt 索引和 `git diff --check`。

## 验收标准

- AI 面板、raw JSON 之外的上下文摘要和外交摘要不再把上述 raw id / 英文 fallback 当作玩家文案。
- 规则、数据 schema、调试 raw id 和旧兼容 id 保持不变。
- 文档同步说明本轮只是展示层收敛，运行时正确性等待云端 CI 或后续授权重测试确认。
