# v2.4 交战日志剩余兵力提示词

> 本阶段承接 `v2.4_unit_combat_strength_outcome_preview.md`。执行前必须先读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和相关源码。

## 目标

- 让真实攻击/反击日志与军队详情里的接战预判口径一致，执行后也能直接看到结算后剩余兵力。
- 在 `CommandExecutor` 的攻击和反击日志中追加目标军队结算后的 `remainingStrength/maxStrength`。
- 死守、包围撤退等额外损失发生时，日志中的剩余兵力必须反映额外损失后的最终值；歼灭时显示 `0/maxStrength` 并保留“被歼灭”文案。

## 范围

- 主要文件：
  - `WWIIHexV0/Rules/CommandExecutor.swift`
- 文档文件：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改 `CombatRules`、`GeneralInfluence`、`MovementRules`、`WarCommandExecutor`、`RuleEngine`、JSON schema、UI 入参或 Xcode project。

## 实现要求

- 继续让 `CommandExecutor.executeAttack` 通过既有 `CombatRules.attackDamage`、`counterAttackDamage` 和 `resolveCombatResult` 处理真实战斗。
- 只扩展私有 `CombatResultSummary` 或等价私有 helper，让日志能拿到结算后的剩余兵力。
- 攻击日志和反击日志走同一套 `combatLog` 逻辑，避免两条路径口径分叉。
- 不把余兵写入新的持久状态字段；只作为当前事件日志文本。

## 禁止项

- 不允许改变真实伤害公式、反击规则、撤退规则、死守额外损失、包围额外损失、武将加成规则、道路规则、命令验证或执行链路。
- 不允许 UI、日志或 Agent 直接修改 `GameState`。
- 不允许新增本机 Xcode build/test、模拟器、Probe、Smoke、Stage Regression、Dynamic Theater Regression、Full 或性能测试。
