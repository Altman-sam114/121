# v2.4 军队当前官道受压预判兼容层提示词

## 1. 目标

继续强化“武将、道路、交战”的只读态势反馈：`selectedUnitMobilityPreviewNotes` 已显示基础/有效机动、武将官道加成、当前位置或郡县官道状态，以及本回合可达官道的安全/受压数量。本轮补齐“当前军队已经站在官道上时，这个官道是否正受敌控区压迫”，让玩家判断这支军队当前道路位是安全机动点还是接敌压力点。

## 2. 范围

- 仅修改 `WWIIHexV0/App/AppContainer.swift` 的 `selectedUnitMobilityPreviewNotes` 只读派生。
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改 `UnitInspectorView` 入参或布局。
- 不修改 `MovementRules`、`CombatRules`、`GeneralInfluence`、`WarCommandExecutor`、`CommandValidator`、`RuleEngine` 或真实执行规则。

## 3. 实现要求

- 复用现有 `MovementRules.isEnemyZoneOfControl` 判断当前 hex 是否受 hostile 敌军邻接压迫。
- 仅当当前 hex `hasRoad == true` 时，把“当前位置：已在官道”扩展为“受敌控区压迫”或“未受敌控区压迫”。
- 当前不在官道时保留既有郡县官道数量/暂无官道文案。
- 保留既有 `reachableRoadAccessNote` 可达官道安全/受压统计。
- 该摘要只读展示当前静态态势，不执行移动、不模拟路径、不修改状态。

## 4. 禁止项

- 不改变移动范围、道路加成、敌控区、攻击、反击、撤退、补给、围城、粮道或武将加成规则。
- 不新增路径安全、攻击合法性、隐藏敌军推断、胜率或敌方回合模拟。
- 不新增 UI API、JSON schema 或 Codable 字段。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 选中军队当前站在官道上时，“道路机动”预判显示当前官道是否受敌控区压迫。
- 当前不在官道时既有郡县官道数量或暂无官道文案保持不变。
- 可达官道安全/受压统计保持不变。
- 真实道路、移动、敌控区、补给、交战和命令执行规则保持不变。
- 文档明确本轮只是只读 UI 派生文案改进，运行时 UI 表现仍需云端 CI、后续 Agent C artifact 复判或人工授权运行检查。
