# v2.4 武将面板麾下军队道路与接敌摘要提示词

## 1. 目标

继续强化“武将、道路、交战”的玩家决策闭环：在 `GeneralCommandPanelView` 的“麾下军队”列表中，除了兵力、粮草、军令和行动状态，还显示每支军队当前官道态势和可见敌军接敌态势，让玩家不切到军队详情也能判断麾下部队是否据道、受压、可战或受敌射。

## 2. 范围

- 仅修改只读 UI 派生与展示：
  - `AppContainer`
  - `GeneralCommandPanelView`
  - `RootGameView`
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改真实移动、攻击、敌控区、道路修缮、粮道、补给、外交或命令执行规则。

## 3. 实现要求

- 新增轻量只读 row model，避免 `GeneralCommandPanelView` 在 SwiftUI body 内推导道路、敌军或接战。
- `AppContainer` 负责从当前 `GameState` 派生“麾下军队”行摘要。
- 摘要保留既有兵力、粮草、军令和行动状态。
- 官道态势至少区分：
  - `据官道`
  - `官道受压`
  - `离官道`
- 接敌态势只能来自当前 viewer 可见、未毁灭、且 `Faction.isHostile(to:)` 判定 hostile 的敌军，避免隐藏敌军泄漏。
- 接敌态势可以显示：
  - 可攻击敌军：`可战 ... 距 ...`
  - 受敌方射程威胁：`受敌射 ... 距 ...`
  - 双方互在射程：`接敌 ... 互在射程`
  - 无射程内敌军时显示最近可见敌军距离。
- `RootGameView` 在所有 `GeneralCommandPanelView` 入口传入新 row model。
- 展示行允许多行换行，不用固定 2 行截断。

## 4. 禁止项

- 不在 SwiftUI View 内调用 `MovementRules` / `CombatRules` 做 per-unit 规则推导。
- 不改变 `MovementRules.isEnemyZoneOfControl`、`CombatRules`、`SupplyRules`、`CommandValidator`、`RuleEngine` 或 `CommandExecutor`。
- 不让不可见敌军进入武将面板麾下军队接敌摘要。
- 不把摘要写成完整路径安全、攻击合法性、胜率、同盟借道或敌方回合反制判断。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 武将面板“麾下军队”列表显示每支已展示军队的兵力、粮草、军令、行动、官道和接敌状态。
- 接敌摘要只来自可见 hostile 敌军，不泄漏隐藏敌军，不误把非敌对势力写成敌军。
- `GeneralCommandPanelView` 只接收 row model 并展示，不在 View 内推导规则。
- 真实移动、交战、道路、粮道、补给、外交和命令执行规则不变。
- 文档明确这是只读态势摘要，不是完整路径安全、胜率或攻击许可制度。
