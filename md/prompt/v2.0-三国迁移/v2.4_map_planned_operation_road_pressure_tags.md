# v2.4 地图计划军令官道与受压标识兼容层

## 目标

在玩家宏观军令的地图计划层中，把源/目标代表 hex 的官道状态和敌军压迫状态直接并入计划标识，让玩家不用打开详情面板也能初步判断“这道军令是否依托官道、目标锚点是否被敌军压迫”。

## 范围

- 修改 `WWIIHexV0/SpriteKit/BoardScene.swift` 的 `drawPlannedOperations` 只读渲染。
- 复用 `PlayerCommandState.plannedOperations`、`FrontZone.generalAssignment`、`HexTile.hasRoad` 和 `Faction.isHostile(to:)`。
- 只对源/目标 region 的代表 hex 或防区 HQ 代表 hex 做短摘要，不实现完整路径搜索。
- 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 禁止项

- 不改 `PlayerCommandState` schema，不新增存档字段。
- 不改 `MovementRules`、`CombatRules`、`GeneralInfluence`、`WarCommandExecutor`、`CommandValidator`、`RuleEngine` 或真实移动/攻击/道路/补给/交战结算。
- 不让 `BoardScene` 直接修改 `GameState`、hex controller、单位坐标、动态方面、防区、资源、道路或生产队列。
- 不把源/目标锚点摘要伪装成完整行军路径、真实胜率、移动后同回合攻击或敌方下回合反制预测。
- 不新增本机 Xcode build/test、模拟器、Probe、Smoke、Stage Regression、Dynamic Theater Regression、Full 或性能测试。

## 当前架构依据

- 玩家宏观军令由 `GeneralCommandPanelView -> AppContainer.submitPlayerDirective -> GeneralAgent.plan -> WarCommandExecutor -> RuleEngine` 执行。
- `AppContainer` 已记录 `PlayerPlannedOperation`，`BoardScene` 只读当前回合、当前查看势力的计划军令并画箭头/圆环。
- `HexTile.hasRoad` 是战术层道路状态来源；敌军压迫必须用 `Faction.isHostile(to:)`，不能用二元 `!= faction`。
- 计划军令没有唯一出发军队，因此本轮只读源/目标代表 hex，不调用单位路径搜索。

## 实现步骤

1. 将 `operationPoint` 拆为可返回 `HexCoord` 的 helper，绘制时再转换为像素点。
2. 为每条计划军令读取源锚点和可选目标锚点的 `hasRoad`。
3. 对源/目标锚点扫描敌对且未毁灭军队，距离 0 或 1 视为受压。
4. 在计划标签中追加短 tag：`双道`、`源道`、`目道`、`无道`、`据道`、`离道`，受压时追加 `/受压`。
5. 在源/目标锚点附近绘制小路况点：官道为金色，离道为深色，受压时用红色描边。

## 验收标准

- 玩家当前回合进攻计划的地图标签包含武将/短战术和源/目标官道短 tag。
- 玩家当前回合防御计划的地图标签包含武将/短战术和源点官道短 tag。
- 源/目标锚点附近有小路况点；受敌军压迫时描边转为红色。
- 敌军压迫判断使用 `Faction.isHostile(to:)`。
- 真实命令执行、道路、交战、补给、战略同步、微操锁和面板计划军令列表行为不变。

## 风险提示

- 本轮不做运行时 UI 烟测；多个计划箭头靠近、长标签和小屏缩放下的重叠仍需云端 CI、后续 Agent C artifact 复判或人工授权运行检查。
- 该标识只读展示源/目标代表 hex，不代表真实行军路径全程安全，也不模拟同盟借道、敌方回合反制或完整胜率。
