# v2.4 武将骑战官道机动规则对齐兼容层

## 目标

对齐武将 UI 提示与规则层道路机动触发条件：带 `cavalry_charge` / 骑兵突击技能的武将，若其麾下军队所在郡县存在官道网络且武将忠诚/满意度达标，应能和粮道调度、疾行、甲骑专精类技能一样借用郡县官道网络触发 `GeneralInfluence` 的道路机动加成。

## 范围

- 仅调整 `WWIIHexV0/Rules/GeneralInfluence.swift` 中道路网络技能集合。
- 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 和 `md/prompt/README.md`。
- 保持 `GeneralSkillDisplay`、`AppContainer` 现有只读提示口径不变，因为它们已经把 `cavalry_charge` 归入官道/骑战提示。

## 禁止项

- 不改 `Command`、`ZoneDirective`、`DirectiveEnvelope` 或 JSON schema。
- 不改路径搜索、移动成本、交战伤害、AI 目标选择、控制权、动态战区、部署防区或借道制度。
- 不新增本机 Xcode build / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full 验证。

## 架构依据

- `GeneralInfluence` 是武将道路机动与交战攻防修正的规则入口。
- `MovementRules.effectiveMovementLimit` 读取 `GeneralInfluence.roadMobilityBonus`，真实移动仍经 `CommandValidator -> CommandExecutor -> RuleEngine`。
- UI 的武将技能短提示和无加成原因已经把 `cavalry_charge` 视为官道/骑战相关技能；规则层必须避免出现“面板提示可借官道，实际无加成”的分叉。

## 实现步骤

1. 在 `GeneralInfluence` 内抽出私有道路网络技能集合，包含 `logistics`、`rapid_exploitation`、`armor_expert`、`cavalry_charge`。
2. 让 `roadMobilityBonus` 和 `usesRoadNetwork` 复用同一集合。
3. 确认当前位于官道时仍不依赖技能即可触发基础道路网络条件。
4. 同步文档，明确这是骑战/官道机动规则口径对齐，不是完整借道或骑兵系统重写。

## 验收标准

- `cavalry_charge` 与粮道调度、疾行、甲骑专精共用道路网络技能集合。
- 带骑兵突击技能的武将在郡县存在官道时，可触发 `GeneralInfluence` 的道路机动条件；加成上限仍为 2。
- UI 短提示、武将面板无加成原因和规则层触发条件不再分叉。
- 本地只跑轻量检查；重验证等待 `origin/main` 的 GitHub Actions `ci-results`。
