# v2.4 军队接战预判可见敌军口径提示词

## 1. 目标

继续强化“武将、道路、交战”的玩家决策闭环：军队详情的“接战预判”和地图攻击高亮只能基于玩家视角可见 hostile 军队生成，避免不可见敌军通过伤害、敌将、威胁、官道接近或高亮泄漏。

## 2. 范围

- 仅修改只读 UI 派生与交互提示：
  - `WWIIHexV0/App/AppContainer.swift`
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改 `UnitInspectorView` API，不修改真实攻击、移动、道路、粮道、补给、外交、AI 指令执行或 `RuleEngine`。

## 3. 实现要求

- `AppContainer.selectedUnitCombatPreviewNotes` 收集候选敌军时，继续使用 `Faction.isHostile(to:)` 判断敌对，并追加 `mapDisplayAdapter.isDivisionVisible(target, viewerFaction: playerFaction)`。
- 可见性必须使用玩家视角 `playerFaction`，不要用选中军队自己的 faction 推导敌方视野；观察模式继续由 `mapDisplayAdapter` 的 `revealAll` 兼容处理。
- 无可见 hostile 军队时，保留既有“接战：当前无可预判敌对军队”空状态。
- `refreshHighlights` 生成攻击高亮时，也只高亮当前玩家可见、未毁灭且 hostile 的目标坐标。
- 不改变真实攻击合法性；若玩家点击真实可见敌军，仍走既有 `Command.attack -> RuleEngine`。

## 4. 禁止项

- 不改 `CombatRules`、`MovementRules`、`SupplyRules`、`GeneralInfluence` 的规则数值或公式。
- 不改 `Command` / `ZoneDirective` / `WarCommandExecutor` / `CommandValidator` / `RuleEngine`。
- 不新增持久字段，不修改 Codable schema。
- 不让不可见敌军进入军队详情接战预判、接近候选、敌将摘要、官道接近或攻击高亮。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 军队详情“接战预判”只显示玩家视角可见 hostile 军队的伤害、反击、武将、审计、威胁和官道接近信息。
- 地图攻击高亮只标出玩家视角可见 hostile 目标。
- 观察模式仍可通过 `mapDisplayAdapter` revealAll 看到全图预判。
- 真实移动、攻击、伤害、反击、道路、粮道、补给、外交和命令执行规则不变。
