# v2.4 武将面板当前接敌配对摘要提示词

## 1. 目标

继续强化“武将、道路、交战”的可读链路：在 `GeneralCommandPanelView` 的“道路与交战”摘要中补充当前接敌配对，让玩家不只看到接敌攻防修正范围和最近敌军，也能知道哪两支军队正在形成当前武将攻防修正。

## 2. 范围

- 仅修改 `AppContainer.selectedGeneralInfluenceNotes` 的只读摘要生成。
- 可新增私有 helper，用当前麾下军队、hostile 敌军和 `GeneralInfluence.combatSummary(attacker:defender:in:)` 派生配对文本。
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改 `GeneralCommandPanelView` API、不修改 `PlayerPlannedOperation` Codable schema。

## 3. 实现要求

- 敌军过滤继续使用 `Faction.isHostile(to:)`，并排除已毁灭敌军。
- 当前接敌配对只在当前存在射程内攻击或防御关系时显示；未接敌时不新增空噪声行。
- 攻击配对使用麾下军队攻击 hostile 敌军的 `summary.attackBonus`，文本可类似 `接敌配对：曹骑兵军 -> 袁步卒营，攻+1，距 1 格`。
- 防御配对使用 hostile 敌军攻击麾下军队时，麾下军队作为防御方的 `summary.defenseBonus`，文本可类似 `接敌配对：袁步卒营 -> 曹骑兵军，防+1，距 1 格`。
- 多个配对同时存在时，稳定选择一条：优先非零修正，再按修正绝对值、距离、攻防方向、军队名和 id 排序。
- 摘要只读展示当前静态配对，不模拟路径、胜率、隐藏敌军、同盟借道、移动后同回合攻击或敌方回合反制。

## 4. 禁止项

- 不改 `MovementRules`、`CombatRules`、`GeneralInfluence` 的规则结果。
- 不改 `WarCommandExecutor`、`CommandValidator`、`RuleEngine`、真实移动、攻击、反击、撤退、补给、围城、道路或动态战区。
- 不让 SwiftUI View 直接推导敌军、路径或交战规则；View 只渲染 `influenceNotes`。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 武将面板“道路与交战”摘要在当前接敌时能显示一条当前接敌配对、攻/防修正和 hex 距离。
- 旧的道路机动、官道受益军队、接敌攻防范围、最近敌军和麾下备战摘要仍存在。
- 改动不影响真实移动、道路、敌控区、补给、交战、命令执行或 AI 战术塑形规则。
- 文档明确该摘要是只读当前静态配对，不代表路径安全、胜率或敌方回合预测。
