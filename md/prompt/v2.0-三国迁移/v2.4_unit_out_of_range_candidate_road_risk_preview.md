# v2.4 军队未接敌时接近候选路况与风险预判兼容层

## 目标

当选中军队当前射程内没有敌军，但地图上存在多支敌军时，在 `接近候选` 摘要中补充每个候选目标的本回合可达接近信息、可用官道信息和入敌射程风险，帮助玩家比较“靠近谁更现实、哪条候选更依赖官道、靠近后是否暴露在敌方射程内”。

## 范围

- 修改 `WWIIHexV0/App/AppContainer.swift` 的未接敌 `selectedUnitCombatPreviewNotes` 派生文本。
- 复用 `MovementRules.movementRange`、`MovementRules.isEnemyZoneOfControl`、`HexTile.hasRoad` 和候选敌军 `range`。
- 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 禁止项

- 不改 `MovementRules` 的真实移动范围、道路机动、敌控区、移动成本或行动消耗。
- 不改 `CombatRules` 的射程、伤害、反击、撤退、死守、围城或兵种克制公式。
- 不改 `GeneralInfluence` 的道路加成或攻防修正。
- 不改 `CommandExecutor`、`CommandValidator`、`WarCommandExecutor`、`RuleEngine` 或真实命令执行结果。
- 不新增完整路径规划、目标选择 AI、同盟借道、移动后同回合攻击、跨回合追击或真实胜率模拟。

## 当前架构依据

- `selectedUnitCombatPreviewNotes` 在射程内有敌军时走既有三目标接战预判。
- 当前射程内无敌军时，`combatOutOfRangePreviewNotes` 已显示最近敌军、接近候选、接近武将、接近参考、接近态势、接近威胁、官道接近和官道入射程判断。
- `combatOutOfRangeCandidateText` 已能按距离、名称和 id 稳定排序最多三支候选敌军。
- `MovementRules.movementRange` 已是本回合可达格集合；本轮只读取它，不扩展真实规则。

## 实现步骤

1. 在未接敌分支中只计算一次 `MovementRules.movementRange`，供最近敌军威胁、官道接近和候选摘要复用。
2. 为每个接近候选计算本回合可达且能缩短到该目标距离的格子。
3. 候选行追加 `可达距 ...`，表示本回合可达接近位中距离该目标最近的距离。
4. 若最近可达距离已经进入我方射程，追加 `可入射程`。
5. 若可达接近位里存在官道格，按是否受敌控区压迫追加 `安全官道 ...` 或 `官道受压 ...`。
6. 若可达接近位会进入该候选敌军射程，追加 `入敌射 ...`。
7. 若本回合没有任何可达接近位，追加 `无可近位`。

## 验收标准

- 未接敌且存在多支敌军时，`接近候选` 每个候选除距离、需接近格数、敌方射程和敌将外，还展示可达接近距离。
- 有安全官道接近位时显示安全官道数量；只有受压官道时显示受压官道数量。
- 靠近候选会进入敌方射程时显示入敌射数量。
- 无可达接近位时显示 `无可近位`。
- 射程内有敌军时既有三目标接战预判保持不变。
- 真实移动、攻击、反击、补给、道路、日志结算、命令校验和动态战略同步行为不变。

## 风险提示

- 本轮没有运行 UI 烟测；新增候选行细节在窄屏、长军队名、长武将名和 Dynamic Type 下的换行仍需云端 CI、后续 Agent C artifact 复判或人工授权运行检查。
- 摘要只读展示当前静态可达格和射程关系，不模拟敌方下回合 AI 行动、完整路径安全、同盟借道、移动后同回合攻击或真实胜率。
