# v2.4 武将交战日志审计兼容层

本文记录“武将影响交战要能看见”的小步实现提示词与交付边界。执行前仍必须阅读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和总提示词 `md/prompt/v2.0-三国迁移/codex-v2.0-三国aiagent迁移总提示词.md`。

## 目标

- 保持 `GeneralInfluence` 已有攻防修正公式不变。
- 新增只读 `GeneralCombatInfluenceSummary`，把攻击方武将、守方武将、攻击修正和防御修正整理为日志摘要。
- `CommandExecutor` 在攻击和反击日志中追加武将修正摘要，方便 UI、人工和 Agent C 复判“哪个武将影响了交战”。
- 保持交战仍由 `Command.attack -> CommandValidator -> CommandExecutor -> CombatRules` 执行。

## 非目标

- 不新增完整技能树、单挑、士气、疲劳或战报面板重构。
- 不改伤害公式，不改变 `CombatRules.damage`、撤退或反击规则。
- 不让 `GeneralAgent` 或 UI 直接修改 `GameState`。
- 不跑本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构边界

```text
GeneralAssignment
  -> GeneralInfluence.attackBonus / defenseBonus
  -> GeneralCombatInfluenceSummary
  -> CombatRules.generalInfluenceSummary
  -> CommandExecutor.combatLog
  -> GameState.eventLog
```

- 日志摘要必须和实际 `CombatRules` 使用同一套 `GeneralInfluence` 计算。
- 仅当攻防修正非 0 时追加日志片段，避免噪声。
- 日志是审计输出，不参与规则计算，也不作为后续状态权威。
