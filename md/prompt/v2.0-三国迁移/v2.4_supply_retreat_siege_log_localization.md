# v2.4 SupplyRules 撤退、围城与粮草日志三国化提示词

## 目标

继续围绕“武将、道路、交战”推进三国迁移，把 `SupplyRules` 中玩家可见的补给整补、撤退、撤退失败、围城/包围损耗和撤退整顿事件日志改为中文三国语义。该切片服务玩家读懂粮道、围城、撤退和交战后续损耗。

## 范围

- `WWIIHexV0/Rules/SupplyRules.swift`
  - `applyResupplyRest(to:in:)` 的整补、围城恢复阻断、未恢复日志。
  - `resolveRetreat(for:in:)` 的撤退成功和撤退失败日志。
  - `applyEncirclementAttrition(in:)` 的粮道断绝围城损耗和包围损耗日志。
  - `advanceRetreatStatusIfNeeded(for:in:)` 的撤退整顿完成日志。
- 文档同步：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`

## 非目标

- 不改补给路径、围城判定、撤退目的地、撤退失败惩罚、包围损耗数值、恢复数值或事件类别。
- 不改 `SupplyState`、`RetreatMode`、`Command`、`RuleEngine`、`CommandExecutor`、`CombatRules` 或 `WarCommandExecutor`。
- 不改 JSON schema、Codable 字段、rawValue、旧阿登 fallback 或胜负规则。
- 不运行 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 当前架构依据

- `SupplyRules` 负责补给状态、整补、撤退推进和包围损耗，是粮道、围城和交战后续损耗的规则来源。
- `GameState.appendEvent(...)` 写入玩家可见事件日志；本轮只改事件文案，不改状态写入。
- `SupplyRules.isBesieged` 已用城池/关隘、断粮和敌邻判定围城；本轮只让日志明确“粮道断绝”“围城损耗”“包围损耗”。

## 实现步骤

1. 将 `reinforced in ... strength` 改为“完成整补：兵力 +N”。
2. 将围城恢复阻断和非围城未恢复日志改为中文。
3. 将撤退成功/失败日志改为“后撤至”“额外损失 N 点兵力”。
4. 将 siege / encirclement attrition 日志改为“粮道断绝，遭受围城损耗”和“遭受包围损耗”。
5. 将撤退恢复完成日志改为“完成撤退整顿”。
6. 扫描 `SupplyRules.swift`，确认旧英文事件日志无残留。
7. 更新文档和阶段记录，说明本轮只做显示层日志迁移。
