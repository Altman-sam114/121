# v2.5 宏观军令交战审计提示词

## 目标

继续推进“武将、道路、交战”体验，把 `WarCommandExecutor` 在宏观防区军令中选择 `.attack` 命令时已经可取得的交战目标、预计伤害、反击风险和 `CombatRules.combatAuditSummary` 写入 `WarDirectiveRecord.diagnostics`，让 AI 与玩家武将宏观军令在 Agent 面板中能解释“为什么这支军队打这个目标、交战修正来自哪里”。

## 范围

- 修改 `WWIIHexV0/Commands/WarCommandExecutor.swift`。
- 更新 `md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 非目标

- 不改变 `CombatRules` 伤害、攻防修正、反击、撤退、围城、兵种克制或武将加成公式。
- 不改变 `WarCommandExecutor` 的目标筛选、排序、fallback、命令执行顺序或占领推进。
- 不改 `WarDirectiveRecord` Codable schema。
- 不改 `TurnManager.swift`、`AppContainer.swift`、`AgentPanelView` 或 SwiftUI 布局；上一轮已统一合并 `execution.diagnostics`。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试 / app 启动 / 全量构建。

## 架构依据

- `CombatRules.combatAuditSummary(attacker:defender:in:)` 已是攻击日志和军队详情接战预判共享的只读交战审计入口。
- `WarCommandExecutor` 在生成 `.attack(attackerId:targetId:)` 命令时，执行前的 `GameState` 仍保留真实攻击方、目标和武将/地形/围城上下文，适合生成不改变行为的诊断。
- `WarCommandExecutionResult.diagnostics` 已由 AI `TurnManager` 和玩家 `AppContainer.submitPlayerDirective` 合并进 `WarDirectiveRecord.diagnostics`。

## 实现要求

1. `WarCommandExecutor` 持有只读 `CombatRules`。
2. 生成每条宏观命令后、执行前，若命令是 `.attack`，生成交战审计短句。
3. 交战审计包含攻击方/目标展示名、距离、预计伤害、可反击伤害或无反击、`CombatAuditSummary.logFragment` 中的攻防/武将/地形/围城/侧击因素。
4. 诊断文案必须是纯中文玩家文案，不包含 raw id、JSON、英文 schema 或下划线机器字段，避免 `AgentPanelView.displaySafeDiagnostics` 兜底过滤。
5. 单条 directive 最多展示前三条交战审计，更多命令用“另 N 条交战审计已记录。”汇总，避免面板过长。
6. 道路审计继续保留，并与交战审计共同返回到 `WarCommandExecutionResult.diagnostics`。

## 验收标准

- AI 防区指令记录和玩家武将宏观军令记录都能携带执行器交战审计。
- 交战审计复用 `CombatRules.combatAuditSummary`，不复制或改写战斗公式。
- Agent 面板可直接展示诊断，不需要 UI 结构变化。
