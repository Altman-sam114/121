# v2.4 军队未接敌时接近武将预判兼容层

## 目标

当选中军队当前射程内没有敌军时，在“接战预判”的接近信息中补充最近敌军对应的我方/敌方武将身份和非零武将修正参考，让玩家知道接近的是哪位敌将、哪些武将会影响后续交战。

## 范围

- 修改 `WWIIHexV0/App/AppContainer.swift` 的 `combatOutOfRangePreviewNotes` 派生文本。
- 复用 `CombatRules.generalInfluenceSummary` 的 `attackerGeneralName/id`、`defenderGeneralName/id`、`attackBonus` 与 `defenseBonus`。
- 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 禁止项

- 不改 `GeneralInfluence` 的武将查找、道路加成、攻击加成或防御加成。
- 不改 `CombatRules` 伤害、反击、撤退、死守、围城或兵种克制公式。
- 不改 `MovementRules` 的移动范围、道路机动、敌控区、移动成本或行动消耗。
- 不改 `CommandExecutor`、`CommandValidator`、`WarCommandExecutor`、`RuleEngine` 或真实命令执行结果。
- 不承诺移动后同回合可攻击；本轮只展示当前位置对最近敌军的武将参考。

## 当前架构依据

- 未接敌接近预判已经在 `selectedUnitCombatPreviewNotes -> combatOutOfRangePreviewNotes` 中生成。
- `CombatRules.generalInfluenceSummary` 已把当前军队和目标敌军对应的武将快照与攻防修正汇总为只读摘要。
- 真实移动、攻击和敌控区限制仍由 `CommandValidator -> CommandExecutor -> RuleEngine` 执行。

## 实现步骤

1. 在 `combatOutOfRangePreviewNotes` 中为最近敌军计算 `GeneralCombatInfluenceSummary`。
2. 若任一侧有武将分配，追加 `接近武将：我方 ...，敌方 ...`；缺少一侧显示“未任命”。
3. 若存在非零 `attackBonus` 或 `defenseBonus`，追加 `接近参考：武将修正 ...`。
4. 保留既有最近敌军、距离、射程、需接近格数和官道接近摘要。

## 验收标准

- 选中军队且当前射程内没有敌军但地图上存在敌军时，若最近敌军接近预判任一侧有武将分配，接战预判显示 `接近武将...`。
- 若当前最近敌军存在非零武将攻防修正，接战预判显示 `接近参考：武将修正...`。
- 射程内有敌军时既有三目标接战预判保持不变。
- 真实移动、攻击、日志结算、命令校验和动态战略同步行为不变。

## 风险提示

- 本轮没有运行 UI 烟测；新增“接近武将/接近参考”文本在窄屏、长武将名和 Dynamic Type 下的换行仍需云端 CI、后续 Agent C artifact 复判或人工授权运行检查。
- 摘要只做当前位置对最近敌军的只读武将参考，不模拟移动后攻击、跨回合追击、完整路径安全、同盟借道或真实胜率。
