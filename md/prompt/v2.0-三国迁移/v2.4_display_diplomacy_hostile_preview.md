# v2.4 地图与面板外交敌对显示兼容层提示词

## 1. 目标

继续推进三国多势力、武将、道路和交战迁移：上一轮已让 `AppContainer` 的只读预览按 `DiplomaticRelation.status` 区分敌对，本轮补齐剩余地图与 SwiftUI 面板显示口径，让计划军令地图短标签、军令状态文案和武将目标预览区块不再直接用静态 `Faction.isHostile(to:)` 推导“敌军/非敌对”。

## 2. 范围

- `WWIIHexV0/SpriteKit/BoardScene.swift`
  - 计划军令源点/目标点官道“受压”短标签。
  - 计划军令地图最近可见敌军短标签。
- `WWIIHexV0/UI/CommandPanelView.swift`
  - 已选择非己方军队时的“敌军/非敌对军队”状态文案。
- `WWIIHexV0/UI/GeneralCommandPanelView.swift`
  - 目标预览区块显示 gate 不再自行读取 `targetZone.faction.isHostile(to:)`。
- `WWIIHexV0/UI/RootGameView.swift`
  - 传入 `DiplomacyState` 和 `AppContainer` 派生的目标预览显示布尔值。
- `WWIIHexV0/App/AppContainer.swift`
  - 提供 `selectedGeneralTargetUsesDiplomaticHostility` 作为显示层派生。

## 3. 实现要求

- `BoardScene` 只读地图短标签通过 `state.diplomacyState.isHostile(between:and:)` 判断 hostile / atWar，并保留可见性过滤。
- `CommandPanelView` 只用 `DiplomacyState` 影响状态文案，按钮可用性仍只看玩家己方、当前阶段和是否已行动。
- `GeneralCommandPanelView` 只展示 `AppContainer` 已派生的 `shouldShowTargetPreview`，不重复推导外交关系。
- 缺外交建档时继续依赖 `DiplomacyState.isHostile` 内部 fallback。

## 4. 禁止项

- 不修改 `Faction.isHostile(to:)` 语义。
- 不修改 `CommandValidator`、`RuleEngine`、`WarCommandExecutor`、`MovementRules`、`SupplyRules`、`WarDeploymentManager`。
- 不修改地图点击自动攻击、攻击高亮、武将“进攻郡县”按钮可用性、`selectedAttackTarget` 或玩家命令提交入口。
- 不实现完整借道、同盟通行、共同作战堆叠、补给共享或外交宣战制度。
- 不新增持久字段，不修改 JSON schema，不改 Xcode project。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 计划军令地图官道受压和近敌短标签与 `DiplomaticRelation.status` hostile / atWar 口径一致。
- 军令面板状态文案按外交关系显示敌军或非敌对军队。
- 武将面板目标预览区块与 `AppContainer.selectedGeneralTargetPreviewNotes` 使用同一外交敌对派生。
- 真实攻击、移动、道路、粮道、部署、补给和命令提交规则不变。
- 本机只跑轻量检查，重验证交给 GitHub Actions。
