# v2.3 三国兵种模板兼容层小步记录

完成日期：2026-07-04

## 目标

沿 `codex-v2.0-三国aiagent迁移总提示词.md` 的 v2.3 方向，先完成“官渡默认剧本不再复用二战 unit template id”的最小可验证小步：

- 新增三国兵种模板资源 `sanguo_unit_templates.json`。
- 官渡预览初始单位改用三国 templateId。
- 保留旧 `unit_templates.json` 和旧 component rawValue 作为阿登 fallback。
- 不在本轮实现完整围城、士气、兵种克制或多势力 turn order。

## 实现范围

- `WWIIHexV0/Data/sanguo_unit_templates.json`
  - 新增 `infantry_camp`、`cavalry_wing`、`archer_camp`、`siege_engine_camp`、`garrison_camp`、`naval_fleet`。
  - id 全部使用 ASCII，中文只放在 `displayName`。
- `WWIIHexV0/Core/Division.swift`
  - `ComponentType` 新增 `cavalry`、`archer`、`siegeEngine`、`naval`、`guard`。
  - 旧 `tank/motorizedInfantry/infantry/artillery` 保留，继续兼容旧 JSON / 测试。
- `WWIIHexV0/Data/DataLoader.swift`
  - `loadUnitTemplates()` 优先读 `sanguo_unit_templates.json`，缺文件时才 fallback 到旧 `unit_templates.json`。
  - `makeDivisions()` 使用 template `maxHP` 作为 `maxStrength` 上限，避免官渡初始兵力被旧默认 10 截断。
- `WWIIHexV0/Data/guandu_200_scenario.json`
  - 官渡初始单位 templateId 改为 `garrison_camp`、`cavalry_wing`、`siege_engine_camp`、`infantry_camp`。
- `WWIIHexV0.xcodeproj/project.pbxproj`
  - 将 `sanguo_unit_templates.json` 加入 Data group、iOS 主 target 和 macOS 主 target resource phase。

## 禁止项

- 不恢复 organization。
- 不改 `Command` / `ZoneDirective` / `WarCommandExecutor` / `RuleEngine` 的统一执行边界。
- 不新增完整围城/士气/兵种克制规则。
- 不跑本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Full。

## 验收要点

- 官渡 JSON 初始单位不再引用 `*_division` 模板 id。
- 三国模板 components 权重和为 1。
- 新 component rawValue 可以被 `ComponentType` 解码。
- 旧阿登模板仍可通过 `unit_templates.json` 或 fallbackComponents 兼容。
- 轻量检查通过后提交并 push 到 `origin/main`，等待 `ci-results` 云端重验证。
