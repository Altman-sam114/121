# v2.4 经济与生产日志三国化提示词

本文记录“经济结算、自动补员和生产部署事件日志仍有英文残留”的小步实现提示词与交付边界。执行前仍必须阅读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和总提示词 `md/prompt/v2.0-三国迁移/codex-v2.0-三国aiagent迁移总提示词.md`。

## 目标

- 继续推进三国多势力、武将、道路、粮道和交战迁移，把 `EconomyRules` 中玩家可见的经济、粮草短缺、自动补员、生产完成和后方部署事件日志改为中文三国语义。
- 资源摘要统一使用“人口 / 军械 / 粮草”，避免 `MP / IC / SUP`、`stockpile`、`upkeep`、`reinforcement`、`supplies` 等英文缩写进入事件日志。
- 生产部署日志优先显示三国军队展示名和郡县后方安全格，方便玩家理解太守生产、后方补员、粮草与交战压力的关系。

## 非目标

- 不修改 `EconomyResources` 字段名、`ProductionKind` rawValue、生产队列 schema、订单 id、Codable 或旧 JSON 兼容。
- 不修改收入、维护费、自动补员、生产部署、安全后方筛选、敌邻判断、道路修缮、外交 hostile gate、战斗、移动、撤退或动态战区推进规则。
- 不新增完整屯田、治安、民心、独立粮仓库存或多势力经济回合制度。

## 架构依据

- 生产和太守建议仍必须走 `Command.queueProduction -> CommandValidator -> CommandExecutor -> EconomyRules`。
- 经济结算仍在 active faction 结束回合时由 `CommandExecutor.executeEndTurn -> EconomyRules.resolveFactionTurn` 触发。
- 自动补员仍只处理本阵营、未毁灭、未撤退、`supplied`、兵力未满且不与外交 hostile 军队相邻的军队。
- 生产完成军队仍只能部署在本方控制、passable、空置、非敌邻，且位于合格郡县或补给源的后方 hex。

## 实现步骤

1. 在 `WWIIHexV0/Rules/EconomyRules.swift` 中只替换 `appendEvent(...)` 的玩家可见字符串。
2. 将 `resourceSummary(_:)` 从 `MP / IC / SUP` 改为“人口 / 军械 / 粮草”。
3. 自动补员和部署日志使用 `Division.thematicDisplayName`，避免生产单位名由旧兼容 id 或英文 fallback 直出。
4. 部署日志增加只读郡县名辅助；缺郡县资料时使用“后方安全格 q,r”。
5. 同步更新 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 禁止项

- 禁止绕过 `Command` / `RuleEngine` 直接修改经济、地图、军队或生产状态。
- 禁止改动补员数值、生产成本、部署筛选、敌对判定或供应路径。
- 禁止运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

- `md/prompt/v2.0-三国迁移` 与 `md/prompt/README.md` 索引一致性检查。
- `git diff --check`

## 验收标准

- `EconomyRules` 的经济启动、排产资源不足、排产成功、府库结算、战略粮草耗尽、自动补员、粮草辎重完成、生产军队部署和无安全后方部署格事件日志均为中文三国语义。
- 资源摘要显示人口、军械、粮草。
- 规则行为、schema、rawValue、命令入口和数值无变化。
- 本机只完成允许的轻量检查，重验证等待 `origin/main` 云端 CI artifact。
