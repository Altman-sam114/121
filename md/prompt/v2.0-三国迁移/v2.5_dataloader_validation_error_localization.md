# v2.5 DataLoader 验证错误文案三国化

## 目标

继续推进三国迁移的玩家/编辑器可见文案收口，把 `DataLoader` 旧阿登兼容校验中可能透出的 German / Allied / guderian 英文错误说明改为中文三国语义，减少默认加载失败、编辑器验收或调试面板中的旧题材泄露。

## 范围

- 只修改 `WWIIHexV0/Data/DataLoader.swift` 的 `DataValidationError.message` 文案。
- German supply source 错误显示为“剧本缺少曹军补给源。”
- Allied supply source 错误显示为“剧本缺少袁军补给源。”
- 兼容 agent 引用未知 division 错误显示为“兼容武将配置引用了未知军队：...。”
- `guderian.assignedDivisionIds` 兼容校验错误显示为“兼容武将配置必须完整覆盖曹军初始军队。”
- 缺少 `guderian` agent 配置错误显示为“剧本缺少兼容武将配置。”
- 同步 `md/prompt/README.md` 与 `update_log.md`。

## 禁止项

- 不修改 `DataLoader` 的校验条件、函数名、fallback 顺序或 `scenario.id == "ardennes_v0"` 分支。
- 不修改 `Faction` rawValue、`germany/allies` 数据身份、`guderian` 兼容 id、JSON、schema、测试文件、规则、AI 或命令管线。
- 不处理 `loadArdennesDataSet()` / `loadArdennesV02Regions()` API 名，也不重命名旧阿登数据文件。
- 不运行 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构依据

- 项目当前仍保留旧阿登数据和 Legacy Agent D 作为 fallback / 回归参考，但玩家可见文案应尽量中文三国化。
- `DataValidationError.message` 可能通过加载失败、编辑器验收或调试输出被人看到；本轮只改展示语义，不改变数据身份。
- 当前兼容 `.germany/.allies` 在默认三国语义中对应曹操 / 袁绍方向，和 `Faction.displayName`、MapEditor 兼容势力标签保持一致。
