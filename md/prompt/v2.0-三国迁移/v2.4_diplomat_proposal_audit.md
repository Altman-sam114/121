# v2.4 外交提案审计兼容层

本文记录“君主、外交、太守、军师、武将 AI Agent 分层”中的外交官小步实现提示词与交付边界。执行前仍必须阅读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和总提示词 `md/prompt/v2.0-三国迁移/codex-v2.0-三国aiagent迁移总提示词.md`。

## 目标

- 新增 deterministic `DiplomatAgent`，承接君主姿态并读取 `DiplomacyState` 的国家、集团、关系、紧张度和战线压力。
- 新增 `DiplomatDecisionRecord` 与 `DiplomaticProposal`，把同盟、停战、借道、称臣、讨伐檄文、奉表勤王等外交提案写入 `DiplomacyState.diplomatRecords`。
- `TurnManager` 在君主姿态塑形之后、太守内政建议之前调用外交层，并把提案追加到 `DirectiveEnvelope.theaterContext` 与 AI raw JSON。
- `AgentPanelView` 和 `DiplomacyPanelView` 显示外交官、提案、对象、目标郡县和 rationale。

## 非目标

- 不直接修改 `DiplomaticRelation`、国家、集团、资源、借道权限或战争状态。
- 不新增外交 directive validator / executor，不实现真实同盟、停战、称臣、贡赋或借道规则。
- 不改完整多势力 turn order，不让三国新增势力进入完整 AI 回合循环。
- 不新增真实 LLM、网络调用、模型路径或 API key。
- 不跑本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构边界

```text
RulerAgent.adjust
  -> DiplomatAgent.plan
  -> DiplomatDecisionRecord + DirectiveEnvelope.theaterContext
  -> GovernorAgent.plan
  -> StrategistAgent.plan
  -> GeneralAgent.plan
  -> WarCommandExecutor
  -> RuleEngine
```

- `DiplomatAgent` 只读 `GameState` 并输出 Codable 记录。
- 外交提案可以影响后续 Agent 的上下文解释，但本轮不生成底层 `Command`，也不改变真实外交关系。
- 后续若让外交提案真实生效，必须先新增结构化外交 directive、validator 和 executor，并保持玩家/AI 命令仍经规则系统。
