# v2.4 Command 审计展示名安全收敛

## 目标

继续推进三国迁移中武将、道路和交战相关的可见审计闭环，把 AI 命令结果记录中复用 `Command.displayName` 导致的 raw 军队 id、坐标、郡县 id、国家 id 暴露风险收敛为中文命令类别。

## 范围

- 保留 `Command.displayName` 的兼容输出，不改变 Codable、旧调试字段或命令执行规则。
- 新增 `Command.auditDisplayName`，供 `CommandResultSummary` 记录安全审计标题。
- `CommandResultSummary.commandDisplayNameForDisplay` 同时兼容旧 raw 文案、英文旧文案和新安全审计文案。
- 修正 `ZoneCommanderAgent` 对最近固守命令的识别，不再只检查旧英文 `Hold`。
- Legacy Agent D 映射错误文案不再把 `divisionId` 放进玩家/AI 可见错误说明。

## 非目标

- 不重命名 `Command` case、`Division`、`Faction` 或数据 id。
- 不改变移动、攻击、固守、撤退、补给、修路、生产、外交或结束回合规则。
- 不删除旧阿登 fallback 数据，不改胜利条件主逻辑。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构依据

- 玩家、AI 和后续聊天命令必须经 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine`。
- `Command.displayName` 当前仍服务兼容审计和旧代码，不能在未全面审查前直接改语义。
- AI 面板实际展示通过 `CommandResultSummary.commandDisplayNameForDisplay`，因此安全收敛应在记录/展示边界处理。

## 实现步骤

1. 在 `WWIIHexV0/Commands/Command.swift` 增加 `auditDisplayName`。
2. 在 `WWIIHexV0/Agents/AgentDecisionRecord.swift` 的 summary builder 中改用 `auditDisplayName`。
3. 扩展 `commandDisplayNameForDisplay`，兼容新旧记录。
4. 在 `WWIIHexV0/Agents/ZoneCommanderAgent.swift` 用 `commandDisplayNameForDisplay == "固守命令"` 判断最近静态防御。
5. 在 `WWIIHexV0/Agents/AgentCommandMapper.swift` 收口缺目标错误文案，不插入 raw `divisionId`。
6. 更新 `md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 验收标准

- 新生成的 AI 命令结果记录只保存“进军命令 / 交战命令 / 固守命令 / 修缮道路命令”等安全审计名。
- 旧记录仍能通过 `commandDisplayNameForDisplay` 显示为中文命令类别。
- 最近静态防御识别同时适配新安全记录和旧 raw/英文记录。
- Legacy Agent D 缺目标错误不再直出 raw 军队 id。
