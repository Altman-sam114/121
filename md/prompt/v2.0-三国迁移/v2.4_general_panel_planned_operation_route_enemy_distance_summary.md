# v2.4 武将面板计划军令源目与敌距摘要提示词

## 1. 目标

继续强化玩家宏观军令中的武将、道路和交战可读性：在 `GeneralCommandPanelView` 的计划军令列表里，把上一轮的目标名摘要扩展为源→目标路线，并追加源/目标代表 hex 到最近敌对军队的距离，方便玩家判断这条军令从哪里出发、往哪里压、距离接敌还有多远。

## 2. 范围

- 仅修改只读 UI 派生与展示：
  - `AppContainer.selectedGeneralPlannedOperationRows`
  - `GeneralCommandPanelView`
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改 `PlayerPlannedOperation` Codable schema。

## 3. 实现要求

- 进攻计划摘要优先显示 `源郡县→目标郡县`。
- 防御计划摘要显示源郡县或防区名。
- 保留已有武将、指令类型、最终战术、官道状态和源/目标受压摘要。
- 新增只读最近敌距：
  - 有源和目标时显示 `敌距 源N/目M`。
  - 只有源点时显示 `近敌 N`。
- 最近敌距只统计 `Faction.isHostile(to:)` 判定为敌对且未毁灭的军队。
- 敌距是当前静态代表 hex 到敌军坐标的距离，不代表完整行军路径、战后胜率、移动后同回合攻击或敌方回合反制。

## 4. 禁止项

- 不改 `MovementRules`、`CombatRules`、`GeneralInfluence`、`WarCommandExecutor`、`CommandValidator`、`RuleEngine`。
- 不改变真实移动成本、道路加成、敌控区、攻击、反击、撤退、补给、围城或粮道规则。
- 不让 SwiftUI View 直接推导道路、敌军压迫或最近敌距；这些只读派生放在 `AppContainer`。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 武将面板计划军令列表能显示源→目标和最近敌距。
- 旧的武将、战术、官道与受压摘要仍存在。
- 规则执行、状态 schema、真实道路/交战/补给逻辑不变。
- 文档明确敌距摘要是只读代表 hex 距离，不是路径或胜率模拟。
