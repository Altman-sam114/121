# v2.4 君主姿态塑形兼容层

本文记录 v2.4 三国迁移中“君主姿态塑形”这一小步实现提示词与交付边界。执行前仍必须阅读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和总提示词 `md/prompt/v2.0-三国迁移/codex-v2.0-三国aiagent迁移总提示词.md`。

## 目标

- 把既有 `RulerAgent` 接入当前 AI 回合主链路，作为 `DirectiveEnvelope` 执行前的君主姿态塑形层。
- 让 `.marshalDirective` 和显式 `.zoneDirective` 在进入 `WarCommandExecutor` 前都经过 `RulerAgent.adjust`。
- 将君主姿态、优先防区、目标 region 和理由写入 `DiplomacyState.rulerRecords`，并追加 diplomacy 日志。
- 保持执行权威不变：最终单位命令仍必须由 `WarCommandExecutor -> RuleEngine` 产生和执行。

## 非目标

- 不新增完整君主 / 军师 / 太守 / 武将 / 外交 Agent 分层。
- 不新增真实 LLM、网络调用、模型路径或 API key。
- 不改多势力 turn order，不让 `cao/yuan/liuBei/sun/...` 进入完整可行动循环。
- 不改 hex、region、theater、front zone 的权威边界。
- 不恢复 Cabinet / Minister / StrategicDirective 旧污染管线。
- 不跑本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构依据

当前 AI 主链路：

```text
MarshalAgent
  -> TheaterDirective JSON
  -> TheaterDirectiveDecoder
  -> TheaterDirectiveCompiler
  -> DirectiveEnvelope / ZoneDirective
  -> RulerAgent.adjust
  -> adjusted DirectiveEnvelope / ZoneDirective
  -> WarCommandExecutor
  -> RuleEngine
```

显式 `.zoneDirective` 路径：

```text
TheaterCommanderPool / ZoneCommanderAgent
  -> DirectiveEnvelope / ZoneDirective
  -> RulerAgent.adjust
  -> WarCommandExecutor
  -> RuleEngine
```

关键边界：

- `RulerAgent` 只能读取 `GameState` 派生 `RulerStrategicSnapshot`。
- `RulerAgent` 只能调整 `ZoneDirective` 的姿态参数和 `DirectiveEnvelope.theaterContext`。
- `RulerAgent` 可以写 `RulerDecisionRecord` 到 `DiplomacyState`，但不能直接修改地图、军队、资源、动态战区或部署归属。
- `TurnManager.executeDirectiveEnvelope` 仍是 directive 执行统一入口。

## 实现步骤

1. 在 `TurnManager` 中加入可注入的 `RulerAgent?`，默认使用 `RulerAgent.automatic(for:in:)`。
2. 在 `runMarshalDirectiveTurn` 中，元帅编译出 `DirectiveEnvelope` 后调用 `applyRulerAdjustment`，把调整后的 envelope 传给 `executeDirectiveEnvelope`。
3. 在 `runDirectiveTurn` 中，对 `TheaterCommanderPool` 生成的 envelope 同样调用 `applyRulerAdjustment`。
4. `applyRulerAdjustment` 负责：
   - 调用 `ruler.adjust(envelope:in:)`。
   - 复制 `GameState` 并追加 `DiplomacyState.rulerRecords`。
   - 追加一条 diplomacy 事件日志。
   - 返回调整后的 `DirectiveEnvelope` 与诊断文本。
5. 将 `RulerStrategicPosture.displayName` 和可见 rationale / diagnostics 调整为中文口径。
6. 更新 `README.md`、`AGENTS.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 验收标准

- `.marshalDirective` 与显式 `.zoneDirective` 均会在执行前经过 `RulerAgent.adjust`。
- `RulerDecisionRecord` 会写入 `DiplomacyState.rulerRecords`，并能通过既有 UI / 审计路径读取。
- `WarCommandExecutor` 和 `RuleEngine` 仍是唯一执行权威。
- Legacy Agent D 仍仅在 `.legacyAgentOrder` 显式分支运行。
- 文档不再把君主层描述为“未接入主链路”。

## 风险

- 本轮未做运行时 AI 回合烟测，真实行为正确性等待云端 CI 和后续 Agent C artifact 复判。
- 君主姿态当前只做 deterministic envelope shaping，不等于完整 v2.4 多 Agent 宫廷系统。
- `.defensive` 等姿态对具体战术的影响仍是最小兼容层，后续若要加入军师/武将细分 schema 需要单独版本迁移。
