# v2.4 命令结果与拒绝原因中文化兼容层

本文记录“命令执行结果和拒绝原因仍暴露英文 rawValue”的小步实现提示词与交付边界。执行前仍必须阅读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和总提示词 `md/prompt/v2.0-三国迁移/codex-v2.0-三国aiagent迁移总提示词.md`。

## 目标

- 沿续“玩家可见中文优先”和“注重武将、道路、交战”的 v2.4 方向，中文化命令结果、规则拒绝原因和 AI 指令诊断。
- 为 `CommandValidationError` 增加中文展示层，保留 rawValue / Codable 兼容。
- 让 `RuleEngine` 的 `CommandResult.message` 输出“命令已执行 / 命令被拒绝”的中文文案。
- 让 `AgentDecisionRecord`、`TurnManager` 和 `WarCommandExecutor` 记录中文拒绝原因，避免 AI 面板、事件日志和 `WarDirectiveRecord` 暴露 `destinationOccupied`、`wrongPhase` 等枚举名。

## 非目标

- 不改 `CommandValidator` 的校验条件和枚举 rawValue。
- 不改移动、交战、外交、生产、动态方面推进或 fallback hold 的规则行为。
- 不一次性清除所有历史英文文案、旧 prompt、测试基线、region id、zone id 或 JSON id。
- 不跑本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构边界

```text
Command / ZoneDirective
  -> CommandValidator
  -> CommandValidationError.displayName
  -> RuleEngine / WarCommandExecutor / TurnManager
  -> CommandResultSummary / WarDirectiveRecord / EventLog / AgentPanelView
```

- 展示层中文不改变校验枚举身份，既有测试仍可断言 `CommandValidationError` case。
- `CommandResultSummary.errors` 保存中文展示值，供 AI 面板直接显示。
- `WarCommandExecutor.run` 只改变拒绝日志文本；fallback 判断仍使用原校验结果。

## 验收标准

- `CommandResult.message` 成功和失败均为中文。
- `CommandValidationError` rawValue 保持不变，中文只作为 display 层。
- AI 面板命令结果和 `WarDirectiveRecord.diagnostics` 不再展示常见校验错误 rawValue。
- 玩家和 AI 命令仍走 `Command / ZoneDirective -> WarCommandExecutor / RuleEngine`，没有绕过规则系统。

## 风险

- 本轮不是全项目文案清零；region id、zone id、JSON id、旧测试、历史 prompt 和部分调试文本仍可能保留英文。
- 未做运行时 UI 烟测，AI 面板实际换行和日志宽度仍待云端 CI、后续 Agent C artifact 复判或人工授权运行检查。
