# v2.4 MockAI 可见文本三国化

## 目标

将 Legacy `MockAIClient` 输出到 `AgentDecisionRecord`、AI 面板、调试 JSON 和命令结果里的 intent / reason 文案改为三国、官道、粮草、器械和防区部署语义；不改目标选择、stance 字符串、排序、评分或 JSON schema。

## 范围

- `WWIIHexV0/Agents/MockAIClient.swift`
- 同步 `README.md`、`md/flow/flow.md`、`md/prompt/README.md` 和 `update_log.md`

## 禁止项

- 不改 `objective = context.objectives.first { $0.name == "Bastogne" }` 的 fallback 目标选择。
- 不改 `stance` 字符串，例如 `fireSupport`、`breakthrough`、`roadAdvance`、`frontAttack`、`depthReserve`。
- 不改 `orderPriority`、`attackScore`、`isArmor`、`isArtillery` 或任何命令生成算法。
- 不改 `AgentDecisionEnvelope`、`AgentOrder`、Codable schema、parser、mapper 或执行链。
- 不跑本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构依据

- Legacy Agent D / MockAI 只作为兼容和回归参考路径保留，默认战争 AI 主路径不得退回旧管线。
- 旧算法中的 Bastogne / armor / artillery 仍影响目标、排序和评分；本轮只处理玩家可见文本，避免把文案迁移误做成行为迁移。
- `SupplyState.displayName` 已提供粮道通畅、粮草紧张、粮道断绝等中文展示，可用于 reason 文本。

## 实现步骤

1. 将顶部注释中的 Bastogne 推进描述改成 legacy objective / 官道推进语义。
2. 将普通 resupply / move / hold 的 reason 改成粮草、官道、可见机会等中文文案。
3. 将普通 intent 改成沿官道集中机动力量、等待器械支援、突破要地等三国语义。
4. 将 FrontZone deployment 兼容路径的 reason / intent 改成防区、前线、纵深预备队、守备军等中文文案。
5. 将 `attackReason` 的器械攻城和普通攻击原因改成中文。
6. 更新 README、flow、prompt 索引和 update log，明确算法和 schema 未变化。

## 验收标准

- `MockAIClient` 可见 intent / reason 不再输出 Bastogne、armor、artillery 或英文部署说明。
- Bastogne fallback 目标选择、stance 字符串、排序、评分和 Codable schema 保持不变。
- 本轮只改展示文本，规则执行链仍是 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine`。

## 遗留风险

- `MockAIClient` 算法仍按旧 fallback 目标和 armor/artillery 语义排序/评分；后续若要迁移行为，应单独做 `v2.4_mockai_behavior_sanguo_compatibility.md`。
