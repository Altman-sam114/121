# v2.4 上游 Agent fallback 显示名三国化兼容层

## 目标

减少 AI 面板、审计记录和兼容 fallback 路径中的二战英文指挥者残留，让君主、军师、武将/防区指挥者的默认显示名使用当前 `Faction.displayName` / `shortDisplayName` 的三国语义。

## 范围

- `WWIIHexV0/Agents/RulerAgent.swift`
- `WWIIHexV0/Agents/ZoneCommanderAgent.swift`
- `WWIIHexV0/Agents/GameAgent.swift`
- `WWIIHexV0/App/AppContainer.swift`
- 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 和 `md/prompt/README.md`

## 禁止项

- 不改 agent id、Codable rawValue、JSON schema、战术选择、命令执行或 AI 规则。
- 不删除旧 `guderian` 兼容数据、不改旧阿登 fallback、不改测试 fixture。
- 不改 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine` 执行链。
- 不跑本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构依据

- `.germany` / `.allies` 仍作为兼容 rawValue 存在，但 `Faction.displayName` / `shortDisplayName` 已映射到曹操势力 / 袁绍势力和曹军 / 袁军。
- `RulerAgent.automatic`、`MarshalAgentConfig.automatic`、`TheaterCommanderPool.automatic` 和 `AppContainer.buildCommanderPool` 是缺 registry 或兼容路径时的默认上游 Agent 命名来源。
- 显示名变化只影响 AI 面板、日志和审计可读性，不改变决策、校验、移动、交战、道路或占领规则。

## 实现步骤

1. 将 `.germany` / `.allies` 的 `RulerAgentConfig.name` fallback 改为基于 `Faction.displayName` 的君主/军议显示名。
2. 将 `.germany` / `.allies` 的 `MarshalAgentConfig.name` fallback 改为基于 `Faction.shortDisplayName` 的军师显示名，并把 personality 文案改为中文道路/战线/预备队语义。
3. 将自动防区 commander 名称改为 `短势力名 + 防区指挥官`。
4. 将 `AgentRole.displayName` 改为君主、军师、武将。
5. 将 `AppContainer` 兼容 mock / observer agent 显示名改为中文势力语义。

## 验收标准

- 上游自动 Agent fallback 不再显示 German Ruler、Allied Supreme Council、Rundstedt、Eisenhower 或 Allied Mock Commander。
- 自动防区指挥者显示名使用曹军/袁军等三国短势力名。
- Agent role 展示为君主、军师、武将。
- 本轮只改显示层，不影响 AI 决策和规则执行。

## 遗留风险

- `general_agents.json`、`GameAgent.guderianFallback` 和 `MockAIClient` 仍保留旧兼容 MockAI / 阿登回归参考语义；后续可另拆数据与 fallback 行为迁移。
