# v2.5 缺失数据资源错误展示安全补丁

## 目标

继续推进三国迁移的玩家/编辑器可见文案收口，避免 `DataLoaderError.missingResource` 直接把 `ardennes_v0_scenario.json`、`ardennes_v02_regions.json` 等 raw 资源名拼入错误说明。

## 范围

- 只修改 `WWIIHexV0/Data/ScenarioDefinition.swift` 中 `DataLoaderError.description` 的展示文案。
- 保留 `DataLoaderError.missingResource(String)` payload，保留所有资源名、加载函数、fallback 和调用逻辑。
- 常见资源名映射为中文安全类别：官渡剧本数据、官渡郡县数据、兼容剧本数据、兼容郡县数据、三国军队模板、兼容军队模板、地形规则数据、兼容武将配置。
- 未知资源名显示为“指定数据文件”，不直接暴露 raw 文件名。
- 同步 `md/prompt/README.md` 与 `update_log.md`。

## 禁止项

- 不修改 `DataLoaderError` enum case、resourceName 参数、`DataLoader.resourceURL(...)`、`loadArdennes*` API、旧阿登资源名或 JSON 文件。
- 不修改 MapEditor 的 `missingResource(URL)`，不处理 fallback `unit_templates.json` 的 displayName。
- 不修改规则、AI、命令管线、测试文件或 Xcode project。
- 不运行 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构依据

- 默认新局优先加载官渡数据，旧阿登资源仍是兼容 fallback；本轮不能删除或重命名旧资源。
- `DataLoaderError.description` / `errorDescription` 可能通过加载失败、编辑器验收或调试面板被用户看到，属于展示层文案。
- 目标是隐藏旧题材 raw 文件名，同时保留错误定位到“哪一类数据缺失”的基本诊断价值。

## 验收标准

- `DataLoaderError.description` 不再包含英文 `Missing data resource`。
- `ScenarioDefinition.swift` 不再在缺资源错误展示中直接拼出旧阿登 raw 资源名。
- enum case、资源名、fallback、加载逻辑和调用方不变。
