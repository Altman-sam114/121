# v2.4 武将技能道路与交战效果提示兼容层提示词

## 1. 目标

继续强化“武将、道路、交战”的可读链路：武将技能已经能从 raw id 显示为中文标签，本轮在不改变任何规则的前提下，为已有技能标签补充极短的道路/交战效果提示，让玩家在武将军令面板、武将档案和军队详情“所属武将”摘要中能看出“这个技能大致影响官道机动、骑兵攻势、攻城、守备或渡河防御”。

## 2. 范围

- 修改 `WWIIHexV0/Agents/GeneralRegistry.swift` 的 `GeneralSkillDisplay` 展示 helper。
- 修改以下 UI 只读展示：
  - `WWIIHexV0/UI/GeneralCommandPanelView.swift`
  - `WWIIHexV0/UI/GeneralProfileView.swift`
  - `WWIIHexV0/UI/UnitInspectorView.swift`
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`

## 3. 实现要求

- 新增 UI-only helper，例如 `GeneralSkillDisplay.effectHint(for:)` 和 `displayNameWithHint(for:)`。
- `displayName(for:)` 继续只返回纯中文技能标签，避免影响 `GeneralAgent` rationale、日志和其他需要短标签的旧入口。
- 效果提示必须短，优先使用“官道机动”“骑兵突击”“攻城修正”“地形防御”“渡河防御”等玩家能理解的词。
- 未登记的新技能继续只显示 fallback 技能名，不编造规则效果。
- 武将档案技能卡片可把技能名和效果提示分行显示；军队详情和武将军令面板可用紧凑的“技能名：效果”拼接。

## 4. 禁止项

- 不修改 `GeneralInfluence`、`MovementRules`、`CombatRules`、`CommandExecutor`、`WarCommandExecutor`、`RuleEngine` 或真实道路/交战计算。
- 不修改 `GeneralData.skills`、`GeneralAssignment.skills`、JSON schema、Codable 字段、raw skill id 或排序。
- 不新增完整技能树、士气、疲劳、单挑、武将成长、胜率或敌方回合模拟。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 武将军令面板、武将档案和军队详情所属武将摘要能看到技能中文名及对应的短效果提示。
- `GeneralAgent` rationale、规则匹配、道路机动、攻防修正、攻击/反击日志和命令执行链路保持不变。
- 文档明确该提示只是只读解释层，不代表完整技能详情、胜率、士气、单挑或成长系统。
