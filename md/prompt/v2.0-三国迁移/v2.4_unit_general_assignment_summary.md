# v2.4 军队所属武将摘要提示词

> 本阶段承接 `v2.4_general_skill_display_labels.md`、`v2.4_unit_road_mobility_preview.md`、`v2.4_unit_combat_target_comparison.md` 和 `v2.4_region_road_summary.md`。执行前必须先读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和相关源码。

## 目标

- 继续强化“三国要注重武将”，让玩家选中单支军队时能直接看到该军队所属武将。
- 在军队详情中展示武将姓名、风格、忠诚/满意、主要技能和玩家干预次数，帮助解释后续道路机动与交战修正来自谁。
- 保持该信息为只读 UI 派生，不改变武将分配、战术塑形、忠诚满意、移动或交战规则。

## 范围

- 主要文件：
  - `WWIIHexV0/SpriteKit/MapDisplayAdapter.swift`
  - `WWIIHexV0/UI/UnitInspectorView.swift`
- 文档文件：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改 `GeneralInfluence`、`GeneralAgent`、`GeneralDispatcher`、`WarCommandExecutor`、`RuleEngine`、JSON schema 或 Xcode project。

## 实现要求

- 在 `UnitInspectorStrategicState` 增加 `GeneralAssignment?` 只读快照。
- `MapDisplayAdapter.unitInspectorState` 取武将分配时先找显式包含该军队 id 的 `FrontZone.generalAssignment`，再 fallback 到当前 hex 所属防区的 `generalAssignment`。
- `UnitInspectorView` 在“军队详情”中显示“所属武将”摘要；技能使用 `GeneralSkillDisplay.displayName` 中文化，最多显示前三个技能，避免面板过长。
- 未找到武将分配时显示“武将：未任命”，不制造默认武将或写入状态。

## 禁止项

- 不允许 UI 直接任免武将、修改忠诚满意、改变 `FrontZone.generalAssignment` 或写入 `GameState`。
- 不允许改变武将道路机动、攻防修正、战术塑形、移动、交战或命令执行规则。
- 不允许把摘要写入 `eventLog`、`WarDirectiveRecord` 或任何持久状态。
- 不允许新增本机 Xcode build/test、模拟器、Probe、Smoke、Stage Regression、Dynamic Theater Regression、Full 或性能测试。
