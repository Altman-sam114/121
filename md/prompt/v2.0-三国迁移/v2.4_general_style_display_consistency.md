# v2.4 武将风格展示一致性提示词

## 目标

继续推进武将相关迁移，把武将统军风格 `aggressive/balanced/cautious` 的中文展示收束到共享入口，避免武将军令面板、武将档案、军队详情和军机面板各自维护一套重复 switch；同时把可进入军机面板摘要/诊断的 `zone directive(s)` / `theater directive(s)` 等英文短语中文化。

## 范围

- `ZoneCommanderAgentConfig.CommandStyle` 提供共享 `displayName`。
- `GeneralAssignment` 提供 `resolvedCommandStyle` / `commandStyleDisplayName`，让 assignment 快照缺失或坏值时按忠诚/满意推断出的风格与 `GeneralAgent` 执行口径一致。
- `GeneralAgent`、`AgentPanelView`、`GeneralCommandPanelView`、`GeneralProfileView`、`UnitInspectorView` 改为复用共享展示入口。
- `GeneralAgent` 生成 rationale、record `generalName` 和战术检查 config 时优先使用 `GeneralData.localizedName` / `GeneralAssignment.generalDisplayName`，缺名时显示“未命名武将”，不把 raw `generalId` 当作玩家文案。
- `TheaterCommanderPool`、模拟元帅和元帅编译失败 fallback 的玩家可见 context / diagnostics 改为中文。

## 非目标

- 不改变 `CommandStyle` rawValue、Codable、JSON schema 或旧存档兼容。
- 不改变 `GeneralAgent` 战术塑形规则、忠诚/满意阈值、道路/交战数值或执行命令。
- 不改 `WarCommandExecutor`、`RuleEngine`、地图、数据文件或 Xcode project。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Full。

## 架构依据

- 武将层只塑形 `DirectiveEnvelope` / `ZoneDirective` 并写审计记录，不直接修改 `GameState`。
- UI 只读取展示字段或共享展示 helper，不重新解释 raw 值。
- 玩家、AI、聊天命令仍必须落到 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine`。

## 轻量检查

- `swiftc -parse` 针对本轮改动 Swift 文件。
- 扫描重复风格显示 switch 和英文 `zone directive(s)` / `theater directive(s)` / fallback diagnostics。
- 改动文件尾随空白和冲突标记扫描。
- prompt 索引差集检查。
- `git diff --check`。

## 验收标准

- 武将风格展示只通过共享入口输出“进取 / 持重 / 谨慎”。
- assignment 快照风格缺失或坏值时，军队详情显示的风格与 `GeneralAgent` 使用的推断风格一致。
- 武将 rationale、record 显示名和战术检查 config 不再把 raw `generalId` 作为玩家可见 fallback。
- 军机面板可见的防区/方面指令摘要和 fallback diagnostics 不再使用英文 `directive(s)` / `fallback used` 口径。
- 本轮无规则行为变更，重验证交给 `origin/main` 的 GitHub Actions。
