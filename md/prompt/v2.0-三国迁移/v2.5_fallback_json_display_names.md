# v2.5 fallback JSON 顶层展示名三国化提示词

## 目标

继续推进 v2.5 玩家/编辑器可见文案收口，把旧 fallback JSON 的顶层 `displayName` 从 MapEditor 英文默认名改为中文兼容数据展示名，减少缺省/回退路径中暴露的旧编辑器文案。

## 范围

- 修改 `WWIIHexV0/Data/ardennes_v0_scenario.json` 顶层 `displayName`。
- 修改 `WWIIHexV0/Data/ardennes_v02_regions.json` 顶层 `displayName`。
- 同步 `WWIIHexV0/Tests/ScenarioDataTests.swift` 中对 scenario 顶层 `displayName` 的等值断言。
- 更新 `md/prompt/README.md` 和 `update_log.md`。

## 非目标

- 不改 `id`、faction rawValue、unit id、objective id、region id、template id、schema 或 Codable 字段。
- 不改 `City ...`、`Supply ...`、region name、keyLocations、tile cityName、objectives name 或 supply source 名称。
- 不改默认官渡 JSON、DataLoader 加载顺序、MapEditor export schema、规则、AI、UI 布局或命令管线。
- 不跑本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构依据

- 默认新局优先加载 `guandu_200_scenario.json` 和 `guandu_200_regions.json`；旧阿登 JSON 只作为兼容 fallback。
- `ScenarioDefinition.displayName` 在 fallback 加载成功日志中可见。
- `RegionDataSet.displayName` 是 Codable 必填字段，但当前规则链路不依赖它做判断。
- 现有测试只对 scenario 顶层 `displayName` 有 exact 断言；region 顶层名没有 exact 断言。

## 实现步骤

1. 将 `ardennes_v0_scenario.json` 顶层 `displayName` 改为“兼容测试剧本”。
2. 将 `ardennes_v02_regions.json` 顶层 `displayName` 改为“兼容测试郡县数据”。
3. 将 `ScenarioDataTests` 中对应断言改为“兼容测试剧本”。
4. 更新 prompt 索引和 `update_log.md`。
