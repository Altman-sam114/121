# v2.1 中立势力兼容基础实现记录

完成日期：2026-07-04

## 1. 本轮目标

本轮续接 `codex-v2.0-三国aiagent迁移总提示词.md` 的 v2.1 方向，先处理风险最小但阻塞后续多势力的基础问题：

- 让数据层可以表达中立归属。
- 修复 `RegionDataSet` owner/controller 缺省或 null 时错误回退到 `.allies` 的旧问题。
- 把规则和 AI 摘要中的核心敌对判断从二元 `.opponent` 迁到 `Faction.isHostile(to:)`。
- 保持当前默认曹/袁兼容双方的回合、AI、经济和旧 JSON 行为稳定。

## 2. 非目标

本轮明确不做：

- 不一次性扩展到曹、袁、刘、孙、刘表、马腾、汉室等完整三国势力。
- 不改默认回合循环；当前仍是 `.germany -> .allies -> .germany` 的兼容节奏。
- 不迁移默认阿登 JSON 到官渡剧本。
- 不修改 Xcode project。
- 不跑 Xcode build/test、模拟器、Probe、Smoke、Stage Regression 或 Full Regression。

## 3. 实现摘要

代码变化：

- `Faction` 新增 `.neutral`，用于 owner/controller 等数据归属；自定义 `allCases` 仍只返回当前可行动的 `.germany`、`.allies`，避免中立进入默认经济初始化、MapEditor 默认枚举和回合参与者列表。
- `Faction.opponent` 保留为旧兼容 helper，但规则/AI 新代码不再依赖它；新增 `isNeutral` 和 `isHostile(to:)`。
- `RegionDataSet.toRegions()` 将缺省或 null 的 owner/controller 映射到 `.neutral`，不再 fallback 到 `.allies`。
- `CommandExecutor`、`AppContainer`、`TurnManager`、`VictoryState` 补齐 `.neutral` 分支：中立不主动行动、不触发 AI、不计入当前二元胜负统计。
- `SupplyRules`、`RegionSupplyRules`、`RegionCombatRules`、`WarCommandExecutor`、`AgentContexts`、`ZoneCommanderAgent`、`FrontLineManager` 改用 `isHostile(to:)` 判断敌对控制、敌军、敌方目标或前线对手。
- `TerrainStyle` 和 `MapEditorView` 增加中立显示 fallback。

文档变化：

- `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 同步 v2.1 中立兼容基础口径。
- `update_log.md` 追加本轮 v2.1 小步记录。

## 4. 兼容边界

- `HexTile.controller` 仍可为 nil 表示中立 hex；`RegionNode.owner/controller` 是非 optional `Faction`，用 `.neutral` 表示中立 region。
- `Faction.neutral` 当前不是 playable power，也不是完整外交国家。
- `Faction.allCases` 暂不包含 `.neutral`，这是为了避免旧二元回合和经济初始化被中立插入；后续完整多势力迁移时必须重新设计 turn order / player faction / AI control 列表。
- `Faction.isHostile(to:)` 当前是轻量规则：双方不同且都不是中立即敌对。它不是最终外交系统；后续仍需接 `DiplomacyState` / `PowerRelation`。

## 5. 风险与后续

- 这一步只修复中立基础和敌对判断口径，仍不代表 v2.1 完整多势力完成。
- 部分旧测试、历史文档和默认数据仍含 germany/allies/Ardennes/Bastogne，这是兼容层历史语义，不等同于发布 UI 已完成三国化。
- 后续 v2.1 应继续设计完整可行动势力列表、外交关系、turn order、玩家选择势力和 AI 控制策略。
- 后续 v2.2 仍需新增 `guandu_200_scenario.json` / `guandu_200_regions.json` 并切换默认加载入口。
