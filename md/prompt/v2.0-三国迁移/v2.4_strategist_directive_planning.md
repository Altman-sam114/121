# v2.4 军师目标编排兼容层

本文记录 v2.4 三国迁移中“军师目标编排”这一小步实现提示词与交付边界。执行前仍必须阅读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和总提示词 `md/prompt/v2.0-三国迁移/codex-v2.0-三国aiagent迁移总提示词.md`。

## 目标

- 在既有 `RulerAgent.adjust` 之后、`WarCommandExecutor` 之前加入 deterministic `StrategistAgent.plan`。
- 让 `.marshalDirective` 和显式 `.zoneDirective` 共享同一段军师目标编排。
- 记录军师选择的主防区、目标 region、支援 region、承接君主姿态和 rationale，写入 `GameState.strategistRecords`。
- 在 AI 面板展示军师记录，让 AI 回合可解释“君主想要什么、军师选哪里、执行器做了什么”。

## 非目标

- 不新增完整太守、武将或外交 Agent。
- 不新增真实 LLM、网络调用、模型路径或 API key。
- 不改多势力 turn order，不让新增三国势力进入完整可行动循环。
- 不改 hex、region、theater、front zone 的权威边界。
- 不让军师层生成底层 `Command`，不绕过 `WarCommandExecutor -> RuleEngine`。
- 不跑本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构依据

当前 AI 主链路：

```text
MarshalAgent / TheaterCommanderPool
  -> DirectiveEnvelope / ZoneDirective
  -> RulerAgent.adjust
  -> ruler-adjusted DirectiveEnvelope
  -> StrategistAgent.plan
  -> strategist-adjusted DirectiveEnvelope
  -> WarCommandExecutor
  -> RuleEngine
```

关键边界：

- `StrategistAgent` 只能读取 `GameState`、`RulerDecisionRecord` 和 `DirectiveEnvelope`。
- `StrategistAgent` 只能调整 `ZoneDirective` 的目标 region、focus/support/convergence、强度倾向和 `DirectiveEnvelope.theaterContext`。
- `StrategistAgent` 可以写 `StrategistDecisionRecord` 到 `GameState.strategistRecords`，但不能直接修改地图、军队、资源、动态战区或部署归属。
- `TurnManager.executeDirectiveEnvelope` 仍是 directive 执行统一入口。

## 实现步骤

1. 新增 `StrategistDecisionRecord`，并在 `GameState` 中追加兼容字段 `strategistRecords`。
2. 新增 `StrategistAgentConfig`、`StrategistAgent`、`StrategistDirectiveAdjustment` 和 deterministic snapshot。
3. 在 `TurnManager` 中加入可注入的 `StrategistAgent?`。
4. 在 `runMarshalDirectiveTurn` 和 `runDirectiveTurn` 中，先调用 `RulerAgent.adjust`，再调用 `StrategistAgent.plan`，最后执行调整后的 envelope。
5. `applyStrategistPlanning` 负责：
   - 调用 `strategist.plan(envelope:in:rulerRecord:)`。
   - 复制 `GameState` 并追加 `GameState.strategistRecords`。
   - 追加一条事件日志。
   - 返回调整后的 `DirectiveEnvelope` 与诊断文本。
6. `AgentPanelView` 展示 `StrategistDecisionRecord`。
7. 更新 Xcode project、README、`md/flow/*`、`md/prompt/README.md` 和 `update_log.md`。

## 验收标准

- `.marshalDirective` 与显式 `.zoneDirective` 均会在执行前经过 `RulerAgent.adjust` 和 `StrategistAgent.plan`。
- `StrategistDecisionRecord` 会写入 `GameState.strategistRecords`，并在 AI 面板展示。
- `WarCommandExecutor` 和 `RuleEngine` 仍是唯一执行权威。
- Legacy Agent D 仍仅在 `.legacyAgentOrder` 显式分支运行。
- 文档和流程图明确军师层只做 envelope shaping，不直接修改战术状态。

## 风险

- 本轮未做运行时 AI 回合烟测，真实行为正确性等待云端 CI 和后续 Agent C artifact 复判。
- 军师层当前仍是 deterministic 兼容层，不等于完整 v2.4 多 Agent 宫廷系统。
- 太守、武将、外交 Agent 仍需后续单独版本推进。
