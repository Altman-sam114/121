# v2.4 军队接战风险预判兼容层

## 1. 目标

在当前军队详情的“接战预判”基础上，继续强化玩家战前判断：当选中军队射程内存在敌军时，预判行不仅显示预计伤害、敌我剩余兵力和反击风险，也显示可能触发的撤退、死守额外损失、断粮撤退额外损失和歼灭风险。

本轮回应人工强调的“三国要注重武将，做好道路、交战”：交战预判必须更接近真实战斗结算日志，让玩家能在武将、道路、兵力和粮道状态之间建立因果判断。

## 2. 范围

- 修改 `WWIIHexV0/App/AppContainer.swift` 的 `selectedUnitCombatPreviewNotes` 相关私有辅助函数。
- 继续复用现有 `CombatRules.attackDamage` / `counterAttackDamage` 的伤害和 lossRatio。
- 预判只读估算 `CommandExecutor.resolveCombatResult` 当前结算口径：
  - 基础兵力伤害。
  - `RetreatMode.hold` 时约 20% 额外损失。
  - `RetreatMode.retreatable` 且损失比例达到阈值时可能撤退。
  - 粮道断绝且触发撤退时约 50% 额外损失。
  - 结算后兵力归零时提示可能歼灭。
- 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 3. 非目标和禁止项

- 不修改 `CombatRules` 的真实伤害公式。
- 不修改 `CommandExecutor` 的真实攻击、反击、撤退、死守、歼灭执行逻辑。
- 不修改 `SupplyRules.resolveRetreat`、撤退路径、撤退失败惩罚或补给规则。
- 不新增士气、疲劳、单挑、概率胜率、AI 推荐解释或跨回合战斗规划。
- 不让 UI 预判写入 `GameState`、事件日志、审计记录或持久数据。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 4. 架构依据

- Hex 仍是战术权威，单位位置与攻击距离来自 `Division.coord` 和 `HexCoord.distance`。
- 玩家和 AI 的真实攻击必须走 `Command -> RuleEngine -> CommandExecutor`。
- `UnitInspectorView` 只读展示 `AppContainer.selectedUnitCombatPreviewNotes`，不能绕过规则系统。
- `CommandExecutor` 的真实结算日志已经显示结算后剩余兵力，本轮预判应向同一口径靠拢，但必须使用“可能/预估”类措辞。

## 5. 实现步骤

1. 在 `AppContainer` 中为接战预判增加私有 `CombatPreviewOutcome`。
2. 新增只读估算函数，按当前结算顺序从 `Division.strength/maxStrength`、`RetreatMode`、`SupplyState` 和 `CombatDamage.lossRatio` 计算预估剩余兵力、额外损失、撤退和歼灭风险。
3. 将目标预判行从“基础伤害后剩余兵力”升级为“结算预估后剩余兵力”。
4. 当目标预估撤退或歼灭时，把反击文字降为“预计无反击”，避免继续展示误导性的反击伤害。
5. 若目标仍可能反击，对我方反击后剩余兵力也使用同一套风险估算，并在需要时提示我方可能撤退或被歼。
6. 更新核心文档、流程图、阶段索引和版本日志。

## 8. 验收标准

- 选中军队的接战预判最多三目标仍按预计伤害、反击伤害、距离和名称排序。
- 每个目标行显示预估结算后的敌方剩余兵力。
- 死守、断粮撤退、可能撤退和可能歼灭能在目标行中以中文风险片段显示。
- 目标预估撤退或歼灭时，反击提示不再误导玩家一定会被反击。
- 真实攻击、反击、撤退、死守、歼灭、日志和状态写入行为不变。
- 轻量检查通过；重测试交给云端 CI 或后续 Agent C artifact 复判。

## 9. 风险

- 本轮没有运行 UI 烟测，新增长文本在窄屏、长军队名和 Dynamic Type 下的实际换行仍需后续检查。
- 预判不计算撤退目的地和撤退失败惩罚，因此“可能撤退”不是完整撤退路径预测。
- 预判不包含后续士气、疲劳、单挑、天气或多回合围攻概率；这些应在 v2.5/v2.6 另行设计。
