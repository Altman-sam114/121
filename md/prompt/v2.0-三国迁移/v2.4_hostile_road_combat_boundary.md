# v2.4 道路与交战敌对边界兼容层

本文记录“三国要注重武将，做好道路、交战”下的敌对判断收口小步。执行前仍必须阅读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和总提示词 `md/prompt/v2.0-三国迁移/codex-v2.0-三国aiagent迁移总提示词.md`。

## 目标

- 将道路敌控区、攻击目标、粮道阻断和安全补员邻接从旧二元 `faction != faction` 收口到 `Faction.isHostile(to:)`。
- 保持玩家、AI 和未来聊天命令仍通过 `Command / ZoneDirective -> CommandValidator / WarCommandExecutor -> RuleEngine` 执行。
- 为后续汉室/中立、借道和多势力外交制度保留边界：中立不应仅因阵营不同就阻断道路、粮道或成为合法攻击目标。

## 非目标

- 不实现完整外交借道、同盟通行、宣战、臣属、贡赋或多势力 turn order。
- 不改变 `Faction.isHostile(to:)` 的当前最小规则。
- 不改 `GameState` 战术权威，不新增 schema，不改 JSON 数据。
- 不跑本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构边界

```text
Faction.isHostile(to:)
  -> MovementRules.isEnemyZoneOfControl
  -> CommandValidator.validateAttack
  -> SupplyRules.canSupplyPass
  -> EconomyRules safe reinforcement adjacency
```

- 道路 ZOC 和粮道阻断只认敌对军队，避免中立/汉室被旧二元判断误伤。
- 攻击合法性仍由 `CommandValidator` 判断，合法后才进入 `CommandExecutor` 和 `CombatRules`。
- 安全补员仍由 `EconomyRules` 结算，但敌邻判断只看敌对军队。
