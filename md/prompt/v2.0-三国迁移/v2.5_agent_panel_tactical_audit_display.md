# v2.5 军机面板战术审计分组提示词

## 目标

继续推进“武将、道路、交战”体验，把已经进入 `WarDirectiveRecord.diagnostics` 的宏观军令道路审计和交战审计，在 `AgentPanelView` 的防区指令卡片中拆成独立“战术审计”行展示；同时小步优化玩家武将计划军令的官道摘要，让计划列表直观看到源点和目标是否据道、离道或受压。

## 范围

- 修改 `WWIIHexV0/UI/AgentPanelView.swift`。
- 修改 `WWIIHexV0/App/AppContainer.swift` 的私有计划军令摘要文案。
- 更新 `md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 非目标

- 不改变 `WarCommandExecutor` 道路/交战诊断生成逻辑。
- 不改变 `WarDirectiveRecord`、`PlayerPlannedOperation` 或任何 Codable schema。
- 不改变移动、道路加成、目标筛选、攻击、反击、撤退、占领、战略同步、AI 决策或规则数值。
- 不改 SpriteKit 计划线绘制。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试 / app 启动 / 全量构建。

## 架构依据

- `WarCommandExecutor` 已生成“道路审计”和“交战审计”，并通过 `TurnManager` / `AppContainer` 合并进 `WarDirectiveRecord.diagnostics`。
- `AgentPanelView` 已对诊断做玩家可见安全过滤；本轮应先按原始诊断 marker 分组，再对组内文本做安全过滤，避免 marker 被兜底文案覆盖后丢失战术审计分类。
- `AppContainer.selectedGeneralPlannedOperationRows` 已是玩家计划军令列表的唯一摘要来源，适合在私有 helper 中优化官道展示文案。

## 实现要求

1. `AgentPanelView` 先从原始 `directive.diagnostics` 中识别“道路审计”和“交战审计”。
2. 战术诊断在防区指令卡片中显示“战术审计”小标题，并按道路/交战分别使用官道和瞄准图标。
3. 战术诊断仍必须经过 `displaySafePanelText`；若被安全过滤，按类型显示“道路审计已记录”或“交战审计已记录”，不要掉回普通诊断。
4. 非战术诊断继续使用原来的安全过滤和普通诊断展示。
5. 玩家计划军令官道摘要从“官道 双道/源道/目道/无道，源受压”改为“官道：源据道/受压，目离道”这类源目分明的中文摘要。
6. 本轮不改变任何执行行为。

## 验收标准

- AI 防区指令和玩家武将宏观军令中的道路/交战诊断能在军机面板中以“战术审计”分组展示。
- 安全过滤不会让道路/交战诊断从战术分组消失。
- 计划军令列表能显示源/目标官道状态和受压状态，且不改变 `PlayerPlannedOperation` schema。
