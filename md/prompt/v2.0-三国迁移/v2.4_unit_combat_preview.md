# v2.4 军队接战预判与敌对入口过滤提示词

> 本阶段承接 `v2.4_general_influence_panel_summary.md` 和 `v2.4_general_skill_display_labels.md`。执行前必须先读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和相关源码。

## 目标

- 继续强化“武将、道路、交战”体验，让玩家在军队详情里提前看到当前选中军队的接战预判。
- 复用现有 `CombatRules.combatAuditSummary` 和 `GeneralInfluence`，展示预计伤害、反击、武将攻防影响和交战审计因素。
- 统一玩家地图点击攻击、攻击高亮和武将宏观目标选择的敌对判断，避免继续用旧二元 `!= faction` 推断敌我。

## 实现要求

- 在 `AppContainer` 增加只读派生属性，例如 `selectedUnitCombatPreviewNotes`。
- 接战预判只读取当前 `GameState`，选择当前选中军队射程内的敌对目标，输出短文本列表。
- 预判内容至少包含预计伤害和是否反击；如有武将攻防影响或交战审计因素，也要显示。
- `UnitInspectorView` 接收预判文本并以“接战预判”小节展示，保持紧凑、可换行。
- `RootGameView` 只负责把 `AppContainer` 派生属性传给 `UnitInspectorView`。
- 地图点击攻击、单位点击攻击、攻击高亮、武将宏观目标选择统一使用 `Faction.isHostile(to:)`。

## 禁止项

- 不允许 UI 直接执行攻击、移动或修改 `GameState`。
- 不允许把预判结果写入 `eventLog`、`WarDirectiveRecord` 或任何持久状态。
- 不允许改变伤害公式、反击规则、武将加成规则或道路规则。
- 不允许新增本机 Xcode build/test、模拟器、Probe、Smoke、Stage Regression、Dynamic Theater Regression、Full 或性能测试。
