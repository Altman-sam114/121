# v2.4 武将军令复核兼容层

本文记录 v2.4 三国迁移中“武将做了什么”这一小步实现提示词与交付边界。执行前仍必须阅读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和总提示词 `md/prompt/v2.0-三国迁移/codex-v2.0-三国aiagent迁移总提示词.md`。

## 目标

- 在 `StrategistAgent.plan` 之后、`WarCommandExecutor` 之前加入 deterministic `GeneralAgent.plan`。
- 让 `.marshalDirective` 和显式 `.zoneDirective` 共享同一段武将复核。
- 读取 `FrontZone.generalAssignment` 与 `GeneralRegistry`，按武将忠诚、满意度、指挥风格和防区压力收束 `ZoneDirective`。
- 记录每个防区武将对军令的动作与理由，写入 `GameState.generalRecords`，并在 AI 面板展示。

## 非目标

- 不实现完整武将技能系统、头像、美术、真实 LLM 或网络调用。
- 不新增太守、外交 Agent。
- 不改多势力 turn order。
- 不让武将层生成底层 `Command`，不绕过 `WarCommandExecutor -> RuleEngine`。
- 不跑本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构依据

当前 AI 主链路：

```text
MarshalAgent / TheaterCommanderPool
  -> DirectiveEnvelope / ZoneDirective
  -> RulerAgent.adjust
  -> StrategistAgent.plan
  -> GeneralAgent.plan
  -> WarCommandExecutor
  -> RuleEngine
```

关键边界：

- `GeneralAgent` 只能读取 `GameState`、`GeneralRegistry` 和 `DirectiveEnvelope`。
- `GeneralAgent` 只能调整 `ZoneDirective` 的投入强度、预备队和保持/收束倾向。
- `GeneralAgent` 可以写 `GeneralDecisionRecord` 到 `GameState.generalRecords`，但不能直接修改地图、军队、资源、动态战区或部署归属。

## 验收标准

- `.marshalDirective` 与显式 `.zoneDirective` 均会在执行前经过 `RulerAgent.adjust`、`StrategistAgent.plan` 和 `GeneralAgent.plan`。
- `GeneralDecisionRecord` 会写入 `GameState.generalRecords`，并在 AI 面板展示。
- `WarCommandExecutor` 和 `RuleEngine` 仍是唯一执行权威。
- 文档和流程图明确武将层只做 envelope shaping，不直接修改战术状态。
