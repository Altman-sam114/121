# v2.4 郡县检查器敌对可见军队口径提示词

## 1. 目标

继续收口三国多势力敌我判断：在 `RegionInspectorView` 的郡县态势里，将“可见敌军”从旧的“非己方军队”改为真正的敌对军队，并单独展示可见但非敌对的军队，避免中立、友好或后续借道势力被误写成敌军。

## 2. 范围

- 仅修改只读 UI 派生与展示：
  - `MapDisplayAdapter.inspectorState`
  - `RegionInspectorState`
  - `RegionInspectorView`
- 可同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改地图、外交、移动、攻击、粮道、补给或命令执行规则。

## 3. 实现要求

- `RegionInspectorState.visibleEnemyDivisions` 只统计：
  - `division.faction.isHostile(to: viewerFaction) == true`
  - 且 `isDivisionVisible(..., viewerFaction:) == true`
- 新增只读字段展示可见但非敌对的非己方军队：
  - 非 viewer faction
  - 非 hostile
  - 当前可见
- `RegionInspectorView` 保留“可见敌军”行，并新增“可见非敌对军队”行。
- 不改变 `MapDisplayAdapter.isDivisionVisible` 的可见性规则。
- 不改变 `RegionInspectorState.friendlyDivisions` 的己方军队口径。

## 4. 禁止项

- 不改 `Faction.isHostile(to:)` 当前最小实现。
- 不改 `DiplomacyState`、`CommandValidator`、`MovementRules`、`CombatRules`、`SupplyRules`、`RuleEngine`。
- 不把非敌对军队加入合法攻击目标、敌控区、粮道阻断或补员阻断。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 郡县检查器“可见敌军”只显示 hostile 军队。
- 可见但非 hostile 的非己方军队不会消失，而是出现在“可见非敌对军队”。
- 真实攻击、敌控区、道路、粮道、补给和外交关系规则不变。
- 文档明确这是只读检查器口径收口，不是完整同盟/借道制度。
