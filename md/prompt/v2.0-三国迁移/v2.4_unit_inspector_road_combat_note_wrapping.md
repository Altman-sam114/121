# v2.4 军队详情道路机动与接战预判换行兼容层提示词

## 1. 目标

继续强化“武将、道路、交战”的可读链路：`AppContainer.selectedUnitMobilityPreviewNotes` 和 `selectedUnitCombatPreviewNotes` 已经生成大量道路机动、官道接近、敌射威胁、武将对阵、候选目标和风险文本，本轮只调整 `UnitInspectorView` 的展示约束，降低“道路机动”和“接战预判”长行被固定 2 行截断的概率。

## 2. 范围

- 仅修改 `WWIIHexV0/UI/UnitInspectorView.swift` 的 note 展示。
- 可同步文档：
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改 `AppContainer.selectedUnitMobilityPreviewNotes`。
- 不修改 `AppContainer.selectedUnitCombatPreviewNotes`。
- 不修改 `MovementRules`、`CombatRules`、`GeneralInfluence`、`WarCommandExecutor`、`CommandValidator`、`RuleEngine` 或真实执行规则。

## 3. 实现要求

- 保留 `noteSection` 作为统一展示 helper。
- 默认 note 行仍保持 2 行上限，避免“所属武将”等短摘要无意膨胀。
- “道路机动”和“接战预判”调用点传入不限行，让长文本按内容纵向展开。
- 使用 SwiftUI 只读布局修饰，不在 View 中推导道路、敌军、武将、伤害或风险。
- `RootGameView` 外层已有 `ScrollView`，长内容通过现有滚动承接，不新增滚动容器。

## 4. 禁止项

- 不改变移动范围、道路加成、敌控区、攻击、反击、撤退、补给、围城、粮道或武将加成规则。
- 不新增路径安全、攻击合法性、隐藏敌军推断、胜率或敌方回合模拟。
- 不改变 `UnitInspectorView` public 入参。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- 军队详情“道路机动”和“接战预判”note 不再固定截断为 2 行，长摘要可按内容展开。
- “所属武将”等默认 note section 仍保持 2 行上限。
- App 层摘要、真实道路/交战/补给、命令执行和 Codable schema 保持不变。
- 文档明确本轮只是只读 UI 可读性改进，运行时 UI 高度与滚动体验仍需云端 CI、后续 Agent C artifact 复判或人工授权运行检查。
