# v2.4 检查器战略展示名中文化提示词

## 目标

继续推进三国迁移显示层，处理军队详情与郡县详情中玩家可见的战略 raw id / 英文 fallback 残留，让武将、道路、交战相关检查器信息优先显示郡县、方面、防区、战线和要地状态的中文展示名。

## 范围

- `UnitInspectorView` 不再直接显示 `RegionId`、`TheaterId`、`FrontZoneId`、`FrontLineId` 的 `rawValue`。
- `RegionInspectorView` 不再直接显示选中地格和郡县 dominant 方面/防区的 `rawValue`。
- `MapDisplayAdapter` 在只读 adapter 层为检查器生成展示名，优先读取 `RegionNode.name`、`TheaterNode.name`、`FrontZone.name` 和势力简称 fallback。
- `objectiveStatus` 中文化，避免 `None` / `controlled` 出现在玩家可见郡县详情。
- 武将显示名缺失时使用中文占位，不把 `generalId` 直接透出到军队详情。

## 非目标

- 不重命名 `RegionId`、`TheaterId`、`FrontZoneId`、`FrontLineId` 类型或 rawValue。
- 不改变 `GameState`、动态方面、防区、战线、道路、交战或外交规则。
- 不改 JSON schema，不改 Xcode project。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Full。

## 架构依据

- Hex 仍是战术权威；Region/Theater/FrontZone/FrontLine 只在检查器中以展示名呈现。
- `MapDisplayAdapter` 是 SpriteKit/SwiftUI 检查器读取 `GameState` 的只读适配层，适合集中生成展示字符串。
- UI 不应从 typed id 自行 `.rawValue`，避免将兼容 id 泄露为玩家文案。

## 实现要求

1. 在 `RegionInspectorState` 和 `UnitInspectorStrategicState` 中保留 typed id 字段，追加 display name 字段供 UI 使用。
2. 在 `MapDisplayAdapter` 增加只读展示 helper：
   - 郡县：`RegionNode.name`，空时中文 fallback。
   - 方面：`TheaterNode.name`，空名、等于 raw id 或明显内部 id 时用控制势力简称和郡县名 fallback。
   - 防区：`FrontZone.name`，空名、等于 raw id 或明显内部 id 时用势力简称和郡县名 fallback。
   - 战线：用双方 `Faction.shortDisplayName` 生成中文战线摘要。
3. 更新 `UnitInspectorView` / `RegionInspectorView` 消费 display name 字段。
4. 更新 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 和 prompt 索引。

## 轻量检查

- 针对改动文件扫描旧残留：`rawValue ?? "无"`、`None`、`controlled`、`generalId` fallback。
- 改动文件尾随空白和冲突标记扫描。
- prompt 索引差集检查。
- `git diff --check`

## 验收标准

- 军队详情和郡县详情不再直接显示战略 raw id 或英文 objective fallback。
- typed id 仍保留在状态结构中，供内部逻辑和既有测试语义使用。
- 本轮无规则行为变更，重验证交给 `origin/main` 的 GitHub Actions。
