# v2.4 玩家武将宏观军令战术塑形提示词

> 本阶段承接 `v2.4_general_tactic_shaping.md` 和 `v2.4_general_panel_tactic_audit.md`。执行前必须先读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和相关源码。

## 目标

- 让玩家通过武将面板下达的“固守战线 / 进攻郡县”宏观军令，也在执行前经过 `GeneralAgent.plan`。
- 玩家侧 `WarDirectiveRecord`、`GeneralDecisionRecord` 和 `PlayerPlannedOperation` 记录塑形后的最终 tactic。
- `GeneralCommandPanelView` 的计划军令列表显示指令类型、最终战术和目标，让玩家能看到武将如何影响军令。

- 主要文件：
  - `WWIIHexV0/App/AppContainer.swift`
  - `WWIIHexV0/Core/PlayerCommandState.swift`

- 文档文件：
  - `README.md`
  - `md/flow/flow.md`

  - `md/prompt/README.md`

- 不修改 Xcode project、不新增资源、不改 `GeneralAgent` 的塑形算法、不改 `WarCommandExecutor` 或 `RuleEngine`。

## 实现要求

- `AppContainer.submitPlayerDirective` 在刷新 runtime state 和确认 source zone 后，把玩家构造的 `ZoneDirective` 包装为单条 `DirectiveEnvelope`。
- 使用 `GeneralAgent(registry: generalRegistry).plan(envelope:in:)` 获取塑形后的 directive。
- 后续执行、`CommandResultSummary`、`WarDirectiveRecord`、`PlayerPlannedOperation` 都必须使用塑形后的 directive。
- 将 `generalAdjustment.records` 追加到 `GameState.generalRecords`，供 AI 面板和后续审计读取。
- `WarCommandExecutor.execute(... excluding:)` 继续跳过 `PlayerCommandState.micromanagedDivisionIds`，避免覆盖本回合玩家微操单位。
- `PlayerPlannedOperation` 增加可选 `tactic` 字段，旧存档缺字段时应保持兼容。
- `GeneralCommandPanelView` 的计划军令摘要应显示最终 tactic，缺失时显示“未定战术”。

## 禁止项

- 不允许玩家 UI、`GeneralAgent` 或 `GeneralCommandPanelView` 直接修改 hex controller、单位坐标、动态方面、防区、资源、道路或生产队列。
- 不允许让玩家宏观军令绕过 `WarCommandExecutor -> RuleEngine`。
- 不允许玩家宏观军令调用 `TurnManager.runDirectiveTurn` 或自动结束玩家回合。
- 不允许新增本机 Xcode build/test、模拟器、Probe、Smoke、Stage Regression、Dynamic Theater Regression、Full 或性能测试。
