# v2.4 军队道路机动预判提示词

> 本阶段承接 `v2.4_unit_combat_preview.md`。执行前必须先读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和相关源码。

## 目标

- 继续强化“武将、道路、交战”体验，让玩家在军队详情里提前看到当前选中军队的道路机动状态。
- 复用现有 `MovementRules.generalInfluenceSummary` 和 `GeneralInfluence.movementSummary`，展示基础/有效机动、武将官道加成、可达格数和当前位置/郡县官道状态。
- 与上一阶段“接战预判”并列显示，让单支军队层面同时能看道路和交战影响。

## 范围

- 主要文件：
  - `WWIIHexV0/App/AppContainer.swift`
  - `WWIIHexV0/UI/UnitInspectorView.swift`
  - `WWIIHexV0/UI/RootGameView.swift`
- 文档文件：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`
- 不修改 `MovementRules`、`GeneralInfluence`、`CombatRules`、`WarCommandExecutor`、`RuleEngine`、JSON schema 或 Xcode project。

## 实现要求

- 在 `AppContainer` 增加只读派生属性，例如 `selectedUnitMobilityPreviewNotes`。
- 道路机动预判只读取当前 `GameState`，不得执行移动、修路、补给或任何状态写入。
- 预判内容至少包含基础机动、有效机动、可达格数和武将道路加成说明。
- 如果当前位置在官道上，显示当前位置官道状态；否则可显示当前郡县官道数量作为粮道/机动参考。
- `UnitInspectorView` 接收预判文本并以“道路机动”小节展示，和“接战预判”并列。
- `RootGameView` 只负责把 `AppContainer` 派生属性传给 `UnitInspectorView`。

## 禁止项

- 不允许 UI 直接移动军队、修路、改补给或修改 `GameState`。
- 不允许把预判结果写入 `eventLog`、`WarDirectiveRecord` 或任何持久状态。
- 不允许改变道路移动成本、武将道路加成条件、寻路规则或敌控区规则。
- 不允许新增本机 Xcode build/test、模拟器、Probe、Smoke、Stage Regression、Dynamic Theater Regression、Full 或性能测试。
