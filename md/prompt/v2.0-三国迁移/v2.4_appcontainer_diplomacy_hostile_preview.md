# v2.4 AppContainer 外交敌对预览兼容层提示词

## 1. 目标

继续推进三国多势力、武将、道路和交战迁移：`RegionInspector` 已按 `DiplomaticRelation.status` 区分可见敌军与非敌对军队，但 `AppContainer` 中的军队详情、武将面板、计划军令摘要和点击日志仍有若干只读预览使用静态 `Faction.isHostile(to:)`。本轮只让这些 UI 文本/预览使用 `DiplomacyState` 的敌对关系 helper，避免中立、停战、同盟或共同作战势力在只读摘要里被误称为敌军。

## 2. 范围

- 修改 `WWIIHexV0/App/AppContainer.swift`：
  - 增加本地只读 `isDiplomaticallyHostile(_:to:)` helper。
  - 军队接战预判、武将麾下军队摘要、武将目标预览、武将影响摘要、计划军令官道受压/近敌摘要、移动预览可见敌控过滤和非己方军队点击日志使用外交敌对口径。
- 同步文档：
  - `README.md`
  - `md/flow/flow.md`
  - `md/flow/flowchart.md`
  - `md/prompt/README.md`
  - `update_log.md`

## 3. 实现要求

- `AppContainer` helper 调用 `gameState.diplomacyState.isHostile(between:and:)`，继续复用 `DiplomacyState` 内部的缺建档 fallback。
- 只替换只读 UI/预览/日志筛选：
  - `selectedUnitCombatPreviewNotes`
  - `selectedGeneralAssignedDivisionRows`
  - `selectedGeneralTargetPreviewNotes`
  - `selectedGeneralInfluenceNotes`
  - `generalTargetHexIsVisiblyPressured`
  - `nearestVisibleHostileSummary`
  - `plannedOperationNearestHostileSummary`
  - `plannedOperationHexIsPressured`
  - `isVisibleHostileZoneOfControl`
  - `visibilityFilteredPreviewState`
  - 非己方军队点击日志的“敌军/非敌对军队”措辞。
- 保留玩家视角可见性过滤，不泄露迷雾内军队。

## 4. 禁止项

- 不修改 `Faction.isHostile(to:)` 语义。
- 不修改真实攻击、移动、道路、粮道、部署、补给、伤害、撤退或反击规则。
- 不修改 `CommandValidator`、`RuleEngine`、`WarCommandExecutor`、`MovementRules`、`SupplyRules`、`WarDeploymentManager`。
- 不修改地图点击自动提交攻击、攻击高亮、武将宏观目标可用性或其它需要与当前执行规则保持一致的入口。
- 不实现完整借道、同盟通行、共同作战堆叠、补给共享或外交宣战制度。
- 不新增持久字段，不修改 JSON schema，不改 Xcode project。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试。

## 6. 验收标准

- `AppContainer` 的只读敌军/近敌/受压/接战预览与 `DiplomaticRelation.status` 中 hostile / atWar 口径一致。
- 缺外交建档的旧数据仍通过 `DiplomacyState` fallback 到 `Faction.isHostile(to:)`。
- 真实命令提交、攻击合法性、道路通行、补给和部署规则不变。
- 本机只跑轻量检查，重验证交给 GitHub Actions。
