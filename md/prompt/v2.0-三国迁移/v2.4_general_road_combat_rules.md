# v2.4 武将道路与交战规则兼容层

本文记录“三国要注重武将，做好道路、交战”这一小步实现提示词与交付边界。执行前仍必须阅读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和总提示词 `md/prompt/v2.0-三国迁移/codex-v2.0-三国aiagent迁移总提示词.md`。

## 目标

- 让武将分配不只停留在 AI 面板，而是进入道路机动与交战规则。
- 在 `GeneralAssignment` 中保存武将风格和技能快照，供规则层读取。
- 新增 `GeneralInfluence`，只从 `GameState` 推导武将影响，不依赖 UI 或外部 registry。
- `MovementRules` 使用武将道路机动加成调整有效移动上限。
- `CombatRules` 使用武将攻击/防御加成修正交战结果。

## 非目标

- 不实现完整武将技能树、单挑、士气或完整太守系统。
- 不让武将层直接修改 `GameState` 战术权威。
- 不改多势力 turn order。
- 不跑本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 架构边界

```text
GeneralRegistry
  -> GeneralAssignment 快照
  -> GameState.warDeploymentState.frontZones
  -> GeneralInfluence
  -> MovementRules / CombatRules
  -> CommandValidator / RuleEngine
```

- `GeneralInfluence` 只读 `GameState`。
- 移动仍由 `Command.move -> CommandValidator -> MovementRules -> CommandExecutor` 执行。
- 交战仍由 `Command.attack -> CommandValidator -> CombatRules -> CommandExecutor` 执行。
- 武将影响只作为规则修正，不替代道路、地形、补给、围城和兵种克制规则。
