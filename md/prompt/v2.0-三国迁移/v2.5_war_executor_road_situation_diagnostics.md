# v2.5 宏观道路态势审计提示词

## 目标

继续推进“武将、道路、交战”体验，把 `WarCommandExecutor` 宏观军令的“道路审计”从机动值和可达成本扩展到源点/目标点态势，让 AI 防区指令和玩家武将宏观军令能直接解释军队源点、行军目标或攻击目标是否据道、离道、接敌或受 hostile 敌控区压迫。

## 范围

- 修改 `WWIIHexV0/Commands/WarCommandExecutor.swift`。
- 更新 `md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 非目标

- 不改变 `MovementRules` 路径搜索、敌控区、道路加成、移动范围、占领推进或敌对外交判断。
- 不改变 `WarCommandExecutor` 选兵、落点排序、攻击筛选、fallback 或命令执行顺序。
- 不改 `WarDirectiveRecord` Codable schema。
- 不改 `AgentPanelView`、`TurnManager`、`AppContainer` 或 SpriteKit 绘制。
- 不运行本机 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full / 性能测试 / app 启动 / 全量构建。

## 架构依据

- `WarCommandExecutor.roadSelectionDiagnostic` 已在执行前生成道路审计，并通过 `WarCommandExecutionResult.diagnostics` 进入 AI / 玩家 `WarDirectiveRecord.diagnostics`。
- `AgentPanelView` 已按“道路审计”marker 将诊断归入“战术审计”分组。
- `MovementRules.isEnemyZoneOfControl` 已按 `DiplomacyState.isHostile` 判断 hostile 邻接压迫，可只读复用，不重新发明二元敌我判断。

## 实现要求

1. 在 `WarCommandExecutor` 中为道路审计新增私有 helper，输出源点和移动目标点短态势。
2. 源点和目标点态势使用短码：
   - 有官道：`据道`
   - 无官道：`离道`
   - 目标格已有 hostile 军队：追加 `/接敌`
   - 受 hostile 邻接敌控区压迫：追加 `/受压`
3. `.move` 命令的目标点来自移动目的地；`.attack` 命令的目标点来自防守方坐标；`.hold` / `.allowRetreat` 等无目标命令至少显示源点状态。
4. 移动路径成本文案使用 `路径 cost/effectiveMovement`，保持中文玩家可见，不包含 raw id、JSON、英文 schema 或下划线机器字段。
5. 本轮只增加诊断解释，不改变真实移动合法性、隐蔽敌军、同盟借道或完整路径安全制度。
