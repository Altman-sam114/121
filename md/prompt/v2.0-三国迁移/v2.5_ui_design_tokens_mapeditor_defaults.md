# v2.5 UI 视觉 token 与 MapEditor 默认三国模板小步

## 目标

本轮作为 v2.5 发布级三国 UI、美术和交互收口的起步切片，先建立共享视觉 token，并把最常见的主 HUD、战报、信息覆盖层和 SpriteKit 地图底色接入三国视觉语义；同时收敛 MapEditor 的默认部队模板和新建/导出文案，使编辑器默认输出贴近官渡三国数据。

## 范围

- 在 `PlatformStyles.swift` 增加 `SanguoDesignTokens`，提供绢帛面板、墨色文字、朱印、玉色、铜色、河道蓝、8pt 面板圆角和 44pt 触控高度。
- `HUDView`、`EventLogView`、`RootGameView`、`NewGameButton`、`InfoPanelToggle` 小范围接入共享 token。
- `TerrainStyle` 增加地图背景色，`BoardScene` 和 `BoardSceneView` 统一读取。
- `MapEditorView` 默认模板改为 `sanguo_unit_templates.json` 中的步卒营、骑军、弓弩营、器械营、守军、舟师。
- `MapEditorViewModel` 默认军队模板、默认军队名称和新建城池名改为三国语义。
- `MapEditorExporter` 导出错误和无命名补给点 fallback 改为中文郡县/粮仓语义。

## 非目标

- 不新增真实图片、头像、旗帜或 asset catalog，避免版权和 Xcode project 风险。
- 不改规则、命令、AI、外交、经济、移动、交战、补给或胜负逻辑。
- 不改 JSON schema、Codable rawValue、旧阿登 fallback 数据或旧 `unit_templates.json`。
- 不把 v2.5 写成发布级 UI 完成；本轮只是视觉与编辑器默认值起步。

## 架构边界

- UI 只读 `GameState`，不直接修改规则状态。
- SpriteKit 仍只负责显示和点击回调，状态修改仍走 `AppContainer -> Command / ZoneDirective -> WarCommandExecutor -> RuleEngine`。
- MapEditor 默认模板 id 对齐 `sanguo_unit_templates.json`，但导出结构仍保持 `ScenarioDefinition` / `RegionDataSet` 兼容。

## 轻量检查要求

- 对改动 Swift 文件优先做 `swiftc -parse`；如跨文件依赖导致不可行，只记录风险，不扩大到全项目构建。
- 做改动文件尾随空白检查。
- 做冲突标记检查。
- 做 MapEditor 旧默认模板和编辑器文案残留扫描。
- 按当前规范不跑 Xcode / XCTest / 模拟器 / Probe / Smoke / Stage Regression / Dynamic Theater Regression / Full。

## 验收标准

- 主 HUD 和战报使用共享三国视觉 token。
- 主覆盖层和地图背景颜色不再各自硬编码同一底色。
- 顶层关键按钮有 44pt 最小触控高度。
- MapEditor 新建军队默认模板不再是旧二战单位 id。
- MapEditor 新建城池、粮仓 fallback 和导出错误不再使用 `City` / `Supply` / `Region` / `Hex` 英文玩家可见文案。
- 文档同步说明本轮只改变显示和编辑器默认值，不改变规则或数据 schema。
