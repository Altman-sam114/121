# v2.4 军队未接敌时敌方射程威胁预判兼容层

## 目标

当选中军队当前射程内没有敌军时，在“接战预判”的最近敌军接近信息中补充敌方射程威胁，让玩家判断自己当前是否已在敌方远程/器械射程内，以及本回合可达的接近位是否会进入敌方射程。

## 范围

- 修改 `WWIIHexV0/App/AppContainer.swift` 的 `combatOutOfRangePreviewNotes` 派生文本。
- 新增只读 helper，复用 `MovementRules.movementRange` 与 `HexCoord.distance` 计算接近威胁。
- 同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/prompt/README.md` 和 `update_log.md`。

## 禁止项

- 不改 `MovementRules` 的移动范围、敌控区、道路机动、移动成本或行动消耗。
- 不改 `CombatRules` 的射程、伤害、反击、撤退、死守、围城或兵种克制公式。
- 不改 `GeneralInfluence` 的道路加成或攻防修正。
- 不改 `CommandExecutor`、`CommandValidator`、`WarCommandExecutor`、`RuleEngine` 或真实命令执行结果。
- 不承诺敌方一定会攻击，也不模拟 AI 下回合行动；本轮只展示静态射程威胁。

## 当前架构依据

- 未接敌接近预判由 `selectedUnitCombatPreviewNotes -> combatOutOfRangePreviewNotes` 生成。
- 当前预判已展示最近敌军、接近武将、接近参考、接近态势、官道接近和官道入射程判断。
- `Division.range` 已用于真实攻击合法性判断；这里仅将同一距离关系转为只读玩家提示，不执行命令、不修改 `GameState`。

## 实现步骤

1. 在 `combatOutOfRangePreviewNotes` 中为最近敌军调用新的只读威胁 helper。
2. helper 读取 `target.range` 和当前双方 hex 距离，显示敌军射程以及我方当前距敌方射程的格数，或当前已在敌方射程内。
3. helper 复用 `MovementRules.movementRange` 统计本回合可达且更接近目标的位置中，有多少会进入敌方射程。
4. 保留既有接战距离、接近武将、接近参考、接近态势、官道接近和官道入射程判断。

## 验收标准

- 选中军队且当前射程内没有敌军但地图上存在敌军时，接战预判显示最近敌军的 `接近威胁：...`。
- 若我方当前位置已在最近敌军射程内，显示当前已在敌方射程内。
- 若本回合可达的接近位会进入敌方射程，显示对应数量。
- 射程内有敌军时既有三目标接战预判保持不变。
- 真实移动、攻击、反击、补给、道路、日志结算、命令校验和动态战略同步行为不变。

## 风险提示

- 本轮没有运行 UI 烟测；新增“接近威胁”文本在窄屏、长军队名和 Dynamic Type 下的换行仍需云端 CI、后续 Agent C artifact 复判或人工授权运行检查。
- 摘要只读展示当前静态射程威胁，不模拟敌方下回合 AI 行动、完整路径安全、同盟借道或真实胜率。
